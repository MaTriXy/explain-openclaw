#### Remove Qwen OAuth integration (qwen-portal-auth) ([`dad68d319`](https://github.com/openclaw/openclaw/commit/dad68d319))

The `qwen-portal-auth` OAuth plugin (`extensions/qwen-portal-auth/`) was deprecated by the Qwen team due to traffic impact on their user base and is removed in this commit. The removal eliminates the complete OAuth lifecycle for this provider: token fetch and storage (`oauth.ts`, `oauth.runtime.ts`), refresh scheduling (`refresh.ts`), and credential synchronization with `external-cli-sync`. Associated test coverage (`refresh.test.ts`) and documentation are removed as well.

**Security posture improvement:** OAuth plugins carry inherent attack surface — token storage, refresh race conditions, CSRF state validation, and secrets management. Removing the plugin entirely (rather than patching individual weaknesses) eliminates these surfaces cleanly. Users who previously relied on qwen-portal-auth for Qwen provider access can migrate to the Alibaba Cloud Coding Plan (Model Studio) provider (`qwen-modelstudio`), which uses a standard API key rather than OAuth.
