#### Clarify archive staging intent ([`20237358d`](https://github.com/openclaw/openclaw/commit/20237358d))

Refactors archive extraction test helpers to document the symlink safety strategy. Extracts `createDirectorySymlink()` and `expectPathMissing()` test helpers and adds a clarifying comment that tar must extract into a private staging directory before merging through safe-open boundary checks — explicitly documenting the defense against symlink traversal attacks. No functional change; the clarification supports the coordinated hardening chain in commits `9c6450882`, `6565ae185`, and `658cf4bd9`.

#### Rename tar archive preflight checker ([`9c6450882`](https://github.com/openclaw/openclaw/commit/9c6450882))

Renames the old function (createTarEntrySafetyChecker) to `createTarEntryPreflightChecker` throughout `src/infra/archive.ts` and `src/agents/skills-install-extract.ts` to better reflect the function's role as a pre-extraction validator rather than a general safety checker. No behavioral change; improves code clarity for future security reviews of the archive extraction path.

#### Extract archive staging helpers ([`6565ae185`](https://github.com/openclaw/openclaw/commit/6565ae185))

Extracts symlink traversal validation logic from `src/infra/archive.ts` into a new dedicated module `src/infra/archive-staging.ts`. The new module consolidates `prepareArchiveDestinationDir()`, `assertNoSymlinkTraversal()`, and `assertResolvedInsideDestination()` alongside `ArchiveSecurityError`. Isolating these security-critical functions into a single module makes the staging boundary explicit and auditable, enabling `658cf4bd9`'s runtime enforcement to build on a clean API.

#### Harden archive extraction destinations ([`658cf4bd9`](https://github.com/openclaw/openclaw/commit/658cf4bd9))

Closes a symlink traversal vector in tar and zip extraction. `prepareArchiveDestinationDir()` now rejects destination directories that are themselves symlinks (instead of silently following them). `withStagedArchiveDestination()` implements two-phase extraction: archives are extracted into a temporary staging directory, then merged into the final destination only after passing all symlink boundary checks. New tests cover destination-symlink and destination-symlink-traversal attack scenarios. Addresses Audit 1 Claim 6 for the skills install extraction path. Relates to GHSAs for archive staging symlink races.

#### Fix stale runtime model reuse on session reset ([`53374394f`](https://github.com/openclaw/openclaw/commit/53374394f))

Adds `stripRuntimeModelState()` in `src/gateway/server-methods/sessions.ts` to clear cached model provider, model name, context token count, and system prompt report when resetting sessions after compaction failures. Without this, a failed run's stale model configuration could persist into the next session, causing unexpected provider selection or token accounting drift. This is a session state hygiene fix that defends against state corruption in multi-turn agent sessions.

#### Secrets: reject exec SecretRef traversal ids across schema/runtime/gateway ([`d30dc28b8`](https://github.com/openclaw/openclaw/commit/d30dc28b8))

Adds multi-layer validation for exec secret reference IDs to prevent path traversal sequences (`../../../`) from reaching the secret resolution path. Three enforcement layers:

1. **Zod schema** (`src/config/zod-schema.core.ts`): Rejects invalid IDs at config parse time via `isValidExecSecretRefId()`.
2. **Runtime** (`src/secrets/ref-contract.ts`, `src/secrets/resolve.ts`): `isValidExecSecretRefId()` enforces the `EXEC_SECRET_REF_ID_PATTERN` regex (`/^[A-Za-z0-9][A-Za-z0-9._:/-]{0,255}$/`) at resolution time.
3. **Gateway** (`src/gateway/protocol/schema/primitives.ts`): Validates IDs at the protocol boundary before they reach runtime.

A 199-line parity test suite (`src/secrets/exec-secret-ref-id-parity.test.ts`) verifies schema/runtime/gateway consistency. Addresses Audit 1 Claim 6 (path traversal in agent dirs) and Audit 2 Claim 1 (config injection).

#### SecretRef: harden custom/provider secret persistence and reuse ([`36d2ae2a2`](https://github.com/openclaw/openclaw/commit/36d2ae2a2))

Part of the SecretRef hardening chain following `d30dc28b8`:

- **Runner**: Strips SecretRef marker headers from resolved models before they propagate to downstream consumers, preventing marker tokens from appearing in provider API payloads.
- **Config**: Projects runtime writes onto the source config snapshot rather than a live reference, preventing race-condition overwrites.
- **Models**: Gates custom provider keys by usable-secret semantics to prevent stale `apiKey` preservation for marker-managed providers.
- **Audit extension**: Extends `src/secrets/audit.ts` to scan the active agent `models.json` path. Adds `MAX_AUDIT_MODELS_JSON_BYTES = 5 * 1024 * 1024` (5 MB) input bound and reports non-regular `models.json` files (e.g., directories) as `REF_UNRESOLVED` findings.

Addresses Audit 1 Claim 6 (path traversal — marker header leakage) and Audit 2 Claim 1 (config injection — stale key reuse prevention).

#### Harden secret-file readers ([`201420a7e`](https://github.com/openclaw/openclaw/commit/201420a7e))

Centralizes secret file reading into a new shared module `src/infra/secret-file.ts` with a `MAX_SECRET_FILE_BYTES = 16 * 1024` (16 KB) size limit, consistent symlink detection, and file-type validation. Previously each channel implemented its own reader with varying safety properties.

Migrated to the shared reader:
- `src/acp/secret-file.ts` (ACP secret files)
- `extensions/irc/src/accounts.ts` (IRC credentials)
- `extensions/nextcloud-talk/src/accounts.ts` (Nextcloud Talk credentials)
- `extensions/zalo/src/token.ts` (Zalo tokens)

The shared reader prevents overread attacks (large file DoS via secret paths) and ensures consistent symlink/type validation across all channels. The `MAX_SECRET_FILE_BYTES` limit is exported and tested for regression.

Addresses Audit 2 Claim 5 (gateway auth hardening — prevents unbounded token retry loops).

#### Implicit streamToParent for mode=run without thread ([`9f5dee32f`](https://github.com/openclaw/openclaw/commit/9f5dee32f))

Fixes ACP session output routing for `mode=run` without thread binding. Previously, spawning an ACP session from a Discord context with `mode=run` and `thread=false` would still post output to Discord because `hasDeliveryTarget` was true when called from a Discord context. This commit adds implicit `streamToParent` routing:

- Checks `isSubagentSessionKey()` and `deliveryContextFromSession()` to determine if the caller is a subagent session.
- Moves the implicit `streamToParent` check to after `resolveSpawnMode()` so both explicit `mode="run"` and omitted mode (which defaults to `"run"` when thread is false) correctly trigger parent routing.
- Prevents agent-to-agent supervision patterns from inadvertently broadcasting results to messaging channels.

The 297-line test expansion in `src/agents/acp-spawn.test.ts` covers the routing scenarios.

#### Pass real workspace to sessions_spawn when workspaceAccess is ro ([`3495563cf`](https://github.com/openclaw/openclaw/commit/3495563cf))

Fixes sandbox workspace path inheritance through `sessions_spawn`. When an agent session runs in a `ro` (read-only) or none-access sandbox with a copied workspace, `sessions_spawn` was passing the sandbox copy path to spawned subagents instead of the original workspace path — causing subagents to operate in the sandbox copy rather than the real workspace.

Adds `spawnWorkspaceDir` parameter to `createOpenClawTools()` options in `src/agents/openclaw-tools.ts`. The parameter defaults to `workspaceDir` but callers can set it to the real workspace path when running in a sandbox copy. `src/agents/pi-tools.ts` and `src/agents/pi-embedded-runner/run/attempt.ts` updated to pass the correct `spawnWorkspaceDir`.

The 373-line test suite in `src/agents/pi-embedded-runner/run/attempt.spawn-workspace.test.ts` covers workspace inheritance through the spawn chain for ro/none access modes. Addresses Audit 1 Claim 6 (sandbox workspace isolation).
