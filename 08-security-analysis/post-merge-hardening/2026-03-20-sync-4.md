#### Hardening: refresh stale device pairing requests and pending metadata ([`8e132aed6`](https://github.com/openclaw/openclaw/commit/8e132aed6e))

Device pairing now expires and refreshes stale pending requests to prevent stale state from accumulating. `refreshPendingDevicePairingRequest()` in `src/infra/device-pairing.ts` handles incoming duplicate pairing requests by refreshing the existing pending entry rather than accumulating multiple stale entries. On state load, `pruneExpiredPending()` (from `src/infra/pairing-files.ts`) runs against the pending request map using a 5-minute TTL (`PENDING_TTL_MS = 5 * 60 * 1000`). Expired pending entries are dropped before any approvals are processed.

The change also adds interactive pairing visibility preservation when a device is superseded — the `refreshPendingDevicePairingRequest()` path retains interactive visibility flags on the refreshed pending entry. Tests span the gateway server approve-supersede path, CLI devices command, the device-pair notify extension, and UI device views.

#### Matrix: guard private-network homeserver access ([`f62be0ddc`](https://github.com/openclaw/openclaw/commit/f62be0ddcf))

The Matrix channel transport layer now enforces SSRF-class protection for homeserver connections. `createMatrixGuardedFetch()` in `extensions/matrix/src/matrix/sdk/transport.ts` wraps the outgoing fetch with an `SsrFPolicy` from `src/infra/net/ssrf.ts`, routing Matrix HTTP calls through `createPinnedDispatcher()` with SSRF blocking enabled. This prevents Matrix clients from being redirected to private-network homeservers via malicious server-side redirects or misconfigured `m.homeserver.base_url` values.

The change is adjacent to Audit 2 Claim 4 (DNS rebinding SSRF via web-fetch): it applies the same SSRF-policy infrastructure already used for web-fetch to the Matrix transport layer, closing a parallel attack surface for homeserver-targeted SSRF.

#### Gateway: harden OpenResponses file-context escaping ([`de9f2dc22`](https://github.com/openclaw/openclaw/commit/de9f2dc227))

A new `src/media/file-context.ts` module centralizes XML injection prevention for OpenResponses file-context blocks. Four new functions are introduced:

- `xmlEscapeAttr()` — XML attribute escaping (escapes `<`, `>`, `&`, `"`, `'`)
- `escapeFileBlockContent()` — neutralizes embedded `<file>` and `</file>` tags in content to prevent tag injection
- `sanitizeFileName()` — strips CRLF from filenames to prevent header/log injection
- `renderFileContextBlock()` — safe, unified file-context block generator using the above helpers

`sanitizeMimeType()` in `src/media-understanding/apply.ts` shifted to lines 72-82 as a result of this commit's reorganization.
