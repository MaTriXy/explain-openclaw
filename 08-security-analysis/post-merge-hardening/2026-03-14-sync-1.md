#### Trusted-Proxy Browser Scope Preservation ([`8661c271e`](https://github.com/openclaw/openclaw/commit/8661c271e9))

Fixes CVE-2026-32302 / GHSA-5wcw-8jjv-m286 (HIGH severity). In `attachGatewayWsMessageHandler`, the condition guarding `clearUnboundScopes()` previously included `!isControlUi` as an additional exclusion: `if (!device && (!isControlUi || decision.kind !== "allow"))`. This meant non-control-UI browser connections in trusted-proxy mode that were correctly authorized (decision = `"allow"`) still had their scopes cleared, breaking the authentication chain. The fix removes the `isControlUi` condition so that any connection where `evaluateMissingDeviceIdentity` returns `"allow"` preserves its scopes regardless of UI context. A new test file `server.auth.browser-hardening.test.ts` covers the scope-preservation behavior for trusted-proxy non-control-UI browser sessions.

#### Mongolian Selector Stripping in Exec Obfuscation Detector ([`b2e21e379`](https://github.com/openclaw/openclaw/commit/b2e21e3792))

Expands the `INVISIBLE_UNICODE_CODE_POINTS` set in `exec-obfuscation-detect.ts` to include Mongolian variation selectors U+180B, U+180C, U+180D, and U+180F. These code points are visually invisible and can be interleaved within command text to break pattern matching in `detectCommandObfuscation` — for example, `c\u180Burl` renders as `curl` but defeats regex-based detection. The fix ensures these selectors are stripped during the `stripInvisibleUnicode` normalization pass before obfuscation patterns are evaluated. A corresponding test in `exec-obfuscation-detect.test.ts` verifies detection is not bypassed by Mongolian selectors.
