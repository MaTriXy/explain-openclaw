#### Recognize localized Windows SYSTEM account in ACL audit ([`944abe0a6`](https://github.com/openclaw/openclaw/commit/944abe0a6))

The `TRUSTED_BASE` set was expanded to recognize localized SYSTEM account names on French (`autorite nt\\système`), German (`nt-autorität\\system`), Spanish (`autoridad nt\\system`), and Portuguese (`autoridade nt\\system`) Windows installations. The `classifyPrincipal()` function gained a NFD diacritics-stripping fallback: when a principal is not matched in `TRUSTED_BASE`, it is Unicode-normalized and accent characters are stripped, allowing unrecognized locale variants to match the base set. The `TRUSTED_SIDS` set includes `s-1-5-18` for SID-based trusted classification; icacls permission-reset commands now use the well-known SID `*S-1-5-18` instead of locale-sensitive "SYSTEM" strings, ensuring consistent permission management across all Windows locales.

#### Unify non-core spawn handling across acp qmd and docker (openclaw#31750) thanks @Takhoffman ([`cd653c55d`](https://github.com/openclaw/openclaw/commit/cd653c55d))

Non-core process spawning for Docker, ACP client, and QMD memory manager was unified through the `resolveWindowsSpawnProgram()` and `materializeWindowsSpawnProgram()` abstractions from `src/plugin-sdk/windows-spawn.ts`. Three new dispatch functions were added: `resolveDockerSpawnInvocation()` in `src/agents/sandbox/docker.ts`, `resolveAcpClientSpawnInvocation()` in `src/acp/client.ts`, and `resolveSpawnInvocation()` in `src/memory/qmd-manager.ts`. All three enforce `windowsHide: true` to prevent spawn window leakage and route `.cmd`/`.bat` wrappers through the correct shell fallback on Windows, reducing argument-handling inconsistencies that could be exploited via shell injection.
