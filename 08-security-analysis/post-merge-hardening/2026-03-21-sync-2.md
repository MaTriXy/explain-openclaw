#### Outbound: skip broadcast channel scan when channel is explicit ([`5e417b44e`](https://github.com/openclaw/openclaw/commit/5e417b44e))

`handleBroadcastAction` in `src/infra/outbound/message-action-runner.ts` was refactored to defer config-level channel scanning only when the broadcast target is unspecified (`"all"`). When an explicit channel is provided, the scan is bypassed entirely. Auth and channel validation remain enforced at resolve-time regardless of this path. No security impact; the change is a performance optimization.

#### Enhance web search provider config validation and compatibility handling ([`b71686ab4`](https://github.com/openclaw/openclaw/commit/b71686ab4))

Added a bundled web search plugin allowlist compatibility layer (`withBundledPluginAllowlistCompat` in `src/plugins/bundled-compat.ts`) to normalize legacy Brave provider configs without emitting validation warnings. In `src/config/validation.ts`, the `ensureCompatConfig` closure lazy-applies the compat transform before config validation runs, ensuring backward compatibility without weakening structural validation checks. A new `src/plugins/runtime/runtime-matrix-boundary.ts` boundary isolates Matrix plugin dynamic loading: `loadMatrixModule` uses safe jiti loader caching and confines dynamic import to the boundary file. **Audit Claim 11** (config validation catches structural errors): compat handling is applied pre-validation, so structural guardrails still fire on malformed configs.

#### Pass incoming prompt to assemble ([`e78129a4d`](https://github.com/openclaw/openclaw/commit/e78129a4d))

Extended the context-engine registry in `src/context-engine/registry.ts` to support an optional `prompt` parameter alongside the existing `sessionKey` for context plugins that benefit from retrieval hints. The legacy compatibility path (`invokeWithLegacyCompat`, `detectRejectedLegacyCompatKeys`, `isLegacyCompatErrorForKey`) was generalized to iteratively detect rejected legacy parameter keys, replacing a brittle per-key approach. The `prompt` value is passed as-is to plugin implementations which own validation; no new prompt-injection vectors are introduced at the registry layer.

#### Add context engine transcript maintenance ([`751d5b784`](https://github.com/openclaw/openclaw/commit/751d5b784))

Added a safe transcript rewriting API for context engines. Engines request rewrites via `runtimeContext.rewriteTranscriptEntries()`; the runtime owns session DAG updates exclusively. Implementation lives in `src/agents/pi-embedded-runner/transcript-rewrite.ts`: `rewriteTranscriptEntriesInSessionManager` applies in-memory rewrites under write-lock with branch isolation; `rewriteTranscriptEntriesInSessionFile` handles durable file-backed sessions. Helper functions `remapEntryId` and `appendBranchEntry` ensure consistent parent/target ID remapping across rewrite operations, preventing transcript corruption. Tool-result truncation was also refactored to route through the rewrite path, ensuring sensitive data handling remains consistent between compaction and maintenance code paths. **Audit Claim 10** (session transcript validation): write-lock acquisition and branch isolation in the new maintenance API prevent concurrent transcript corruption.

#### Add anthropic-vertex provider for Claude via GCP Vertex AI ([`6e20c4baa`](https://github.com/openclaw/openclaw/commit/6e20c4baa))

Integrated GCP Vertex AI as an auth provider for Claude models via new files `src/agents/anthropic-vertex-provider.ts` and `src/agents/anthropic-vertex-stream.ts`. Auth resolution follows secure patterns: `resolveAnthropicVertexRegion` validates the region string against a `^[a-z0-9-]+$` regex; `resolveAnthropicVertexProjectId` reads project ID from the ADC credentials file path (`application_default_credentials.json`) with error handling rather than environment variable injection; `hasAnthropicVertexAvailableAuth` gates provider availability on explicit env flag opt-in or ADC path existence (checked via `fs.existsSync`, no path traversal). Credentials are abstracted as `gcp-vertex-credentials` token type — no raw secrets are exposed to model context. The new auth path integrates cleanly with the existing model-auth profile ordering in `src/agents/model-auth.ts`.

#### Sanitize malformed replay tool calls ([`c3972982b`](https://github.com/openclaw/openclaw/commit/c3972982b))

Added robust sanitization for replay tool calls in `src/agents/pi-embedded-runner/run/attempt.ts` to prevent malformed inputs from reaching the model. `sanitizeReplayToolCallInputs` validates each tool call: name must be ≤64 chars, contain no whitespace, and match the allowlist; ID must be non-empty. `resolveReplayToolCallName` normalizes raw names against the allowlist before use. `replayToolCallHasInput` enforces non-null input presence as a guard. `sanitizeAnthropicReplayToolResults` filters tool results to only those matching a preceding valid tool-use ID from the assistant message, preventing orphaned results from being replayed. In `src/agents/session-transcript-repair.ts`, `repairToolUseResultPairing` was updated to preserve errored assistant message tool results only when `preserveErroredAssistantResults` is explicitly set by the replay sanitizer — closing a path where synthetic invalid `tool_result` blocks could be injected. **Audit Claims 8, 9, 10**: name validation and allowlist matching (Claim 8/9); write-path sanitization before session storage (Claim 10).
