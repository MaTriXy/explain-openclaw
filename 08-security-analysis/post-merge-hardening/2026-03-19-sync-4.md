#### Discord: enforce strict DM component allowlist auth ([`0f0cecd2e`](https://github.com/openclaw/openclaw/commit/0f0cecd2e))

Discord DM component interactions (buttons, model pickers) now enforce an allowlist gate when `dmPolicy` is set to `"allowlist"`. The `ensureDmComponentAuthorized()` function gains an early-exit branch: if `dmPolicy === "allowlist"`, it calls `resolveAllowMatch()` against `ctx.allowFrom` and returns false immediately if the interacting user is not on the list, replying with an ephemeral rejection. Allowlisted users proceed to the existing store-based checks as before. `resolveAllowMatch()` is a new helper that consolidates allowlist normalization and matching logic used by this early-exit path. This prevents a non-allowlisted user from triggering agent component actions in DMs even if they can reach the interaction endpoint.
