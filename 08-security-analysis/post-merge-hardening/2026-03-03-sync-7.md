#### Harden file installs and race-path tests ([`dbbd41a2e`](https://github.com/openclaw/openclaw/commit/dbbd41a2e))

Plugin file installation is hardened against TOCTOU (time-of-check-time-of-use) attacks via symlink retargeting. `installPluginFromFile()` now uses `writeFileFromPathWithinRoot()` instead of raw `fs.copyFile()`, enforcing that the resolved target path stays within the extensions directory even if a symlink is retargeted between validation and write. Path-escaping attempts now throw `outside-workspace` errors at copy time. `installPluginFromPath()` gains a matching test case using a `createRebindableDirectoryAlias()` helper that verifies hardlink alias overwrites are blocked via path-escape detection. Platform-specific test guards (`it.runIf(process.platform !== 'win32')`) are removed from the symlink race coverage, ensuring protections are validated uniformly across all CI platforms.

#### Support legacy install entry fallback ([`fbd1210ec`](https://github.com/openclaw/openclaw/commit/fbd1210ec))

`ensureOpenClawExtensions()` is refactored from async to sync, eliminating a potential ordering hazard during extension bootstrap where async resolution could race with callers that expected synchronous availability. A new `MISSING_EXTENSIONS_ERROR` constant consolidates the error message and troubleshooting URL emitted when built-in extensions are absent, replacing scattered inline strings and providing users with a consistent recovery path. These changes strengthen plugin registration reliability and contribute defense-in-depth for Audit 2 Claim 1 (plugin registration integrity).

#### Strip HTML tags for plain-text messaging surfaces ([`62d0cfeee`](https://github.com/openclaw/openclaw/commit/62d0cfeee))

Introduces `sanitizeForPlainText()` and `isPlainTextSurface()` in `src/infra/outbound/sanitize-text.ts`. The `PLAIN_TEXT_SURFACES` constant enumerates channels that render as plain text (WhatsApp, Signal, SMS, IRC, Telegram, iMessage, Google Chat). Before stripping tags, common LLM-generated inline HTML — `<br>`, `<b>`, `<i>`, `<p>`, `<s>`, `<del>` — is converted to channel-appropriate lightweight-markup equivalents (for example, *bold*, *italic*, ~strike~ for WhatsApp/Signal). Any remaining HTML tags are then stripped. This prevents raw markup from appearing verbatim on plain-text messaging surfaces when LLMs produce formatted responses.

#### Harden plain-text HTML sanitization paths ([`e1bc5cad2`](https://github.com/openclaw/openclaw/commit/e1bc5cad2))

Hardens the outbound delivery pipeline in `src/infra/outbound/deliver.ts`. `normalizeEmptyTextPayload()` is introduced to handle empty-text edge cases cleanly, preventing downstream failures when LLM responses produce blank payloads. The channel normalization path (previously scoped to `normalizeWhatsAppPayload()`) is expanded to cover additional plain-text surfaces, ensuring consistent HTML stripping across channels. A Telegram HTML passthrough is added so messages destined for Telegram — which uses `textMode:"html"` natively — bypass the HTML-stripping pipeline and preserve formatting. Test coverage in `deliver.test.ts` is expanded to cover the full sanitization integration.

#### Bound webhook body read time ([`6df36a8b3`](https://github.com/openclaw/openclaw/commit/6df36a8b3))

Prevents slow-body (Slowloris-style) DoS attacks on the Synology Chat webhook handler. The custom body-read loop is replaced by `readRequestBodyWithLimit()` from the plugin-sdk, enforcing both a `maxBytes: 1_048_576` cap and a `timeoutMs: 30_000` deadline. Timed-out or oversized body reads are handled via `isRequestBodyLimitError()` and `requestBodyErrorToText()`, returning structured error responses rather than hanging the handler indefinitely. This prevents a malicious sender from monopolizing the webhook listener with a slow or never-ending body stream.

#### Harden Docker browser container chromium flags (#23889) ([`a19a7f5e6`](https://github.com/openclaw/openclaw/commit/a19a7f5e6))

Hardens the Docker browser container used for Chromium-based screenshot and PDF rendering tasks. A `dedupe_chrome_args()` function is added to `scripts/sandbox-browser-entrypoint.sh` to deduplicate flags and ensure security-relevant flags are applied exactly once. Explicitly enforced flags include `--disable-3d-apis`, `--disable-gpu`, `--disable-software-rasterizer`, and `--disable-extensions`, reducing the attack surface of the sandboxed Chromium process. Associated changes to `call.ts` and `credentials.ts` align browser session launch parameters with the hardened flag set.
