#### Order bootstrap cache clear after embedded run wait ([`f17f2f918`](https://github.com/openclaw/openclaw/commit/f17f2f918))

`clearBootstrapSnapshot` was previously called before `waitForEmbeddedPiRunEnd` in `ensureSessionRuntimeCleanup`, creating a race window where a dying embedded run could repopulate the bootstrap cache after the clear. The fix moves `clearBootstrapSnapshot` to after `waitForEmbeddedPiRunEnd` returns, ensuring the run has fully stopped before the cache entry is invalidated. This prevents stale bootstrap snapshots from persisting across `/new` or `sessions.reset` turns when embedded-run cleanup races with cache clearing.

#### Stage installs before publish ([`6aa80844b`](https://github.com/openclaw/openclaw/commit/6aa80844b))

`installPackageDir` was refactored to copy incoming plugin files to a temporary staging directory (`stageDir`) under the install base, run `afterCopy` validation against the staged location, then atomic-rename the stage to the final target. `isRelativePathInsideBase` verifies the target path stays within the install base before staging, and `assertInstallBaseStable` confirms the install base real path hasn't been rebound between the initial check and the publish rename. This staging-then-rename pattern prevents partial-replace scenarios and reduces symlink-rebind exposure during install failures.

#### Harden zip extraction writes ([`31acad4e8`](https://github.com/openclaw/openclaw/commit/31acad4e8))

ZIP entry writes now use `buildArchiveAtomicTempPath` to create a same-directory temp file opened with `OPEN_WRITE_CREATE_FLAGS` (which sets `O_EXCL` to prevent overwriting an existing file), then rename the temp to the target path. After the atomic rename, `verifyZipWriteResult` re-opens the destination via `openFileWithinRoot` with `rejectHardlinks: true`, confirming the renamed file has not been hardlink-aliased to a path outside the destination root. This closes GHSA-r54r-wmmq-mh84 (ZIP extraction TOCTOU race) and strengthens Audit 1 Claim 5 (file write safety).

#### Land #39064 from @Narcooo ([`2f59a3cff`](https://github.com/openclaw/openclaw/commit/2f59a3cff))

`isSecureWebSocketUrl` in `src/gateway/net.ts` previously only accepted `ws:` and `wss:` scheme URLs, causing loopback cron announce URLs using `http://` to fail security validation. The fix normalizes `http:` → `ws:` and `https:` → `wss:` before applying the private-network policy, so loopback and private-network addresses are correctly accepted for both WebSocket and HTTP aliases. Non-loopback `http://` URLs remain rejected, maintaining the public-network guard. This partially addresses Audit 2 Claim 4 (DNS rebinding SSRF) by ensuring protocol-alias consistency.

#### Harden plugin HTTP route auth ([`ac86decce`](https://github.com/openclaw/openclaw/commit/ac86decce))

`createGatewayPluginRequestHandler` now accepts an optional `dispatchContext` parameter containing a `gatewayAuthSatisfied` flag, set by the outer gateway auth layer in `server-http.ts` only after token validation succeeds. If a matched gateway-auth route reaches the plugin dispatch layer with `gatewayAuthSatisfied === false`, the handler fails closed — no plugin handler is invoked. This closes GHSA-v865-p3gq-hw6m (encoded-path auth bypass in plugin `/api/channels` route classification) and GHSA-mwxv-35wr-4vvj (gateway plugin auth bypass via encoded dot-segment traversal in protected `/api/channels` paths).

#### Sanitize validation log output to prevent control character injection ([`4c2cb7305`](https://github.com/openclaw/openclaw/commit/4c2cb7305))

Both schema validation error formatters in `src/config/io.ts` now wrap `iss.path` and `iss.message` with `sanitizeTerminalText` before concatenating them into log output. Previously, a crafted config file could embed ANSI escape sequences or other control characters in path or message fields that would propagate unescaped into terminal log streams. This partially addresses Audit 2 Claim 7 (control character injection via log output from external input).

#### Require admin for chat config writes ([`5f8f58ae2`](https://github.com/openclaw/openclaw/commit/5f8f58ae2))

A new shared helper `requireGatewayClientScopeForInternalChannel` in `command-gates.ts` checks whether the requesting gateway client carries a required OAuth scope before allowing commands on internal message channels. Config write commands now gate through this helper with `allowedScopes: ["operator.admin"]`; gateway clients without the `operator.admin` scope receive a rejection reply and the handler returns early. This closes Audit 2 Claim 5 for the `/config write` gateway path, ensuring config mutations require elevated authorization.

#### Prevent plaintext apiKey writes to models state ([`17ab46aed`](https://github.com/openclaw/openclaw/commit/17ab46aed))

`normalizeProviders` in `src/agents/models-config.providers.ts` adds a reverse-lookup step: if an `apiKey` value is a non-env-var-name string (fails the `ENV_VAR_NAME_RE` pattern `/^[A-Z_][A-Z0-9_]*$/`), it calls `resolveEnvApiKeyVarName` to find the canonical env var for that provider and checks whether `process.env[envVarName]` matches the resolved value. If matched, the plaintext secret is replaced with the env var name before the providers object is written to `models.json`. This closes Audit 1 Claim 1 for the models config persistence path (fixes #38757).

#### Harden install base drift cleanup ([`d72734946`](https://github.com/openclaw/openclaw/commit/d72734946))

`resolveInstallPublishTarget` was added to re-verify the install base real path immediately before the staged-publish rename; if the base directory has been moved or rebound since setup, it throws `INSTALL_BASE_CHANGED_ERROR_MESSAGE`, and the caller aborts with `INSTALL_BASE_CHANGED_ABORT_WARNING` without writing outside the original base. If drift is detected before backup cleanup, `INSTALL_BASE_CHANGED_BACKUP_WARNING` is emitted and the backup is left in place as a safe fallback. This prevents install-time TOCTOU attacks where an attacker swaps the install base directory between the initial real-path check and the final rename, partially closing GHSA-77hf-7fqf-f227.

#### Block dangerous override-only env pivots ([`e27bbe498`](https://github.com/openclaw/openclaw/commit/e27bbe498))

The `blockedOverrideKeys` list in the host environment security policy was expanded from 2 entries (`HOME`, `ZDOTDIR`) to 25, adding `GIT_SSH_COMMAND`, `GIT_SSH`, `GIT_PROXY_COMMAND`, `GIT_ASKPASS`, `SSH_ASKPASS`, `EDITOR`, `VISUAL`, `SUDO_EDITOR`, `PAGER`, `MANPAGER`, `PERL5DB`, `OPENSSL_CONF`, `PYTHONSTARTUP`, `WGETRC`, `CURL_HOME`, and others. A new `blockedOverridePrefixes` field was also added with entries `GIT_CONFIG_` and `NPM_CONFIG_`, checked by `isDangerousHostEnvOverrideVarName`. This substantially extends Audit 2 Claim 8 (env injection hardening) coverage and adds additional depth to the Legitimate Gap 1 blocklist beyond the earlier closures.

#### Harden fs-safe copy writes ([`74ecdec9b`](https://github.com/openclaw/openclaw/commit/74ecdec9b))

`copyFileWithinRoot` now writes source content to a temp path generated by `buildAtomicWriteTempPath` (opened with `O_EXCL` to prevent clobbering existing files), then atomically renames the temp to the final destination. After the rename, `verifyAtomicWriteResult` calls `openFileWithinRoot` with `rejectHardlinks: true` to confirm the renamed file hasn't been hardlink-aliased outside the root directory. This closes GHSA-x82f-27x3-q89c (TOCTOU symlink race in `writeFileWithinRoot`) and GHSA-cfvj-7rx7-fc7c (stageSandboxMedia destination symlink traversal), and strengthens Audit 1 Claim 5 across the file copy path.

#### Harden workspace skill path containment ([`253e15970`](https://github.com/openclaw/openclaw/commit/253e15970))

`loadWorkspaceSkillEntries` and `syncSkillsToWorkspace` in `src/agents/skills/workspace.ts` now verify that each skill directory entry resolves within the workspace root before processing; symlinked skill directories that escape the workspace root via `fs.realpath` are silently skipped rather than loaded or synced. Test coverage explicitly exercises the symlink-escape case, confirming that a skill linked to a path outside the workspace is excluded from both the skill prompt and the synced workspace. This closes Audit 1 Claim 6 (path traversal) for the workspace skill sync path.

#### Keep gateway tokens out of URL storage ([`10d0e3f3c`](https://github.com/openclaw/openclaw/commit/10d0e3f3c))

The web dashboard's `persistSettings` function now uses the `PersistedUiSettings` type, which explicitly omits the `token` field, so gateway auth tokens are never written to `localStorage` under the `openclaw.control.settings.v1` key. On the macOS side, `GatewayEndpointStore.swift` was updated to append the gateway token as a URL fragment (`#token=…`) rather than a query parameter (`?token=…`), keeping the credential out of server logs and browser history. Any legacy persisted token is scrubbed from `localStorage` on load. This strengthens Audit 1 Claim 1 (credential storage) across both the browser and native macOS clients.

#### Gate methods before auth lockout accounting ([`44820dcea`](https://github.com/openclaw/openclaw/commit/44820dcea))

The HTTP method guard for `/hooks/*` endpoints in `src/gateway/server-http.ts` was moved to execute before the `safeEqualSecret` token check and its associated auth-failure accounting. Previously, a non-POST request to `/hooks/wake` with a wrong token would still consume the auth lockout budget, allowing an attacker to exhaust the budget with repeated GET/DELETE requests and block legitimate webhook delivery. Now, non-POST requests return `405 Method Not Allowed` immediately without touching the failure counter. This closes an auth-throttle budget exhaustion vector related to Audit 2 Claim 5 (RBAC enforcement on gateway endpoints).

#### Honor commands.allowFrom in guild slash auth ([`262fef6ac`](https://github.com/openclaw/openclaw/commit/262fef6ac))

Discord guild slash-command pre-dispatch authorization now checks `commands.allowFrom.discord` (and the global `commands.allowFrom["*"]` fallback) before rejecting a sender as unauthorized. Previously the allowlist was not consulted during guild slash auth, causing allowlisted senders to be incorrectly rejected. New test coverage exercises both the provider-specific and global allowFrom variants, as well as the rejection case for non-listed senders. This closes Audit 2 Claim 5 (RBAC) for the Discord guild slash-command path.
