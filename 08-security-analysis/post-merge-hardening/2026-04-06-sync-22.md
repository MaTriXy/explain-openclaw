#### Move bootstrap channel logic behind extension seams ([`bc457fd1b`](https://github.com/openclaw/openclaw/commit/bc457fd1b))

`getBundledChannelContractSurfaceModule()` (`src/channels/plugins/contract-surfaces.ts:199`) is now the single entry point through which core loads extension-specific bootstrap logic. `src/config/channel-configured.ts` is refactored to call `hasConfiguredState()` and `hasPersistedAuthState()` via this surface rather than hardcoding env var checks; `src/config/telegram-command-config.ts` is introduced as a new wrapper that loads the telegram command config contract surface the same way. `collectBlueBubblesStatusIssues()` (`src/plugin-sdk/bluebubbles.ts:313`) is moved from a core status-issues file to the extension-owned plugin SDK facade, so its implementation is co-located with the channel that owns it. `expectCoreSourceStaysOffPluginSpecificSdkFacades()` (`src/channels/plugins/contracts/channel-import-guardrails.test.ts:477`) is added as a compile-time boundary test preventing core production code from re-importing plugin-specific SDK facades (e.g., `src/plugin-sdk/telegram-command-config`, `src/plugin-sdk/matrix-runtime-heavy`). No bypass paths are introduced; existing validation logic is unchanged. No audit claims addressed.

#### Move channel-specific config surfaces out of core ([`1246e2b03`](https://github.com/openclaw/openclaw/commit/1246e2b03)) — Escalation confirmed

Companion commit to `bc457fd1b`. WhatsApp-specific config types and JID regexes were removed from `src/plugin-sdk/channel-config-helpers.ts`, relocated into extension-owned `config-accessors.ts` files (iMessage, WhatsApp). Adds `secret-contract.ts` files to the BlueBubbles, IRC, MS Teams, and Nextcloud Talk extensions, giving each channel a formal surface for secret and config access; these surfaces are reachable through the `getBundledChannelContractSurfaceModule()` entry point already established by `bc457fd1b`. The companion test file `src/plugin-sdk/channel-config-helpers.test.ts` was also removed. Together with `bc457fd1b`, this completes the channel bootstrap boundary refactor — channel-specific config and secret surfaces now live in extension modules rather than in the core plugin SDK. Defense-in-depth; no audit claims addressed.
