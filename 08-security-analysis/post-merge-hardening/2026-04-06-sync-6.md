### Audit Cross-Reference

No audit claims (Audit 1 or Audit 2) directly addressed in this batch. No legitimate gaps closed.

**`ddd250d13`** introduces per-agent skill allowlist enforcement as a new capability control. The `agents.defaults.skills` and per-agent `agents.list[].skills` config fields are processed by `resolveEffectiveAgentSkillFilter()` to determine which skills each agent may invoke at runtime. The `matchesSkillFilter()` helper detects configuration drift, and skill snapshots now cache the applied filter and refresh when it changes via `shouldRefreshSnapshotForVersion()`, preventing privilege escalation where an agent could access skills beyond its configured allowlist. This control propagates through agent command execution, sandbox context setup, Pi embedded runtime, and auto-reply skill command resolution.

**`865fa2ba7`** narrows `auth_permanent` classification to high-confidence unrecoverable failures — revoked/disabled API keys, deactivated accounts, and org-level OAuth denials. Ambiguous signals (bare `invalid_api_key`, generic `permission_error`) are demoted to the shorter `auth` cooldown lane. New config knobs `authPermanentBackoffMinutes` (10 min default) and `authPermanentMaxMinutes` (60 min default) replace the prior blanket 60 min lockout, reducing false-positive permanent lockouts and allowing faster failover recovery during transient provider auth incidents. The `resolveAuthCooldownConfig()` function resolves both backoff and max parameters independently.

**`d6e89f96d`** extracts `resolveEffectiveSharedGatewayAuth()` from inline config handler logic and exports it as a public helper in `src/gateway/auth.ts`, consolidating shared gateway auth resolution (token vs password modes) used across config change detection (`didSharedGatewayAuthChange()` at `src/gateway/server-methods/config.ts:225`) and test infrastructure. A new `src/gateway/server-methods/config.test-helpers.ts` provides `createConfigHandlerHarness()`, `createConfigWriteSnapshot()`, and `flushConfigHandlerMicrotasks()` factory functions that decouple test setup from implementation details.
