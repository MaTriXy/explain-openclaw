#### Fail closed talk provider selection ([`b7ad8fd66`](https://github.com/openclaw/openclaw/commit/b7ad8fd66))

`TalkSchema` in `src/config/zod-schema.ts` gains a `.superRefine()` validation that enforces provider consistency at config load time. Two new checks:
1. If `talk.provider` names a provider not present as a key in `talk.providers`, Zod emits a custom issue (`talk.provider must match a key in talk.providers`).
2. If `talk.providers` defines multiple entries but `talk.provider` is unset, Zod requires disambiguation (`talk.provider is required when talk.providers defines multiple providers`).

Previously, a misconfigured multi-provider block could silently fall through to an unexpected provider. The new `TalkProviderEntrySchema` validates each provider entry. Secret fields use `SecretInputSchema` for consistency with the schema-aligned secret handling introduced in commit `87640f9a6`.

#### Restore OpenRouter compatibility for Perplexity (#39937) ([`28e46d04e`](https://github.com/openclaw/openclaw/commit/28e46d04e))

The web search tool now routes Perplexity requests to different base URLs based on API key prefix. `PERPLEXITY_KEY_PREFIXES = ["pplx-"]` routes to `PERPLEXITY_DIRECT_BASE_URL` (`https://api.perplexity.ai`). `OPENROUTER_KEY_PREFIXES = ["sk-or-"]` routes to `DEFAULT_PERPLEXITY_BASE_URL` (`https://openrouter.ai/api/v1`). Keys not matching either prefix default to OpenRouter.

The routing logic at `src/agents/tools/web-search.ts` uses prefix matching against the normalized API key to pick the transport. Authentication headers (`HTTP-Referer`, `X-Title`) are applied correctly per endpoint. This prevents auth failures when OpenRouter keys (prefixed `sk-or-`) are used to reach Perplexity via the OpenRouter gateway, which broke when a previous change hardcoded the direct endpoint.

#### Resolve live config paths in status and gateway metadata ([`d9e8e8ac1`](https://github.com/openclaw/openclaw/commit/d9e8e8ac1))

The `models status` command and gateway health state metadata previously used the compile-time constant `CONFIG_PATH` for the config file path display. This commit replaces all static references with `createConfigIO().configPath` — a runtime resolution call — so the actual path seen by the running process is reported, not the default baked at build time.

This matters in deployments where the config path is customized (e.g., via env var or alternate config dir). The old behavior could show a path that the process was not actually using, which could mislead operators during debugging or audit. `src/commands/models/list.status-command.ts` now calls `createConfigIO().configPath` at command invocation time and propagates it through the JSON status output and CLI display.

#### Add Brave Search LLM Context API mode for web_search ([`8a1015f1a`](https://github.com/openclaw/openclaw/commit/8a1015f1a))

New opt-in mode for Brave Search: when `tools.web.search.brave.mode = "llm-context"`, the tool calls `BRAVE_LLM_CONTEXT_ENDPOINT` (`https://api.search.brave.com/res/v1/llm/context`) instead of the standard search endpoint. The response is handled by `runBraveLlmContextSearch()` and typed as `BraveLlmContextResponse` (grounding results with per-page snippet arrays).

The LLM Context endpoint returns pre-extracted page content optimized for model grounding. The cache key for this mode omits `count` and `ui_lang` parameters (which the LLM Context API does not accept) to avoid unnecessary cache misses. Result content is wrapped with `externalContent: { untrusted: true }` consistent with all other web tool output.

#### Align talk config secret schemas ([`87640f9a6`](https://github.com/openclaw/openclaw/commit/87640f9a6))

`src/gateway/protocol/schema/primitives.ts` gains three new TypeBox schemas that formalize how secrets are referenced in the gateway protocol:

- `SecretRefSourceSchema` — string enum union of `"env"`, `"file"`, `"exec"` describing how a secret is resolved
- `SecretRefSchema` — strict object (`additionalProperties: false`) with `source: SecretRefSourceSchema` and `id: string`, plus optional `key: string` for env/file path refinement
- `SecretInputSchema` — union of `Type.String()` and `SecretRefSchema`, allowing either a literal string or a structured reference

Previously, secret fields in talk config accepted arbitrary shapes; any extra properties would silently pass validation. With `additionalProperties: false` on `SecretRefSchema`, unexpected fields now fail at the schema boundary. This aligns with Audit 1 Claim 1 (plaintext token storage) as a defense-in-depth measure — structured validation makes it harder for malformed secret objects to bypass processing logic.

#### Add @tloncorp/api to pnpm onlyBuiltDependencies allowlist ([`b38f37163`](https://github.com/openclaw/openclaw/commit/b38f37163))

`@tloncorp/api` was added to `pnpm.onlyBuiltDependencies` in `package.json`. This allowlist controls which packages are permitted to run `install`/`postinstall` scripts (native code compilation) during `pnpm install`. Adding a package to this list explicitly trusts it to execute code during installation.

The `@tloncorp/api` package requires native compilation at install time. Without the allowlist entry, pnpm's default behavior (in strict mode) would block the build script. This is a supply chain trust boundary expansion: any native-building package on the list is implicitly trusted to run arbitrary code at install time. The change is intentional and necessary for the dependency to function, but reviewers should verify that `@tloncorp/api` is a well-maintained, audited package with a known release process.
