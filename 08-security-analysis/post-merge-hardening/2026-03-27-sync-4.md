#### Migrate more boundary parsing to zod ([`a1f995053`](https://github.com/openclaw/openclaw/commit/a1f995053))

Extends Zod schema validation to multiple extension and core boundary parsers: `extensions/googlechat/src/accounts.ts`, `extensions/nextcloud-talk/src/monitor.ts`, `extensions/nostr/src/nostr-state-store.ts`, `extensions/synology-chat/src/client.ts`, `src/agents/pi-auth-json.ts`, `src/agents/sandbox/registry.ts`, and `src/auto-reply/reply/strip-inbound-meta.ts`. Each previously used ad-hoc manual parsing or type assertions; they now use Zod's `safeParse()` to validate structure at trust boundaries. Test files (`*.test.ts`) are added for each migrated module to verify schema rejection of malformed inputs. Strengthens defense against malformed payloads reaching internal processing (Audit 2 Claims 3, 4; Audit 1 Claim 3).

#### Adapt compaction request auth ([`b1d853d88`](https://github.com/openclaw/openclaw/commit/b1d853d88))

Fixes a gap where custom auth headers were not forwarded when the compaction safeguard invoked `summarizeChunks()`. Adds `params.headers` to the `summarizeChunks` call in `src/agents/compaction.ts` so that the `withSummaryHeaders()` helper can merge operator-supplied headers with the model's defaults. Without this fix, providers that require custom auth headers (in addition to or instead of an API key) would fail silently during compaction. The `compaction-safeguard.ts` extension is updated to use a unified auth resolution path instead of a separate API key + header fallback pair (Audit 1 Claims 4, 5).

#### Restore codex oauth refresh fallback ([`490b2f881`](https://github.com/openclaw/openclaw/commit/490b2f881))

Restores the OpenAI Codex OAuth token refresh fallback in the provider runtime contract. The test mock was previously pointing to the wrong module path (`extensions/openai/src/openai-codex-provider.runtime.js` vs. `extensions/openai/openai-codex-provider.runtime.js`) and mocking `getOAuthApiKey` instead of `refreshOpenAICodexToken`. Both are corrected so the OAuth refresh path is exercised under test. A config-misc test is updated to use a non-deprecated config key (`memorySearch.provider` instead of removed legacy `agents.list` + `routing.allowFrom`), removing a false-positive legacy-config test scenario (Audit 1 Claims 4, 8).

#### Use session origin provider in chat delivery routing ([`6e5bc5647`](https://github.com/openclaw/openclaw/commit/6e5bc5647))

Ensures that `chat.send` uses the session's stored origin provider metadata (`session.origin.provider`, `session.origin.accountId`, `session.lastTo`) when routing delivery for CLI-mode clients, not just the connect metadata. Before this fix, CLI sessions could lose the originating channel context and route delivery incorrectly. A new test case verifies that `OriginatingChannel`, `OriginatingTo`, and `AccountId` are correctly populated in the dispatch context when the session carries origin metadata. Prevents routing to the wrong provider account (Audit 2 Claim 5; Audit 1 Claim 6).

#### Add zod helpers for json file readers ([`35b132884`](https://github.com/openclaw/openclaw/commit/35b132884))

Introduces `src/utils/zod-parse.ts` with two utility functions: `safeParseWithSchema<T>()` (validates a parsed value against a Zod schema, returns `T | null`) and `safeParseJsonWithSchema<T>()` (parses raw JSON then validates, returns `T | null`). These replace scattered try/catch + manual validation patterns in `src/config/sessions/store-read.ts`, `src/infra/device-auth-store.ts`, `src/infra/gateway-lock.ts`, and `src/shared/tailscale-status.ts`. Each file now parses its JSON file with a Zod schema, returning `null` on malformed input rather than throwing or returning unvalidated data. Test coverage is added for `store-read.ts`. Addresses Audit 2 Claim 7 (JSON file readers lack schema validation).

#### Share zod setup validators across channels ([`e6c5ce136`](https://github.com/openclaw/openclaw/commit/e6c5ce136))

Introduces `createZodSetupInputValidator<T>()` in `src/channels/plugins/setup-helpers.ts` and exports it via `src/plugin-sdk/setup.ts`. Nine channel extensions (bluebubbles, googlechat, irc, line, mattermost, nextcloud-talk, signal, tlon, zalo) are migrated from duplicated inline input parsing to this shared validator in their `setup-core.ts` files. The shared validator uses Zod's `safeParse()` internally and surfaces the first issue message on failure. Eliminates parser divergence risk across channel setup flows where a single un-validated field could have led to inconsistent error handling (Audit 2 Claim 1).

#### Parse extension event payloads with zod ([`1d1f36adf`](https://github.com/openclaw/openclaw/commit/1d1f36adf))

Migrates ACPX (`extensions/acpx/src/runtime-internals/events.ts`) and Mattermost (`extensions/mattermost/src/mattermost/monitor-websocket.ts`) extension event payload parsing to use Zod schemas instead of `as unknown as T` type casts or manual property checks. Malformed event payloads from external sources now return `null` from the Zod parse step rather than reaching event processing with potentially unsafe structure (Audit 2 Claim 3).

#### Avoid stale sessions_send reply carryover ([`5b4669632`](https://github.com/openclaw/openclaw/commit/5b4669632))

Fixes a phantom reply delivery bug in the `sessions_send` tool (`src/agents/tools/sessions-send-tool.ts`). Before starting an agent run, the tool now fetches the last `SESSIONS_SEND_REPLY_HISTORY_LIMIT` (50) messages from the session and snapshots the most recent assistant reply via `resolveLatestAssistantReplySnapshot()`. After the run completes, it compares the post-run reply fingerprint against the baseline; if unchanged, it suppresses delivery, preventing the prior turn's reply from being re-delivered as if the agent responded to the new invocation.

#### Avoid stale subagent reply carryover ([`6ef6e80ab`](https://github.com/openclaw/openclaw/commit/6ef6e80ab))

Applies the same baseline-snapshot pattern to the subagent control path (`src/agents/subagent-control.ts`). When a subagent run completes, the result is only reported as a new reply if the session history advanced past the pre-run baseline. Prevents stale reply carryover when a subagent exits without producing output and the prior turn's response would otherwise propagate to the orchestrating agent as if it were new.

#### WSL2 Ollama networking and provider discovery diagnostics ([`c959ac3a2`](https://github.com/openclaw/openclaw/commit/c959ac3a2))

Improves error diagnostics for Ollama in WSL2 environments where `undici` wraps network errors as `TypeError: fetch failed` with the real error in `.cause`. Updates `src/agents/ollama-stream.ts` to extract and log the nested cause (`err.cause.message`) when surfacing stream errors. Updates `src/agents/models-config.providers.discovery.ts` to include the nested cause in discovery failure warnings. Also updates the undici global dispatcher test to include the WSL module in its mock set. Improves operator visibility into WSL2 networking failures without changing error handling behavior (Audit 2 Claim 6).
