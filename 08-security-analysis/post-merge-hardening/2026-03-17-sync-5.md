#### Add speech provider registration ([`662031a88`](https://github.com/openclaw/openclaw/commit/662031a88))

TTS provider selection transitions from a hardcoded enum (`elevenlabs | openai | microsoft`) to a plugin-based registry. Three new registry functions are introduced: `listSpeechProviders()` returns all registered providers, `getSpeechProvider()` resolves a provider by ID, and `normalizeSpeechProviderId()` normalizes input (trim, lowercase, legacy `"edge"` → `"microsoft"` alias). The `TtsProviderSchema` validator changes from an enum to string-based validation against registered IDs, so provider validation now queries the registry at runtime. The gateway's `setProvider` handler validates incoming provider IDs against the registry before accepting the change.

No injection vectors introduced — provider IDs pass through normalization before any dispatch. No direct audit claim impact.
