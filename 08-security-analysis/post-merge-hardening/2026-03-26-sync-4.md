#### Block reset-profile on lower-privilege browser request surfaces ([`1b3a1246d`](https://github.com/openclaw/openclaw/commit/1b3a1246d))

`isPersistentBrowserProfileMutation()` now includes `reset-profile` alongside `create-profile` and `delete-profile` in its list of persistent browser profile mutation operations. Gateway calls to `reset-profile` from lower-privilege request surfaces (those lacking admin scope) are now blocked. Previously only profile creation and deletion were gated; resetting a persistent browser profile to defaults was unguarded for non-admin gateway clients.

#### Fix/telegram writeback admin scope gate ([`b7d70ade3`](https://github.com/openclaw/openclaw/commit/b7d70ade3))

Introduces `TELEGRAM_ADMIN_SCOPE` constant (`"operator.admin"`) and threads `gatewayClientScopes` through the Telegram send handler and delivery queue pipeline. Config and cron writeback for resolved Telegram chat IDs is now gated: if the gateway caller lacks `operator.admin` scope, the writeback is skipped with a log entry. This prevents lower-privilege gateway callers from persisting resolved Telegram chat IDs to configuration, closing a privilege escalation vector via persistent target resolution.

#### Honor sandbox alsoAllow and explicit re-allows ([`edb5123f2`](https://github.com/openclaw/openclaw/commit/edb5123f2))

Extracts sandbox tool policy resolution into `resolveEffectiveSandboxToolPolicyForAgent()`. Previously, agents passing legacy session aliases could lose their `alsoAllow` lists during policy resolution, causing sandbox tool restrictions to be applied more broadly than intended (or inconsistently across tool invocation points). The refactored function correctly merges default sandbox allowlists with agent-specific `alsoAllow` lists, ensuring policy is applied consistently regardless of how the agent session is referenced.

#### Tests: isolate security audit home skill resolution ([`da60aff17`](https://github.com/openclaw/openclaw/commit/da60aff17))

Hardens security audit test isolation by mocking `os.homedir()` and restoring path-resolution environment variables (HOME, USERPROFILE, HOMEDRIVE, HOMEPATH, OPENCLAW_HOME, OPENCLAW_STATE_DIR, OPENCLAW_BUNDLED_PLUGINS_DIR) in cleanup. Prevents leaked environment variables from polluting test state in ways that could mask auth path resolution bugs in CI environments.

#### Apply host-env blocklist to auth-profile env refs in daemon install ([`d1bfe0842`](https://github.com/openclaw/openclaw/commit/d1bfe0842))

Adds `isDangerousHostEnvVarName` and `isDangerousHostEnvOverrideVarName` checks to the daemon install path for auth-profile environment variable references. When building daemon startup environment from auth-profile `env` refs, `normalizeEnvVarKey` normalizes the key and both blocklist functions are consulted before allowing the variable to propagate. Variables matching dangerous patterns (including `LD_*`, `DYLD_*`, and related override names) are rejected, preventing env injection via the daemon install path. This partially addresses Legitimate Gap 1 on the daemon install side; the gateway-side env var blocklist remains open.

#### Enforce operator.admin scope on /voice set config writes ([`e34694733`](https://github.com/openclaw/openclaw/commit/e34694733))

Adds a scope gate in the talk-voice extension: `/voice set` commands executed in webchat context now check `ctx.gatewayClientScopes?.includes("operator.admin")` before writing voice configuration. Gateway clients without `operator.admin` scope receive a warning response and the config write is rejected. This extends the RBAC enforcement pattern to the voice configuration surface, consistent with the Telegram writeback gate above.

#### Reject path traversal and home-dir patterns in media parse layer ([`4797bbc5b`](https://github.com/openclaw/openclaw/commit/4797bbc5b))

Adds `hasTraversalOrHomeDirPrefix()` to `src/media/parse.ts` as a shared guard for both `isLikelyLocalPath()` (for media approval) and `isValidMedia()` (for media validation). Any candidate containing `../`, `..\`, or `~` as a prefix is rejected before reaching downstream load or send logic. The bare-filename fallback path in `isValidMedia()` also now rejects traversal/home-dir patterns, closing a bypass where `../../.env` could match the `HAS_FILE_EXT` pattern and be treated as a valid bare filename.

#### Extract codex auth identity helper ([`c3d1dbc69`](https://github.com/openclaw/openclaw/commit/c3d1dbc69))

Refactors OpenAI Codex JWT token parsing and identity resolution into a dedicated `resolveCodexAuthIdentity()` helper in `extensions/openai/openai-codex-auth-identity.ts`. Centralizes email extraction and profile ID derivation from JWT payload, reducing token handling duplication across the OAuth flow. No new security controls introduced; this is a code quality improvement that makes the identity resolution code easier to audit.

#### Separate profile ids from email metadata ([`d363af8c1`](https://github.com/openclaw/openclaw/commit/d363af8c1))

Separates profile identity via `buildAuthProfileId()` from email and display name metadata. Adds a `displayName` field to auth profiles and credential types so user-facing display labels are independent of profile identifiers or email addresses. No new security controls; this is an auth data model improvement that reduces metadata coupling and clarifies profile identity semantics.

#### Align outbound media access with fs policy ([`c92002e1d`](https://github.com/openclaw/openclaw/commit/c92002e1d))

Replaces `getAgentScopedMediaLocalRoots(cfg, agentId)` with `getAgentScopedMediaLocalRootsForSources({cfg, agentId, mediaSources})` in the outbound delivery path. Adds `collectPayloadMediaSources()` to extract media URLs from the actual reply payloads being sent. By scoping media local root resolution to the specific media sources in the payload, the outbound layer cannot access media roots that are not relevant to the current delivery context, tightening the fs policy boundary for outbound media operations.
