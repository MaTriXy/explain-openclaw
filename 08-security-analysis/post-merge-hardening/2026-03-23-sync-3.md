#### Allowlist Entry Whitespace Normalization ([`b4656f193`](https://github.com/openclaw/openclaw/commit/b4656f193a))

The shared `createPairingPrefixStripper()` utility in `pairing-adapters.ts` now calls `.trim()` both before and after the prefix-stripping regex, closing a whitespace-based bypass vector in DM policy enforcement. Previously, an allowlist entry stored as `"  user@example  "` would fail to match a valid entry `"user@example"` after prefix stripping, potentially allowing unauthorized access if the comparison was against a trimmed form. The same `.trim()` normalization was applied to `normalizeEntry` and `normalizeAllowEntry` in the Nextcloud Talk, Zalo, Slack, Discord, Nostr, and ZaloUser channel plugins.

#### sessions.steer Added to operator.write Scope ([`ad24fccff`](https://github.com/openclaw/openclaw/commit/ad24fccff5))

`sessions.steer` is added to the `operator.write` scope group in `METHOD_SCOPE_GROUPS` within `method-scopes.ts`. This ensures the dashboard session-steering gateway method is gated under write-level authorization, following the same `authorizeGatewayMethod()` enforcement pattern used for all other gateway methods. Without this entry, `sessions.steer` would be absent from scope validation.
