#### Browser Profile Mutation Blocked via browser.request ([`f37815b32`](https://github.com/openclaw/openclaw/commit/f37815b32))

The gateway now blocks persistent browser profile creation and deletion via the `browser.request` API. Two new guards were added to `src/gateway/server-methods/browser.ts`: `normalizeBrowserRequestPath()` trims leading/trailing slashes to prevent path-normalization bypasses, and `isPersistentBrowserProfileMutation()` rejects POST `/profiles/create` and DELETE `/profiles/*` requests before they reach the node binding. This fix is referenced as GHSA-vmhq in the changelog and closes a lateral API surface for profile manipulation.

#### Spawned Session Workspace Isolation ([`46a332385`](https://github.com/openclaw/openclaw/commit/46a332385))

Moves `spawnedBy` and `workspaceDir` parameters from the public `agent` gateway method to the internal `sessions.patch` gateway method in `src/gateway/sessions-patch.ts`, then strips them from agent RPC params before invoking subagent spawning in `src/agents/subagent-spawn.ts`. The `spawnedWorkspaceDir` field is persisted in session store (`src/config/sessions/types.ts`) and validated only for subagent sessions via `supportsSpawnLineage()`. A new `cleanupFailedSpawnBeforeAgentStart()` function prevents orphaned workspace directories on spawn failure. This closes a boundary where external callers could supply workspace directory overrides through the public agent API.

#### Chat Abort Authorization Boundary ([`c5ea6134d`](https://github.com/openclaw/openclaw/commit/c5ea6134d))

Adds device/connection ownership enforcement for chat run aborts via the `ChatAbortRequester` type in `src/gateway/server-methods/chat.ts`, which tracks `connId`, `deviceId`, and `isAdmin` flags. A new test file `src/gateway/server-methods/chat.abort-authorization.test.ts` validates that explicit run aborts require matching owner device ID and conn ID, blocking cross-device takeover attempts. Also adds `sanitizeUsage()` to validate usage metadata from transcripts, preventing UI crashes from malformed JSON values in usage fields.

#### Exec Approval Invisible Character Escaping ([`2504cb6a1`](https://github.com/openclaw/openclaw/commit/2504cb6a1))

The `UNICODE_FORMAT_CHAR_REGEX` pattern (`/\p{Cf}/gu`) and `sanitizeExecApprovalDisplayText()` function in `src/infra/exec-approval-command-display.ts` now escape invisible Unicode format characters (zero-width spaces U+200B, soft hyphens, etc.) in exec approval display payloads before broadcast. Sanitization is applied to command text and commandPreview fields. Characters are encoded as `\u{...}` notation to make injection attempts visible in logs and approval UIs, preventing attackers from disguising malicious commands as benign ones.

#### Sandbox Path Traversal via Symlink Anchoring ([`e95f2dcd6`](https://github.com/openclaw/openclaw/commit/e95f2dcd6))

Fixes symlink-based path traversal in sandbox fs-bridge write and stat operations. `src/agents/sandbox/fs-bridge-path-safety.ts` adds the `AnchoredSandboxEntry` type and `resolveAnchoredSandboxEntry()` method, which resolves the parent directory canonically before constructing the full path. File write plans in `src/agents/sandbox/fs-bridge-shell-command-plans.ts` now receive an `anchoredTarget: AnchoredSandboxEntry` that uses parent + basename (with CD to parent) instead of the full file path, preventing symlink resolution races between the path check and actual file write.

#### Exec Allowlist Glob Matching Tightened ([`82e3ac21e`](https://github.com/openclaw/openclaw/commit/82e3ac21e))

Tightens glob pattern matching in the exec allowlist to close a bypass where overly permissive glob patterns could match unintended executables. This fix addresses a class of exec allowlist bypass vulnerabilities and includes updated changelog documentation. The companion test commit `97683071b` extends coverage to validate the tightened pattern matching behavior.

#### Exec Allowlist Glob Test Coverage ([`97683071b`](https://github.com/openclaw/openclaw/commit/97683071b))

Extends test coverage for exec allowlist glob pattern validation as a companion to the `82e3ac21e` allowlist tightening. Tests validate that the updated glob matching correctly rejects bypass patterns while permitting legitimate executables.

#### Sandboxed session_status Visibility Enforcement ([`99ec687d7`](https://github.com/openclaw/openclaw/commit/99ec687d7))

Enforces access control boundaries for the `session_status` tool in sandboxed agent contexts. The tool in `src/agents/tools/session-status-tool.ts` and `src/agents/tools/sessions-access.ts` now restricts sandboxed agents to only their own session tree keys, preventing a sandboxed agent from querying the session state of its parent or sibling sessions. Tests in `src/agents/openclaw-tools.session-status.test.ts` cover sandboxed access guards, existence oracle behavior, and legacy session_status tree key preservation.

#### Device Scope Baseline Fail-Closed ([`d8d8dc742`](https://github.com/openclaw/openclaw/commit/d8d8dc742))

Implements fail-closed behavior when a device's scope baseline is absent during authentication. Previously, a missing scope baseline could allow a device to proceed with an unvalidated scope. This change ensures device pairing authentication denies access when no scope baseline has been established, eliminating the ambiguous open state.

#### Device Token Scope Capping ([`4f462facd`](https://github.com/openclaw/openclaw/commit/4f462facd))

Limits device authentication tokens to the set of explicitly approved scopes, preventing token privilege escalation via scope manipulation. When a device requests a token, the granted scope is now capped to the intersection of the requested scope and the pre-approved baseline, ensuring tokens cannot claim scopes beyond what was authorized during device pairing.

#### GIT_EXEC_PATH Blocked in Host Env Sanitizer ([`1dcef7b64`](https://github.com/openclaw/openclaw/commit/1dcef7b64))

Adds `GIT_EXEC_PATH` to the blocked environment variable list in `src/infra/host-env-security-policy.json`, preventing a git command injection vector where an attacker could set `GIT_EXEC_PATH` to a directory containing malicious executables that git would then invoke as helpers. The Swift-generated `HostEnvSecurityPolicy.generated.swift` is also updated. This is a partial improvement to Legitimate Gap 1 (gateway-side env var blocklist) — `LD_*`/`DYLD_*` remain unfiltered at the gateway level.

#### Browser Proxy Attachment Media Size Cap Restored ([`18f15850e`](https://github.com/openclaw/openclaw/commit/18f15850e))

Restores the media size limit on proxy file attachments in the browser channel, preventing unbounded resource consumption in proxy-based file serving. The cap had been inadvertently removed in a prior refactor. Restored cap enforces a maximum attachment size, protecting against resource exhaustion via crafted attachment requests.

#### Discord Reaction Allowlist Enforcement ([`487a3ba8c`](https://github.com/openclaw/openclaw/commit/487a3ba8c))

Enforces the configured `users` and `roles` allowlist for Discord reaction ingress in `src/discord/`. Previously, the reaction monitor could ingest reactions from unauthorized users/roles before the allowlist check took effect. This fix applies allowlist validation at the ingress point, ensuring only authorized users and roles can trigger reaction-based interactions.

#### OpenAI WebSocket Replay Hardening ([`980619b9b`](https://github.com/openclaw/openclaw/commit/980619b9b))

Hardens the OpenAI WebSocket stream handler against replay attacks and message injection. Adds validation of message sequence ordering and connection state tracking, ensuring that replayed or out-of-order WebSocket frames are rejected before they can affect model inference or response attribution. Four production files and three tests were updated.

#### Script-Runner Extension + Mutable-File Approval Bypass Fix ([`b7a37c202`](https://github.com/openclaw/openclaw/commit/b7a37c202))

Two-part fix for GHSA-qc36-x95h-7j53. First, TypeScript/ESM script runners `tsx`, `jiti`, `ts-node`, `ts-node-esm`, `vite-node`, and `esno` were not recognized as interpreter-style runners in `src/node-host/invoke-system-run-plan.ts`, producing `mutableFileOperand: null` and silently skipping revalidation during execution. The fix adds these runners to the recognized set so they produce a valid `mutableFileOperand` binding. Second, `src/node-host/invoke-system-run.ts` adds a fail-closed runtime guard that denies execution when a script run should have a mutable-file binding but the approval plan is missing it, preventing unknown future runners from silently bypassing revalidation.

#### State Directory Permission Hardening ([`7e3787517`](https://github.com/openclaw/openclaw/commit/7e3787517))

Adds `tightenStateDirPermissionsIfNeeded()` to `src/config/io.ts`, called during the onboarding flow, which verifies and enforces `0o700` permissions on the OpenClaw state directory (`~/.openclaw`). If the directory is readable by other users, permissions are tightened in place. This prevents local users from reading the state directory contents, including session files and cached credentials.
