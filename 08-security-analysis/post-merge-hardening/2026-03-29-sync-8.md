#### xAI Code Execution Tool ([`396bf20cc`](https://github.com/openclaw/openclaw/commit/396bf20cc))

Adds a new `code_execution` tool backed by xAI's remote sandbox API. The tool is registered in `tool-catalog.ts` with `profiles: ["coding"]` and `includeInOpenClawGroup: true`, and added to the allowed tool list in `system-prompt.ts`. New files include `extensions/xai/src/code-execution-shared.ts` (shared types/config) and `extensions/xai/code-execution.test.ts`. The tool follows existing tool policy gating patterns — it is subject to the same tool authorization and profile restrictions as other tools like `web_search` and `browser`. A new `xai-auth-trace.ts` module extracts auth diagnostic helpers (`shouldTraceProviderAuth`, `summarizeProviderAuthKey`) from `attempt.ts` for reuse.

#### Code Execution Plugin Boundary ([`b7ab0ddb5`](https://github.com/openclaw/openclaw/commit/b7ab0ddb5))

Moves the code_execution implementation from core (`src/agents/tools/`) to the xAI plugin boundary (`extensions/xai/`). This shifts the execution boundary from the core tool registry to the extension layer. The plugin now owns `code-execution.ts` and the shared types via `code-execution-shared.ts`. The refactor also moves `x_search` tool from core to the xAI plugin, consolidating xAI-specific tooling under the extension boundary.
