#### Keep static browser helper seams cold ([`08962b681`](https://github.com/openclaw/openclaw/commit/08962b681))

Extracts browser host-inspection helpers — `parseBrowserMajorVersion`, `readBrowserVersion`, and `resolveGoogleChromeExecutableForPlatform` — into a new cold-loaded module `src/plugin-sdk/browser-host-inspection.ts`. These helpers were previously part of the browser plugin's active runtime load path, meaning any code that touched them would trigger browser plugin module loading even when the browser plugin was disabled. The new module is imported statically and never wired into the browser-runtime facade activation flow. Cleanup helpers `movePathToTrash` and `closeTrackedBrowserTabsForSessions` are similarly moved to a dedicated `src/plugin-sdk/browser-maintenance.ts` module. Thirteen production files updated with new import paths.

No audit claims directly addressed. This is architecture-level isolation hardening: disabled plugins now have a smaller eager-load footprint, reducing the blast radius of plugin-owned code paths when plugins are off.

#### Guard runtime facade activation ([`52a018680`](https://github.com/openclaw/openclaw/commit/52a018680))

Introduces two new loaders in `src/plugin-sdk/facade-runtime.ts` — `loadActivatedBundledPluginPublicSurfaceModuleSync` and `tryLoadActivatedBundledPluginPublicSurfaceModuleSync` — that check plugin enablement state via a cached config snapshot before loading plugin-owned runtime surfaces. Both loaders consult the plugin's enabled state from the config snapshot; `tryLoad...` returns `null` if the plugin is disabled, while `load...` throws. Guards are applied to runtime facades for nine plugins: discord-runtime-surface, discord-thread-bindings, image-generation-runtime, line-runtime, matrix-runtime-surface, media-understanding-runtime, memory-core-engine-runtime, slack-runtime-surface, and speech-runtime.

Session cleanup helpers that must work even when plugins are disabled are moved to `src/plugin-sdk/browser-maintenance.ts` (trash-path cleanup with retry fallback) and `src/plugin-sdk/discord-maintenance.ts` (thread unbind no-op when Discord is disabled). Comprehensive test coverage verifies guard mechanics and no-op behavior.

No audit claims directly addressed. This commit closes a plugin-isolation gap: previously, a disabled plugin's runtime surface could still be loaded if called during cleanup. The guards enforce the policy that plugin runtime code is inaccessible when the plugin is off.

#### Harden Feishu comment-thread delivery ([`ce0ff42ff`](https://github.com/openclaw/openclaw/commit/ce0ff42ff))

Adds input validation and defensive defaults to the Feishu drive tool schema in `extensions/feishu/src/drive-schema.ts`. The `page_size` parameter for `list_comments` and `list_comment_replies` operations now has a `maximum: 100` constraint (was unbounded above the existing `minimum: 1`), preventing excessively large pagination requests. The `file_type` parameter is made optional with defaults across `list_comments`, `list_comment_replies`, `add_comment`, and `reply_comment` operations, improving resilience against missing or inferred file type values. A new `FeishuReplyCommentError` class (in `extensions/feishu/src/drive.ts`) provides structured error context for comment-thread delivery failures. Ambient comment context resolution and a `queryCommentById` preflight check (also in `drive.ts`) are added for whole-comment detection.

No audit claims directly addressed. The `maximum: 100` constraint on `page_size` is a straightforward DoS-mitigation measure: without it, a caller could request 10,000+ comment pages in a single API call, potentially causing excessive data transfer or resource exhaustion.
