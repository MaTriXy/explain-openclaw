#### CLI: prune inactive gateway auth credentials on mode set ([`1878272f6`](https://github.com/openclaw/openclaw/commit/1878272f67))

Adds `pruneInactiveGatewayAuthCredentials()` to `src/cli/config-cli.ts` to automatically remove stale gateway auth credentials from the config file whenever `gateway.auth.mode` is changed. The function checks each config-set operation against `GATEWAY_AUTH_MODE_PATH` using `pathEquals()`, then inspects the current `gateway.auth` object and deletes the credential fields that are not active for the new mode: switching to `token` mode removes the `password` field; switching to `password` mode removes the `token` field; switching to `trusted-proxy` removes both. The removed paths are logged via `runtime.log(info(...))` so the user is notified which credentials were pruned. This is called from `runConfigSet()` immediately after the config patch is applied and before the file is written.

This reduces the attack surface from orphaned credentials that could persist in config after a mode switch (e.g., a token left behind when switching to password-only mode). It does not directly address any of the 16 audit claims but reinforces credential hygiene beyond what the audit assessed.
