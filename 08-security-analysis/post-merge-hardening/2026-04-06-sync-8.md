Each channel imports its per-channel audit function and calls it during security checks. The `sanitizeForPlainText` helper (`src/infra/outbound/sanitize-text.ts:22`) is re-exported from `src/plugin-sdk/outbound-runtime.ts` for channel-controlled text sanitization.

Audit 2 Claim 8 (env variable injection) remains **MITIGATED** — the gateway-side blocklist (`sanitizeHostExecEnv()` at `src/infra/host-env-security.ts:224`) is the primary enforcement point. This commit adds per-channel sanitization hooks and channel-owned audit functions as a complementary layer. No legitimate gaps opened or closed by this batch.

**`03a43fe23`** (refactor(plugins): genericize core channel seams) replaces hardcoded Discord/Matrix channel assertions in `src/security/audit.test.ts` with a dynamic plugin registry approach, improving the auditability of security logic for arbitrary channel implementations. The change reduces core-hardcoded channel coupling in security tests and enables the security audit flow to be exercised against any registered channel plugin.
