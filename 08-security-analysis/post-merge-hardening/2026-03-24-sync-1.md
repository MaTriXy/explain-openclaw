#### Plivo V2 Webhook Replay Key Normalization ([`b0ce53a79`](https://github.com/openclaw/openclaw/commit/b0ce53a79c))

The voice-call plugin's Plivo V2 webhook handler previously constructed replay keys from the full request URL, including any query parameters. Because query parameter ordering is not guaranteed across environments, replay keys could differ for semantically identical requests — potentially allowing an attacker to reuse a captured webhook payload by varying query parameter order. The fix extracts a dedicated `createPlivoV2ReplayKey()` helper that normalizes the URL via `getBaseUrlNoQuery()` before computing the HMAC-based replay key. Query parameters are stripped at the normalization step, so the replay key is now stable regardless of parameter order or encoding variation.

#### Gateway Session Reset Requires Admin Scope ([`50f6a2f13`](https://github.com/openclaw/openclaw/commit/50f6a2f136))

The `/reset` agent command previously lacked a caller-authorization check, meaning any gateway client with write scope could trigger a session reset. The fix introduces `resolveCanResetSessionFromClient()`, which gates the reset operation on `operator.admin` scope via `resolveSenderIsOwnerFromClient()`. Write-scoped gateway clients are now rejected before the reset executes, constraining this sensitive operation to admin-scoped callers only.

#### Proxy Dispatcher Enforced Before OAuth and VLM Calls ([`4e849ac12`](https://github.com/openclaw/openclaw/commit/4e849ac127))

When users configure `HTTP_PROXY` or `HTTPS_PROXY` environment variables, those settings must be registered with the global Undici HTTP dispatcher before any outbound fetch calls are made. Two OAuth flows — `loginMiniMaxPortalOAuth()` and `loginOpenAICodexOAuth()` — and one VLM API entry point — `minimaxUnderstandImage()` — were not calling `ensureGlobalUndiciEnvProxyDispatcher()` before issuing network requests. In those code paths, proxy environment variables were silently ignored, creating a proxy bypass for MiniMax and OpenAI Codex OAuth and image-understanding traffic. The fix calls `ensureGlobalUndiciEnvProxyDispatcher()` at the entry point of each affected function, ensuring proxy enforcement precedes all outbound network I/O.

#### Canvas Route Authentication Bypass Removed ([`d5dc6b657`](https://github.com/openclaw/openclaw/commit/d5dc6b6573))

The HTTP authentication middleware for the gateway's canvas API routes contained a `isLocalDirectRequest()` bypass inside `authorizeCanvasRequest()`. This loopback exemption allowed requests originating from localhost to access canvas routes without a valid bearer token — an attacker or compromised local process could exploit this to reach canvas APIs unauthenticated. The fix removes the bypass entirely (four deleted lines in `http-auth.ts`), so all canvas route access now requires a valid bearer token regardless of request origin.
