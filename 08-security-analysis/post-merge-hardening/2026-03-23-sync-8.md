#### Harden explicit-proxy SSRF pinning ([`55ad5d7bd`](https://github.com/openclaw/openclaw/commit/55ad5d7bd7))

Hardens SSRF pinning in explicit-proxy mode by moving DNS lookup attachment from `proxyTls` to `requestTls` in `createPinnedDispatcher`. Previously the `withPinnedLookup` transformation was applied to proxy-hop TLS options, which meant the pinned lookup was lost for HTTPS proxy tunnels. The fix attaches the pinned lookup via `requestTls` so that HTTPS proxy configurations correctly enforce DNS pinning on the target connection. Prevents plain HTTP bypass attempts against the SSRF guard for proxy-routed fetches. Strengthens Audit 2 Claim 4.

#### Preserve async hook ingress provenance ([`ea579ef85`](https://github.com/openclaw/openclaw/commit/ea579ef858))

Adds `HookExternalContentSource` type (`"gmail" | "webhook"`) and companion functions `resolveHookExternalContentSource` and `mapHookExternalContentSource` in `src/security/external-content.ts`. External content source metadata (webhook vs. Gmail origin) is now preserved across async hook dispatch through normalized session key routing. Before this fix, the external-content boundary was evaluated at dispatch time but not propagated forward, allowing source identity to be lost during async normalization. Maintaining immutable provenance prevents source obfuscation that could weaken prompt injection defenses.

#### Unify dispatch wrapper approval hardening ([`f52eb934d`](https://github.com/openclaw/openclaw/commit/f52eb934d6))

Consolidates exec dispatch wrapper approval hardening in `exec-wrapper-resolution.ts` and `exec-approvals-allowlist.ts`. Unifies the trust-plan evaluation so approval checks are applied consistently regardless of which dispatch wrapper variant is active. Defense-in-depth â€” does not close any of the 16 audit claims.

#### Escape invisible approval filler chars ([`4d50084c6`](https://github.com/openclaw/openclaw/commit/4d50084c6e))

Adds `sanitizeExecApprovalDisplayText` using `EXEC_APPROVAL_INVISIBLE_CHAR_REGEX` (Unicode format-control characters, Hangul filler `\u115F`/`\u1160`/`\u3164`/`\uFFA0`) in `src/infra/exec-approval-command-display.ts`. Invisible characters in command text are replaced with explicit `\u{XXXX}` escape notation before display in approval prompts. The same sanitization is applied in the macOS Swift layer via `ExecApprovalCommandDisplaySanitizer` in `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift`. Prevents invisible-character spoofing of approval dialog content.

#### Unwrap time dispatch wrappers ([`39409b6a6`](https://github.com/openclaw/openclaw/commit/39409b6a6d))

Ensures `time` command invocations are unwrapped by `unwrapTimeInvocation` before `resolveExecWrapperTrustPlan` evaluates the underlying command. Without unwrapping, a `time <cmd>` invocation could present as `time` to the approval system rather than `<cmd>`, masking the actual command from wrapper trust analysis. Defense-in-depth for exec approval correctness.

#### Restrict remote marketplace plugin sources ([`c036e4d17`](https://github.com/openclaw/openclaw/commit/c036e4d176))

Narrows the set of accepted source origins for remote marketplace plugin manifests in `src/plugins/marketplace.ts`. Limits which remote URLs may serve plugin metadata to reduce supply chain attack surface. Defense-in-depth â€” does not close any of the 16 audit claims.

#### Gate internal command persistence mutations ([`09faed6bd`](https://github.com/openclaw/openclaw/commit/09faed6bd8))

Adds gating logic to `persistInlineDirectives` in `src/auto-reply/reply/directive-handling.persist.ts` so that only properly-scoped command execution paths may trigger directive state persistence. Prevents internal command handling paths (including the new `extensions/phone-control` plugin) from writing directive mutations without going through the gated path. Defense-in-depth for gateway command authority.

#### Deny local MEDIA paths for MCP results ([`30ed4342b`](https://github.com/openclaw/openclaw/commit/30ed4342b3))

Rejects local `MEDIA://` URI paths returned by MCP tool results in `src/agents/pi-embedded-subscribe.tools.ts`. Agent-executed MCP tools can no longer reference local media store paths, blocking a potential path disclosure or sandbox-boundary violation via local media URI injection. Covers both the tool handler and the inbound subscription type guards.
