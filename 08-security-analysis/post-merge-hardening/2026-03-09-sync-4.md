#### Require talk resolved payload ([`2ed644f5d`](https://github.com/openclaw/openclaw/commit/2ed644f5d))

The gateway protocol schema for talk (voice) configuration was hardened to make the `resolved` field mandatory. Two new schemas were introduced in `src/gateway/protocol/schema/channels.ts`: `LegacyTalkConfigSchema` for backward compatibility with pre-normalization payloads, and `NormalizedTalkConfigSchema` where `resolved` is a required field (not `Optional`). The `TalkConfigResultSchema` now accepts a `Union([LegacyTalkConfigSchema, NormalizedTalkConfigSchema])`, preventing incomplete voice provider configs from passing schema validation. This closes a gap where a normalized config missing a `resolved` payload could reach downstream voice processing without proper provider resolution.

#### Persist spawned child session history ([`404b1527e`](https://github.com/openclaw/openclaw/commit/404b1527e))

ACP (agent-to-agent communication protocol) child sessions now persist session transcripts, improving the audit trail for spawned agent runs. The implementation calls `resolveSessionTranscriptFile` (from `src/config/sessions/transcript.ts`) to resolve the child session transcript path. Each ACP child run's transcript is written with spawn lineage recorded via the spawnedBy field on the session entry. The persistence is best-effort: errors during transcript-path resolution do not block ACP execution, so a disk-full or permission error cannot prevent an agent spawn from completing. This preserves runtime reliability while adding operator visibility into child agent history.

#### Thread config runtime env through models config ([`749eb4efe`](https://github.com/openclaw/openclaw/commit/749eb4efe))

Provider discovery previously required config API keys to be set in `process.env` before implicit provider detection ran, causing `src/agents/models-config.ts` to mutate the host process environment. This commit adds `createConfigRuntimeEnv()` in `src/config/env-vars.ts` (exported from that module), which builds a merged environment snapshot from the config without mutating `process.env`. Models config now passes this snapshot to provider discovery as a local variable, keeping config env vars isolated from the host process. Tests verify that `process.env.OPENROUTER_API_KEY` remains undefined after provider discovery when the key is only in config.

#### CLI: set local gateway mode in setup ([`0b452a566`](https://github.com/openclaw/openclaw/commit/0b452a566))

The `setupCommand()` function in `src/commands/setup.ts` now auto-sets `gateway.mode` to `"local"` when running setup on a fresh install or upgrading an existing config that lacks an explicit mode. The logic uses `cfg.gateway?.mode ?? "local"` so any existing non-local mode is preserved. A log message is emitted when the mode is newly set. This ensures new OpenClaw installs default to local gateway isolation without requiring manual config editing.

#### Prefer bundled channel plugins over npm duplicates ([`74624e619`](https://github.com/openclaw/openclaw/commit/74624e619))

Plugin resolution during onboarding and release-channel sync was updated to prefer bundled channel plugins over npm-installed duplicates. A new `findBundledPluginSourceInMap()` helper in `src/plugins/bundled-sources.ts` queries a pre-built `Map` without triggering a filesystem rescan per lookup. In `src/plugins/update.ts`, when a plugin has a matching bundled source, the update path now keeps the bundled local path (`source: "path"`) rather than replacing it with an npm install, with an `alreadyBundled` check to skip unnecessary writes. This prevents an attacker-controlled npm package from shadowing a bundled channel plugin during an update sync, and avoids packaging drift between the bundled and npm-distributed versions.

#### Clear stale contextTokens on model switch ([`6c9b49a10`](https://github.com/openclaw/openclaw/commit/6c9b49a10))

The session model-override logic in `src/sessions/model-overrides.ts` now deletes `entry.contextTokens` when a session's selected model changes (`selectionUpdated`) or when the runtime model is present but not aligned with the selection. Previously, a cached `contextTokens` value from a prior model would persist after a model switch, causing status reporting and token-limit enforcement to use the old model's context window size. The fix calls `delete entry.contextTokens` (marking `updated = true`) so the value is recomputed fresh from the newly active model on the next run.

#### Allow compaction model override via config ([`caf1b8482`](https://github.com/openclaw/openclaw/commit/caf1b8482))

Compaction summarization can now use a dedicated model via the `agents.defaults.compaction.model` config key. In `src/agents/pi-embedded-runner/compact.ts`, the new code reads `params.config?.agents?.defaults?.compaction?.model`, parses the `provider/model-id` format, and when the compaction provider differs from the session provider, sets `authProfileId = undefined` before calling `getApiKeyForModel`. This credential isolation prevents the primary session auth profile (e.g., an OpenAI API key) from being passed to a different provider (e.g., OpenRouter) during compaction. When no override is set, compaction continues using the session's primary model and auth profile unchanged.
