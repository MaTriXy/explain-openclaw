#### Plugins: reserve context engine ownership ([`51631e579`](https://github.com/openclaw/openclaw/commit/51631e5797d8191b1ecfc5f126f1764ac41eb74b))

This is the pre-PR-merge branch commit establishing the context engine ownership system, closely related to the PR merge commit already documented in [Mar 16 sync 1](./2026-03-16-sync-1.md). `registerContextEngine()` now returns `ContextEngineRegistrationResult`, which is either `{ ok: true }` on success or `{ ok: false; existingOwner }` when a second caller attempts to claim an already-registered engine ID. The plugin-level wrapper in `plugins/registry.ts` enforces two guards: (1) it blocks registration of the core default context engine slot by comparing `id` against `defaultSlotIdForKey("contextEngine")` and emitting a `pushDiagnostic` error if matched, and (2) it passes `{ owner: 'plugin:${record.id}' }` to tag each plugin's registration, so that conflicts report the existing owner's plugin ID. This prevents a plugin from accidentally or maliciously replacing the core context engine or claiming a slot already registered by another plugin.
