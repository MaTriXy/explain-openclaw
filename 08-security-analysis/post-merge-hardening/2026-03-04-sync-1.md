#### Pin tlon api source and secure hold music url ([`16fd60421`](https://github.com/openclaw/openclaw/commit/16fd60421))

The `@tloncorp/api` npm dependency in the tlon extension was floating on the `#main` branch reference, allowing arbitrary upstream changes to be silently included on the next install. This commit pins the reference to a specific git hash (`7eede1c1a`), closing the supply chain substitution window for that dependency. Additionally, the hold music URL in the voice-call extension was migrated from HTTP to HTTPS, removing a potential MITM downgrade vector on the audio stream URI.

#### Allow audit snapshot and summary cache reuse ([`895abc5a6`](https://github.com/openclaw/openclaw/commit/895abc5a6))

`SecurityAuditOptions` in `src/security/audit.ts` gained two optional caching parameters: `configSnapshot?: ConfigFileSnapshot | null` and `codeSafetySummaryCache?: Map<string, Promise<unknown>>`. When `configSnapshot` is provided, the audit skips `readConfigSnapshotForAudit()` and reuses the preloaded snapshot directly. When `codeSafetySummaryCache` is provided, it is reused via `??` instead of allocating a new empty map. The audit logic and security footprint are unchanged — only the I/O and CPU cost of repeated audit runs is reduced, enabling more efficient multi-level or cascading audit workflows.

#### Use PLATFORM_UNSPECIFIED for Linux in loadCodeAssist ([`008e4804a`](https://github.com/openclaw/openclaw/commit/008e4804a))

The `resolvePlatform()` function in `extensions/google-gemini-cli-auth/oauth.ts` previously fell through to `MACOS` by default and returned `LINUX` for Linux — a value rejected by Google's `loadCodeAssist` API with a 400 Bad Request. The fix explicitly checks `process.platform === 'darwin'` and maps it to `MACOS`, then returns `PLATFORM_UNSPECIFIED` for all other platforms including Linux, matching the pi-ai runtime behavior. This restores OAuth setup on Linux without changing the security posture.
