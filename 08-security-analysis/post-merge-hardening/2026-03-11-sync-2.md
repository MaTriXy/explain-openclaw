#### SecretRef: harden custom/provider secret persistence and reuse ([`fbc66324e`](https://github.com/openclaw/openclaw/commit/fbc66324e))

> **Diamond-merge note:** This commit is a variant SHA of PR #42554, which was also
> present as `36d2ae2a2` in sync 1 (Mar 11) — see
> [Mar 11 sync 1](./2026-03-11-sync-1.md) for the original documentation. The upstream
> history contains both SHAs due to the PR being merged from a feature branch (position 1
> of 31) that diverged from the upstream main. The content and security changes are
> equivalent; this entry documents the merge event.

Hardens SecretRef-based custom/provider secret persistence and reuse across multiple
layers. The new `resolveUsableCustomProviderApiKey()` function gates custom provider keys
by usable secret semantics — returning plaintext keys directly, resolving known environment
variable markers (via `isKnownEnvApiKeyMarker()`) from the process environment, and
rejecting unknown or stale markers rather than returning them as API key values.
`shouldPreserveExistingApiKey()` in `src/agents/models-config.merge.ts` now rejects
stale apiKey preservation for marker-managed providers (`isNonSecretApiKeyMarker()` check
added), preventing fallback to old unencrypted values when SecretRef markers are present.

`readJsonObjectIfExists()` in `src/secrets/storage-scan.ts` gains `ReadJsonObjectOptions`
support (`requireRegularFile` + `maxBytes`). The secrets audit in `src/secrets/audit.ts`
passes `requireRegularFile: true` and `MAX_AUDIT_MODELS_JSON_BYTES` (5 MB) when reading
models.json files, rejecting symlinks/pipes and preventing oversized-JSON DoS from
attacker-controlled agent directories. `listAgentModelsJsonPaths()` now resolves the
active agent directory via `resolveActiveAgentDir()`, which respects `OPENCLAW_AGENT_DIR`
and `PI_CODING_AGENT_DIR` env overrides, ensuring the secrets audit scans the currently
active agent's models.json path.

`projectConfigOntoRuntimeSourceSnapshot()` in `src/config/io.ts` is guarded by
`isCompatibleTopLevelRuntimeProjectionShape()`, which verifies that the runtime snapshot
and candidate config share compatible top-level key shapes before applying projection,
preventing config corruption from incompatible config objects. The model runner in
`src/agents/pi-embedded-runner/model.ts` calls `sanitizeModelHeaders()` with
`stripSecretRefMarkers: true` in `buildInlineProviderModels()` and
`applyConfiguredProviderOverrides()`, stripping SecretRef marker strings from provider
and model HTTP headers to prevent credential marker leakage in resolved models.
