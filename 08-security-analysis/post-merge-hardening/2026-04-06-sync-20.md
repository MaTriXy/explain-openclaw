#### Reuse install safety overrides ([`6b003a7f2`](https://github.com/openclaw/openclaw/commit/6b003a7f2))

`resolveInstallSafetyOverrides()` is introduced in `src/cli/plugins-install-command.ts` (line 45) to consolidate `dangerouslyForceUnsafeInstall` flag resolution into a single helper, replacing direct boolean passing to `tryInstallHookPackFromLocalPath()`. Callers at `installPluginFromNpmSpec()` and `installPluginFromPath()` now spread the resolved overrides object rather than threading the raw boolean, improving API consistency. This is a refactor — no change to the underlying install safety semantics or validation rules; the `dangerouslyForceUnsafeInstall` flag and its enforcement already existed. No audit claims affected.

#### CLI: forward unsafe flag to linked hook-pack probes ([`8a8ea9422`](https://github.com/openclaw/openclaw/commit/8a8ea9422))

`buildHookInstallForwardParams()` in `src/hooks/install.ts` (line 62) is updated to include `dangerouslyForceUnsafeInstall` in the forwarded parameter set, ensuring that when hook-pack probes run during linked install operations the caller's unsafe override flag is propagated. Previously, hook-pack probes were launched without this flag, causing them to reject unsafe-signed packages even when the caller had explicitly set `--dangerously-force-unsafe-install`. This is a parameter propagation consistency fix; no new bypass vectors are introduced and no existing validation is removed. No audit claims affected.

#### Honor unsafe override for linked installs ([`41e16a883`](https://github.com/openclaw/openclaw/commit/41e16a883))

`installPluginFromPath()` paths in `src/cli/plugins-install-command.ts` now correctly receive and forward `dangerouslyForceUnsafeInstall` through both probe and full install phases for linked (`--link`) plugin and hook-pack installations. This closes a parameter passing gap introduced by prior linked-install refactoring — linked installs previously had their unsafe override silently dropped between the CLI entry point and the install logic. The flag controls whether signature validation is bypassed for local development installs; these commits ensure the user's explicit override is honored end-to-end. No audit claims affected.
