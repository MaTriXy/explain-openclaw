#### Surface Env Override Keys in Exec Approvals ([`57204b4fa`](https://github.com/openclaw/openclaw/commit/57204b4fa))

New `buildSystemRunApprovalEnvBinding()` function (in `src/infra/system-run-approval-binding.ts`) extracts requested environment variable keys from the `env` map and attaches them to exec approval requests. Gateway approval handlers receive the env key list via the new requestedEnv field on the params type, allowing operators to see which environment overrides are requested before approving a command execution. Improves transparency of process environment modifications at approval time.

#### Share Missing-Scope Helper ([`43838b1b1`](https://github.com/openclaw/openclaw/commit/43838b1b1))

Extracts `resolveMissingRequestedScope()` to shared `src/shared/operator-scope-compat.ts`, consolidating previously duplicated scope validation logic from `src/gateway/server-methods/devices.ts` and `src/infra/device-pairing.ts`. Both callers now import from the shared module. Scope validation behavior is unchanged; boundary surface is consolidated rather than modified.

#### Share Legacy Pi Auth Token Lookup ([`520d753b2`](https://github.com/openclaw/openclaw/commit/520d753b2))

Extracts `resolveLegacyPiAgentAccessToken()` to `src/infra/provider-usage.shared.ts`. Function reads `~/.pi/agent/auth.json` and returns the access token for the first matching provider alias (e.g., `["z-ai", "zai"]`), enabling backward compatibility with Pi agent credentials. Both `src/infra/provider-usage.auth.ts` and `extensions/zai/index.ts` now import the shared helper. No new token access paths introduced.

#### Share Install Target Flow ([`03c694612`](https://github.com/openclaw/openclaw/commit/03c694612))

Refactors directory-based plugin installation target handling in `src/plugins/install.ts` by extracting helper logic for install result construction into a shared function. No new installation paths, permission surfaces, or validation steps are introduced or removed.

#### Add Image Generation Capability ([`aa2d5aaa0`](https://github.com/openclaw/openclaw/commit/aa2d5aaa0))

Agent defaults schema extended to configure image generation behavior and model selection.

#### Add /plugins Chat Command ([`cc88b4a72`](https://github.com/openclaw/openclaw/commit/cc88b4a72))

`handlePluginsCommand()` in `src/auto-reply/reply/commands-plugins.ts` routes list, show, enable, and disable operations for installed plugins. Command is gated behind the `commands.plugins` config flag (off by default) and `rejectNonOwnerCommand` prevents non-owner access, matching the authorization pattern of other privileged chat commands.

#### Harden Exec Approval Socket Auth ([`be2e6ca0f`](https://github.com/openclaw/openclaw/commit/be2e6ca0f))

New `timingSafeHexStringEquals()` function in `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift` replaces direct HMAC string comparison (`expected != request.hmac`) with a constant-time comparison. Prevents timing-oracle attacks on the macOS exec approval IPC socket where an attacker with local write access could measure response latency to infer HMAC values byte-by-byte. New test suite validates constant-time behavior for equal, length-mismatched, and value-mismatched inputs.

#### Block Canvas Symlink Escapes ([`0d776c87c`](https://github.com/openclaw/openclaw/commit/0d776c87c))

`apps/macos/Sources/OpenClaw/CanvasSchemeHandler.swift` now resolves symlinks on both the session root and the requested file URL before applying the path containment check. New `isFileURL(_:withinDirectory:)` helper performs the prefix verification against fully resolved paths. A symlink inside the canvas session root can no longer redirect file reads to paths outside the authorized directory, closing a potential path traversal escape in the canvas sandbox.
