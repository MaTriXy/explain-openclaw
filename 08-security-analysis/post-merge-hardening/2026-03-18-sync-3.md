#### Action Runtime Extension Isolation ([`9df3e9b61`](https://github.com/openclaw/openclaw/commit/9df3e9b61), [`c3386d34d`](https://github.com/openclaw/openclaw/commit/c3386d34d), [`b3ae50c71`](https://github.com/openclaw/openclaw/commit/b3ae50c71), [`8165db758`](https://github.com/openclaw/openclaw/commit/8165db758))

Four commits move channel-specific action runtimes from `src/agents/tools/` into their respective extension packages. Discord's guild, messaging, moderation, and presence runtimes move to `extensions/discord/src/actions/`; Telegram's runtime to `extensions/telegram/src/action-runtime.ts`; Slack's runtime to `extensions/slack/src/action-runtime.ts`; WhatsApp's runtime to `extensions/whatsapp/src/action-runtime.ts`. Each extension receives a new `runtime-api.ts` entry-point file. `src/plugin-sdk/agent-runtime.ts` is updated to re-export runtime types via the SDK boundary. This isolates channel-specific action execution within the owning extension, preventing cross-channel runtime leakage through shared `src/` imports.

#### Enforce Plugin Boundary Seams ([`131376782`](https://github.com/openclaw/openclaw/commit/131376782))

Large-scale refactor enforcing strict plugin SDK import seams across 8 extensions (Discord, Line, Nostr, Slack, Synology Chat, Telegram, Tlon, WhatsApp). Adds `src/plugin-sdk/discord-core.ts`, `line-core.ts`, `slack-core.ts`, `telegram-core.ts`, and `whatsapp-core.ts` as explicit re-export modules. Extensions are updated to import exclusively via these SDK entry points rather than reaching into `src/` internals. New `src/plugin-sdk/channel-import-guardrails.test.ts` enforces these boundaries via automated import checking. This prevents extensions from accidentally importing internal core modules, reducing the attack surface for supply-chain interference through extension code.

#### Sessions: Move Session Target Shaping to Plugins ([`b942dacf4`](https://github.com/openclaw/openclaw/commit/b942dacf4))

Moves session target shaping logic from `src/agents/tools/sessions-send-helpers.ts` into channel plugin type declarations (`src/channels/plugins/types.core.ts`). Individual channel extensions (Discord, Slack) now declare their own session target shape. New test coverage in `sessions-send-helpers.test.ts` validates the shaping logic per channel. This localizes session routing data transformation to the plugin that owns the channel, reducing the scope of session data exposure across the codebase.

#### CLI: Expand config set with SecretRef/Provider Builders ([`e99963100`](https://github.com/openclaw/openclaw/commit/e99963100))

Major expansion of `openclaw config set` to support `ref:<env_var>` (SecretRef) and `provider:<name>` input modes alongside the existing plain-value mode. New files: `src/cli/config-set-dryrun.ts` (defines `ConfigSetDryRunResult` with `ok`, `operations`, `checks`, `refsChecked`, `errors`), `src/cli/config-set-input.ts` (parses input modes), `src/cli/config-set-parser.ts` (mode-specific parsing), `src/secrets/target-registry-query.ts` (SecretRef resolution). A dry-run mode validates schema conformance and `resolvability` before writing config. Using `ref:` stores a reference to an environment variable rather than its resolved value, preventing plaintext secrets from appearing in config files or shell history.

#### CLI: Skip exec SecretRef Dry-Run Resolution Unless Explicitly Allowed ([`2d3bcbfe0`](https://github.com/openclaw/openclaw/commit/2d3bcbfe0))

Follow-up to `e99963100d`. Gates exec-context SecretRef resolution during dry-run behind an explicit opt-in. Adds `skippedExecRefs` counter and `resolvabilityComplete` boolean to `ConfigSetDryRunResult`. By default, dry-run no longer attempts to resolve exec-scoped SecretRefs (which could trigger side effects such as shell execution), while preserving static exec dry-run validation. This prevents accidental execution of resolver side-effects during config validation.

#### Plugin SDK: Normalize and Harden Message Action Discovery ([`a32c7e16d`](https://github.com/openclaw/openclaw/commit/a32c7e16d))

Creates `src/channels/plugins/message-action-discovery.ts` with two new functions: `resolveMessageActionDiscoveryChannelId()` normalizes raw channel identifiers via `normalizeAnyChannelId()` with a whitespace-trim fallback; `createMessageActionDiscoveryContext()` builds a typed `ChannelMessageActionDiscoveryContext` from caller parameters. Refactors `src/channels/plugins/message-actions.ts` and `src/agents/channel-tools.ts` to use this builder. Centralizing channel ID normalization prevents malformed channel identifiers from bypassing channel-specific action lookups.

#### Agents: Scope Cross-Channel Message Discovery ([`144b95ffc`](https://github.com/openclaw/openclaw/commit/144b95ffc))

Passes full agent session context (`currentChannelId`, `currentThreadTs`, `currentMessageId`, `accountId`, `sessionKey`, `sessionId`, `agentId`, `requesterSenderId`) into `listChannelSupportedActions()` when enumerating actions for non-current channels inside `resolveMessageToolSchemaActions()` and `buildMessageToolDescription()` (both in `src/agents/tools/message-tool.ts`). Previously these context fields were absent for cross-channel enumeration, making it impossible for channel plugins to scope available actions to the requesting session. This allows channel plugins to restrict cross-channel action visibility based on agent identity and session context, directly hardening the cross-channel send surface flagged in issue #20305.

### Audit Cross-Reference

None of the 10 security-relevant commits directly address the 16 Issue #1796 or Medium article audit claims (all 8+8 were already verified as non-exploitable in prior audits). Security improvements in this sync are defense-in-depth:

- **Plugin seam isolation** (`9df3e9b617`, `c3386d34d2`, `b3ae50c71c`, `8165db758b`, `1313767825`): Moves channel action runtimes into extensions and enforces SDK import boundaries. Reduces the blast radius of a compromised extension by preventing it from reaching into core `src/` internals. No specific audit claim maps to this pattern, but it strengthens the overall extension sandboxing model.
- **Session target isolation** (`b942dacf48`): Localizes session routing data to the owning plugin. Defense-in-depth for session isolation.
- **SecretRef config handling** (`e99963100d`, `2d3bcbfe08`): New `ref:<env>` input mode stores secret references (not values) in config. Dry-run resolution now skips exec-scoped refs by default. Mitigates the config secret exposure pattern from issue #13683 (already FIXED) by design — the new flow prevents secrets from ever reaching the config store.
- **Message discovery hardening** (`a32c7e16d2`, `144b95ffce`): Channel ID normalization and session-scoped action enumeration harden the message tool surface flagged in issue #20305 (cross-user sends in multi-tenant deployments). Does not fully close the gap (no per-agent recipient allowlist added), but reduces surface area.
