#### Plugins: add Claude marketplace registry installs ([`ff2e864c9`](https://github.com/openclaw/openclaw/commit/ff2e864c9))

Introduces a Claude marketplace plugin registry, adding new functions `installPluginFromMarketplace`, `listMarketplacePlugins`, and `resolveMarketplaceInstallShortcut` in `src/plugins/marketplace.ts`. Plugin install records are extended with `marketplaceName`, `marketplaceSource`, and `marketplacePlugin` metadata fields. The CLI and config schemas are updated to route marketplace installs alongside existing npm and path-based installs.

**Security note:** This commit adds a new external trust surface. Marketplace plugins are fetched from and validated against the Claude registry. Users installing marketplace plugins grant that plugin the same runtime permissions as local plugins. No sandbox isolation changes were made in this commit. This is a new attack surface that should be monitored: a compromised or malicious marketplace listing could reach users at scale.

#### Shrink public channel plugin SDK surfaces ([`7e74adef9`](https://github.com/openclaw/openclaw/commit/7e74adef9))

Removes channel-specific exports from the top-level `plugin-sdk/index.ts`, routing them instead through dedicated per-channel seam files (`discord.ts`, `slack.ts`, `telegram.ts`, etc.). Critically, `src/security/audit-channel.ts` is updated to import from the plugin SDK seam instead of the extension path directly.

**Security note:** Reducing the public plugin SDK surface limits the blast radius of a misconfigured or malicious plugin — plugins can no longer access all channel APIs through a single import. The `audit-channel.ts` import path update is structural only; the security audit logic itself is unchanged.

#### Finish provider auth boundary cleanup ([`7cc578920`](https://github.com/openclaw/openclaw/commit/7cc578920))

Completes provider auth boundary refactoring across 26 production files. Auth-related imports that previously crossed directly between extensions are routed through a centralized provider-auth boundary. Updates `src/plugins/config-state.ts` and related auth choice logic. Removes fragile cross-extension auth dependencies that could create unexpected coupling between provider auth flows.

**Security note:** Structural improvement only — no auth logic changed. Centralizing auth boundaries reduces the likelihood of future auth-related import errors bypassing the provider auth contract.

**Security note:** Test-only addition. No production auth logic changed. The contract suite establishes a regression baseline for the provider auth surface as the plugin architecture evolves.

#### Unblock CI gates — audit.ts lazy-load refactor ([`cec10703d`](https://github.com/openclaw/openclaw/commit/cec10703d))

Refactors `src/security/audit.ts` to lazy-load gateway probe dependencies via dynamic imports, avoiding circular dependency issues at module load time. Also updates memory limits for statusJson (900 MB → 925 MB) and normalizes path assertions in `src/plugins/discovery.test.ts` for cross-platform compatibility.

**Security note:** The audit module refactor is structural — no security audit logic, check ordering, or gate criteria changed. Lazy-loading avoids a circular import that was causing CI gate failures; the runtime behavior of the security audit remains identical.

#### Remove Chrome extension relay and add MCP doctor migration ([`476d94873`](https://github.com/openclaw/openclaw/commit/476d94873))

Breaking refactor that removes the Chrome extension relay infrastructure (1,068 lines deleted), including `src/browser/extension-relay.ts`, `src/browser/extension-relay-auth.ts`, and their test files. Replaces the extension relay path with MCP-based model context protocol support. Adds `src/commands/doctor-legacy-config.ts` to guide users migrating from the old extension config. Updates 80+ files across browser config schemas, doctor commands, and related tests.

**Security note:** This is a significant attack surface reduction. The extension relay was a browser-facing communication channel between the OpenClaw gateway and a Chrome extension. Its removal eliminates the entire relay code path. All seven documentation references to `extension-relay.ts` that existed before this commit are now stale (file deleted). The MCP-based replacement provides equivalent browser model context access through a different architectural path. No new permissions or trust escalation are introduced by this refactor.
