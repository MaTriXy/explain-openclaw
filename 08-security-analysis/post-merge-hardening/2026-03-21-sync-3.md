#### Bootstrap Token Lifecycle Hardening ([`2fd372836`](https://github.com/openclaw/openclaw/commit/2fd372836e))

Improves QR-based device pairing security via explicit token revocation and a bug fix in single-use enforcement. Two new functions are added to `src/infra/device-bootstrap.ts`: `revokeDeviceBootstrapToken()` revokes a specific bootstrap token by looking it up via constant-time comparison with `verifyPairingToken()`, and `clearDeviceBootstrapTokens()` bulk-clears all stored tokens within a write lock.

A correctness fix was applied to `verifyDeviceBootstrapToken()`: the previous implementation used `delete state[entry.token]` where `entry` was the value object, so `entry.token` was the token hash — not the storage key — which could silently fail to consume the record. The fixed code uses `Object.entries()` to capture the storage key as `tokenKey`, then calls `delete state[tokenKey]`. The iOS `device-pair` plugin now calls `revokeDeviceBootstrapToken()` after successful bootstrap, and adds token expiry display (`expiresAtMs`) to QR presentation so users see when the pairing window closes.

#### Transport Error Sanitization ([`d78e13f54`](https://github.com/openclaw/openclaw/commit/d78e13f545))

Introduces `formatTransportErrorCopy()` in `src/agents/pi-embedded-helpers/errors.ts` to sanitize raw network error strings before surfacing them to end users. The function maps POSIX error codes (ECONNREFUSED, ECONNRESET, ECONNABORTED, ENETRESET, EPIPE, ENOTFOUND, EAI_AGAIN, ENETUNREACH, EHOSTUNREACH, EHOSTDOWN) and transport-level phrases ("socket hang up", "fetch failed", "connection error") to generic messages. This prevents internal LLM provider host addresses and port details from leaking through error messages.

#### Telegram Custom API Endpoint with SSRF Policy Extension ([`6b4c24c2e`](https://github.com/openclaw/openclaw/commit/6b4c24c2e5))

Adds `apiRoot` config support for Telegram, allowing users to point the bot at a self-hosted Telegram Bot API server. The new `resolveTelegramApiBase(apiRoot?)` function in `extensions/telegram/src/fetch.ts` trims the configured value and strips trailing slashes, falling back to the default `api.telegram.org` endpoint when empty. All Telegram API call sites (send, probe, audit, media download, username lookup) are threaded with the resolved base URL.

The previously static `TELEGRAM_MEDIA_SSRF_POLICY` object is replaced by `buildTelegramMediaSsrfPolicy(apiRoot?)` in `extensions/telegram/src/bot/delivery.resolve-media.ts`. The function extracts the custom hostname via `new URL(apiRoot).hostname` and appends it to the allowlist, with a `try/catch` that silently ignores invalid URLs and falls back to `api.telegram.org` only. The `apiRoot` config field is validated with a Zod `.url()` schema in `src/config/zod-schema.providers-core.ts`.
