#### Expand SecretRef coverage across user-supplied credentials ([`806803b7e`](https://github.com/openclaw/openclaw/commit/806803b7e))

Expands the SecretRef mechanism to cover user-supplied credentials across all messaging channels and extensions. Updates `SecretInputSchema` in the zod schema and credential resolution paths, touching 236 files spanning core gateway, CLI tools (`secrets-cli`, `qr-cli`, `memory-cli`), agent runtime env-overrides, and all messaging extensions (bluebubbles, feishu, googlechat, irc, matrix, mattermost, msteams, nextcloud-talk, zalo). Improves normalization of sensitive credentials to reduce plaintext credential handling in the credential pipeline.

**Audit Impact:** Partially addresses Audit 1 Claims 1 and 3 (plaintext OAuth token storage; hardcoded OAuth client secret) by routing user-supplied credentials through the SecretRef normalization pipeline rather than passing them as raw strings. Partially addresses Audit 2 Claim 1 (config injection via `setupCommand`) by tightening `SecretInputSchema` validation in the security schema layer. No legitimate gaps are closed by this commit — the three open gaps (pipe-delimited token format, `outPath` validation, bootstrap/memory file content scanning) are unrelated to credential normalization.
