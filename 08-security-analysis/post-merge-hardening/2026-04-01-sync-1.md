#### Guard executor-only producer writes ([`8fb247c52`](https://github.com/openclaw/openclaw/commit/8fb247c52))

This refactor introduces `createRunningTaskRun()` at `src/tasks/task-executor.ts:42` as the sole authorized path for transitioning a task into a running state. Previously, producer code elsewhere in the system could directly mutate task run state, bypassing any lifecycle enforcement. By centralizing this write behind a facade, the executor boundary ensures only the task executor layer can create running task records.

A dedicated boundary test was added at `src/tasks/task-executor-boundary.test.ts` to explicitly verify the architectural enforcement — callers outside the executor path cannot construct running task state directly.

**Audit cross-reference:** Defense-in-depth RBAC improvement. Not tied to any specific claim in Issue #1796 or the Medium article, but aligns with the RBAC principle relevant to privilege escalation concerns in those audits. The `createRunningTaskRun()` facade reduces the surface for unauthorized task state mutation from any executor-adjacent code path.

**Escalation resolved — non-security.** This 28-file commit adds `channels.matrix.proxy` config support for routing Matrix connections through an HTTP/SOCKS proxy. Escalated due to size; reviewed and cleared.
