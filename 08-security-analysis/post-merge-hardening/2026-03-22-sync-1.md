#### Webchat Image Persistence ([`c9449d77b`](https://github.com/openclaw/openclaw/commit/c9449d77b4))

The new `persistChatSendImages()` function in `src/gateway/server-methods/chat.ts` converts base64-encoded image payloads from webchat inbound messages into buffers and passes them to `saveMediaBuffer()` for disk storage under `~/.openclaw/media/inbound/`. Writes are non-blocking (fire-and-forget) and guarded against ACP bridge clients to avoid persisting ACP protocol frames. Error paths use `formatForLog()` to prevent raw stack traces from leaking in gateway logs. This adds a new write surface separate from the screen_record outPath gap (Legitimate Gap 3) — image content is determined by webchat senders and written to a fixed subdirectory, not a caller-controlled path.

#### Sandbox Module Boundary Refinement ([`d88c68fec`](https://github.com/openclaw/openclaw/commit/d88c68fec1))

Exports `formatSandboxToolPolicyBlockedMessage` directly from `src/agents/sandbox/runtime-status.ts` instead of the broad `src/agents/sandbox.ts` barrel, reducing circular dependency chains between sandbox components. Introduces lazy-loading for media and link understanding in `src/auto-reply/reply/get-reply.ts`: `applyMediaUnderstanding` and `applyLinkUnderstanding` are now loaded via conditional `await import()` calls gated by `hasInboundMedia()` and `hasLinkCandidate()` — limiting module load to replies that actually need those capabilities. No audit claims are affected.

#### Tool Call ID Security Fix ([`265386cd6`](https://github.com/openclaw/openclaw/commit/265386cd6b))

While primarily registering Claude bundle commands natively in the plugin system, this commit replaces `Math.random()` with `generateSecureToken(8)` for inline tool call ID generation in `src/auto-reply/reply/get-reply-inline-actions.ts` (line 244). The previous `Math.random()`-based approach produced weak numeric IDs that could enable replay or correlation attacks if tool call IDs were exposed to untrusted parties. `generateSecureToken(8)` uses the Node.js crypto module for 8 bytes of cryptographically secure random data. This closes local tracking of open issue #10646 (Weak UUID for tool call IDs).
