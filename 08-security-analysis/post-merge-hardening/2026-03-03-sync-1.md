#### Cross-Realm Promise Fix in Boundary Traversal ([`cb9bce902`](https://github.com/openclaw/openclaw/commit/cb9bce902))

Fixes a latent edge case in async path boundary traversal when called from sandboxed or VM execution contexts. The prior code used `instanceof Promise` to detect async results from `readLexicalStat()` and `resolveAndApplySymlinkHop()`. In cross-realm contexts (VM modules, sandboxes), `instanceof Promise` returns `false` even for genuine promise objects because the `Promise` constructor differs across realms.

New `isPromiseLike<T>()` helper at `src/infra/boundary-path.ts:131` checks for a `then` method instead of checking the constructor chain, correctly identifying cross-realm promises. All three call sites in the boundary traversal logic now use `isPromiseLike` and wrap results with `Promise.resolve()` to normalize cross-realm thenables.

Without this fix, a sandboxed session's async symlink resolution result could be treated as a synchronous null stat, potentially causing incorrect boundary decisions during path traversal.
