#### Harden async approval followup delivery in webchat-only sessions ([`5e4a64848`](https://github.com/openclaw/openclaw/commit/5e4a64848))

Introduces defensive logic to prevent execution approval completion messages from silently failing in webchat-only deployments (no external channel configured). The commit adds `resolveExternalBestEffortDeliveryTarget()` (`src/infra/outbound/best-effort-delivery.ts:15`) to select the appropriate delivery channel, and `shouldDowngradeDeliveryToSessionOnly()` (`src/infra/outbound/best-effort-delivery.ts:43`) to detect when external delivery cannot succeed and fall back to session-only delivery. Without this fix, approval completions could be silently dropped in webchat-only sessions, leaving the user with no confirmation that their approval was processed.

#### Add orphaned session key migration ([`b3c69b941`](https://github.com/openclaw/openclaw/commit/b3c69b941))

Fixes accumulated orphaned session keys caused by a write-path bug (#29683) by running `runStartupSessionMigration()` (`src/gateway/server-startup-session-migration.ts:17`) at gateway startup. The migration calls `migrateOrphanedSessionKeys()` (`src/infra/state-migrations.ts:1046`) to canonicalize stale session key entries and log any changes or warnings. The migration is idempotent and best-effort: if it fails, gateway startup continues. This cleans up stale auth state that accumulated since the bug was introduced rather than requiring manual intervention via `openclaw doctor`.

#### Remove xAI auth trace logging ([`8bf86b4cd`](https://github.com/openclaw/openclaw/commit/8bf86b4cd))

Removes the xAI-specific auth tracing module (`src/agents/xai-auth-trace.ts`, now deleted) and its callers across five files. The removed code included `shouldTraceProviderAuth()` and `summarizeProviderAuthKey()`, which created partial API key previews (first 4 + last 4 characters) for debug logging when the provider was xAI. Eliminating this code removes a surface where partial API key material could appear in trace logs, reducing inadvertent credential exposure from debug output.

#### Use source snapshots for config mutations ([`47216702f`](https://github.com/openclaw/openclaw/commit/47216702f))

Migrates all config mutation paths from the previous direct-write pattern to `replaceConfigFile()` (compare-and-swap via `baseHash`), including gateway startup auth persistence in `src/gateway/startup-auth.ts`. The `replaceConfigFile` function reads a hash of the config file on disk before writing and aborts if the file has changed since the snapshot was taken, preventing TOCTOU races where two concurrent config mutations could silently overwrite each other. The `ensureGatewayStartupAuth()` signature (`src/gateway/startup-auth.ts:218`) now accepts a `baseHash` parameter so callers can supply a snapshot hash from their earlier read.
