### Audit Cross-Reference

**Audit 1 Claim 4 (Token Refresh Race Condition) — ADDRESSED**

Four commits complete the remediation of the shared gateway auth session management race condition identified in Audit 1 Claim 4. When gateway auth (shared token or password) changes, previously authenticated WebSocket sessions remained connected until their natural expiry — allowing a revoked credential window.

**`54b269b2c`** introduces the core mechanism: `didSharedGatewayAuthChange()` (`src/gateway/server-methods/config.ts:249`) detects when the effective shared auth has changed; `disconnectClientsUsingSharedGatewayAuth()` (`src/gateway/server.impl.ts:1280`) iterates all connected clients and closes those with `usesSharedGatewayAuth: true` (code 4001). New `usesSharedGatewayAuth` field (set at `src/gateway/server/ws-connection/message-handler.ts:1106`) distinguishes token/password-authenticated clients from device-token clients.

**`97558f232`** extends rotation coverage to `config.patch` and `config.apply` handlers (previously only `config.set`). Tests explicitly verify that device-token-authenticated sessions are NOT disconnected on shared token change.

**`c742963fd`** prevents false-positive disconnects when SecretRef-based tokens are reapplied unchanged: `didSharedGatewayAuthChange()` now uses `isDeepStrictEqual()` comparison (`src/gateway/server-methods/config.ts:249`) instead of shallow `!==`, preventing unnecessary invalidations.

**`7fd9e4096`** refactors `didSharedGatewayAuthChange()` to use `getEffectiveSharedGatewayAuth()` (`src/gateway/server-methods/config.ts:225`) which resolves env vars and SecretRef tokens via `resolveGatewayAuth()` before comparison, ensuring semantic equality rather than reference equality is checked.

**Key verified locations:**
- `getEffectiveSharedGatewayAuth()`: `src/gateway/server-methods/config.ts:225`
- `didSharedGatewayAuthChange()`: `src/gateway/server-methods/config.ts:249`
- `disconnectClientsUsingSharedGatewayAuth()`: `src/gateway/server.impl.ts:1280`
- `usesSharedGatewayAuth` field: `src/gateway/server/ws-types.ts:8`, set at `src/gateway/server/ws-connection/message-handler.ts:1106`

**Other security commits:**

**`e0580e686`** hardens Vitest mock isolation across 23 test files by wrapping `vi.mock()` calls with async `importOriginal` callbacks and spread-merging actual module exports. This prevents cross-test mock pollution where incomplete mocks could mask security-relevant function calls (e.g., `spawnSync` alongside mocked `spawn`).

**`745f1c981`** removes deprecated legacy config aliases (`streamMode → streaming`, `allowPrivateNetwork → dangerouslyAllowPrivateNetwork`, root-level `talk.*` fields) and applies type casts to legacy sandbox access patterns (`legacyAgentSandbox`, `legacyDefaultSandbox`).
