#### Auto-enable channel status state ([`0bcf07690`](https://github.com/openclaw/openclaw/commit/0bcf07690))

Fixes regression in channel status state by applying plugin auto-enable configuration snapshot for channel account state resolution. Changes channels.status handler to use `applyPluginAutoEnable()` instead of raw `loadConfig()` for correct auto-enabled channel configuration. Relates to Audit 2 Claim 5: ensures configuration snapshot is applied consistently, preventing stale config from bypassing auto-enable state.

#### Auto-enable gateway send selection ([`ff348d206`](https://github.com/openclaw/openclaw/commit/ff348d206))

Fixes gateway send selection regression by applying plugin auto-enable configuration snapshot before resolving message channel selection. Updates gateway send handler to use `applyPluginAutoEnable()` instead of raw `loadConfig()` to respect auto-enabled channel configurations. Same Audit 2 Claim 5 pattern — prevents incorrect channel routing from stale config.

#### MSTeams blockStreaming progressive delivery ([`6b0e74000`](https://github.com/openclaw/openclaw/commit/6b0e74000))

Adds `blockStreaming` configuration option to MSTeams channel with progressive delivery support. Adds boolean config validation schema and enables immediate message flushing when `blockStreaming` is enabled. MSTeams was the last channel without this option. No direct audit claim, but improves messaging delivery reliability.

#### Matrix default-account password activity ([`e2a249228`](https://github.com/openclaw/openclaw/commit/e2a249228))

Fixes secret collection logic for Matrix default-account password handling in `src/secrets/runtime-config-collectors-channels.ts`. Adds `defaultAccountAccessTokenConfigured` check to properly gate password secret collection when access-token auth is already configured for the default Matrix account. Uses `normalizeAccountId()` with `DEFAULT_ACCOUNT_ID` constant for consistent account ID comparison. The fix prevents the secrets runtime from incorrectly surfacing password activity when an access token is already the configured auth method.

#### XAI runtime refactor into plugin ([`c4e6fdf94`](https://github.com/openclaw/openclaw/commit/c4e6fdf94))

Major 25-file refactor moving XAI runtime from core into plugin. Security-relevant credential resolution changes:

1. `src/agents/tools/web-search-provider-credentials.ts` — switches from `normalizeResolvedSecretInputString()` to `normalizeSecretInputString()` + `resolveSecretInputRef()`, adding explicit env-ref credential resolution path
2. `src/web-search/runtime.ts` — `hasEntryCredential()` changes to resolve secret refs with env source checking, now returns `true` for non-env configured refs and checks `process.env[configuredRef.id]` for env-source refs
3. Deletes `extensions/xai/src/grok-web-search-provider.ts` (XAI search provider moved into plugin)
4. Adds `src/plugins/provider-model-compat.ts` for provider normalization

The credential resolution changes alter how API keys are validated for presence — no auth bypass, but the resolution path now explicitly handles `resolveSecretInputRef()` with env source checking instead of the previous consolidated `normalizeResolvedSecretInputString()` approach.
