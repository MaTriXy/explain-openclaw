#### `8ae8a5c174` — Empty-string guard in raw config redaction

`replaceSensitiveValuesInRaw()` (`src/config/redact-snapshot.raw.ts:4`) now filters empty strings from `sensitiveValues` before replacement. Previously, an empty string in the sensitive-values array caused `replaceAll("", sentinel)` to insert the redaction sentinel at every character boundary, corrupting the entire raw config snapshot output. The fix adds both deduplication via `new Set()` and a `.filter((value) => value !== "")` guard before sorting. This is a correctness fix for the secret redaction pipeline — ensuring that a degenerate empty-string secret value cannot corrupt snapshot exports. No audit claims affected; new robustness guard.

#### `21e53aea9e` — Gateway WebSocket auth refresh after secrets reload

`attachGatewayWsMessageHandler()` (`src/gateway/server/ws-connection/message-handler.ts:150`) now accepts `getResolvedAuth: () => ResolvedGatewayAuth` (a getter function) instead of a pre-resolved `resolvedAuth: ResolvedGatewayAuth` value. The getter is called lazily at each message-handling invocation (`message-handler.ts:436: const resolvedAuth = getResolvedAuth()`), and the closure is created in `src/gateway/server.impl.ts:637`. This ensures that after a gateway secrets reload, incoming WebSocket messages re-resolve auth against the current credential state rather than using the auth snapshot from connection establishment time. Previously, a credential rotation would leave existing WebSocket connections using stale auth until they disconnected and reconnected. No audit claims affected; new defense against stale credentials persisting across hot-reloads in long-lived connections.
