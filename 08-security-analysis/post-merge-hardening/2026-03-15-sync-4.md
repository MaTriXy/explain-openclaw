#### Make health monitor stale threshold and max restarts configurable ([`ba6064cc2`](https://github.com/openclaw/openclaw/commit/ba6064cc2))

The gateway health monitor previously used hardcoded values for the stale connection threshold and maximum consecutive restart count. This commit exposes both limits as configurable options, enabling operators to tune health monitor aggressiveness per deployment. Tighter limits reduce the blast radius if the health monitor enters a restart loop; wider limits can be set for high-availability deployments that tolerate transient stalls. The change also adds test coverage for the new parameters and updates documentation.

#### Heartbeat: add isolatedSession option for fresh session per heartbeat run ([`2806f2b87`](https://github.com/openclaw/openclaw/commit/2806f2b87))

Heartbeat agent runs can now optionally create a fresh isolated session for each execution via a new `isolatedSession` option. Without isolation, sequential heartbeat runs share session context, creating a potential state-leak vector where artifacts from one run could influence subsequent checks. The new flag makes session isolation explicit and opt-in. Four documentation files were updated to describe the new heartbeat behavior.

#### Security: add secops ownership for sensitive paths ([`432ea1124`](https://github.com/openclaw/openclaw/commit/432ea1124))

Adds CODEOWNERS-style ownership assignments for security-sensitive file paths, routing pull request review requests to the secops team. This is a process control rather than a runtime code fix: it ensures security-relevant changes receive mandatory security review before merge, reducing the risk of inadvertent changes bypassing security scrutiny.

#### Remove debug console.log on node host startup ([`747609d7d`](https://github.com/openclaw/openclaw/commit/747609d7d))

Removes a debug-level `console.log` call present at node host startup. Debug log statements can leak internal state, configuration details, or environment values to stdout/stderr, and may be captured by logging systems accessible to unauthorized parties. Removing the call prevents inadvertent information disclosure during startup.

#### Remove re-introduced auth.mode=none pairing bypass ([`92fc8065e`](https://github.com/openclaw/openclaw/commit/92fc8065e))

This commit re-fixes the `auth.mode=none` pairing bypass in `attachGatewayWsMessageHandler`. The `skipPairing` condition had previously included a check for `resolvedAuth.mode === "none"`, which was originally patched out by PR #43478. Commit `3704293e6f` in this same batch silently re-introduced the bypass by merging from a branch that predated the original fix. This is the third time this specific bypass has appeared and been removed. The fix removes the `auth.mode=none` exemption so that unauthenticated-mode connections are never granted unverified pairing access.

#### Stop inheriting dm allowlist for groups ([`774b40467`](https://github.com/openclaw/openclaw/commit/774b40467))

Fixes a Zalouser authorization bug where the DM (direct message) allowlist was being applied to group message authorization. Group membership and DM allowlist are distinct access controls; using the DM allowlist for groups caused the group allowlist to be effectively bypassed for senders already on the DM allowlist. This maps to GHSA-f5mf-3r52-r83w ("Zalouser allowlist authorization matched mutable group names by default"), where mutable group names could be exploited to satisfy the allowlist check. The fix ensures DM and group authorization paths are fully separated.

#### Harden health monitor account gating ([`29fec8bb9`](https://github.com/openclaw/openclaw/commit/29fec8bb9))

Tightens the account-scoping logic in the gateway health monitor so that channel restart operations are strictly constrained to the originating account ID. Previously, the health monitor could trigger channel restarts that were not fully scoped by account, creating a risk of cross-account interference in multi-tenant deployments. The fix adds explicit account ID gating before any channel restart is dispatched.
