#### Kill stuck ACP child processes on startup and harden sessions in discord threads ([`61f7cea48`](https://github.com/openclaw/openclaw/commit/61f7cea48))

`waitForExit()` and `spawnAndCollect()` in `extensions/acpx/src/runtime-internals/process.ts` gain cooperative abort via `AbortSignal` with SIGTERM → 250ms grace → SIGKILL lifecycle, handling already-exited processes without error.

`reconcileAcpThreadBindingsOnStartup()` in `src/discord/monitor/thread-bindings.lifecycle.ts` is now async and uses `AcpThreadBindingHealthProbe` with `ACP_STARTUP_HEALTH_PROBE_CONCURRENCY_LIMIT = 8` to probe active ACP thread bindings in parallel at startup, cleaning up stale bindings.

New file `src/gateway/server-methods/agent-wait-dedupe.ts` provides `addWaiter()` / `notifyWaiters()` for event-driven deduplication of `agent.wait` / `chat.send` runs, plus `readTerminalSnapshotFromGatewayDedupe()` and `waitForTerminalGatewayDedupe()` for shared runId tracking. `setGatewayDedupeEntry()` replaces direct `context.dedupe.set()` calls in `chat.ts`.

**Audit 2 Claim 5 reinforced**: The event-driven deduplication layer prevents racing agent sessions from replaying in-flight responses, reducing the self-approval surface.

#### Route system events to bound agent sessions ([`c1bb07bd1`](https://github.com/openclaw/openclaw/commit/c1bb07bd1))

`resolveSlackSystemEventSessionKey()` in `src/slack/monitor/context.ts` provides binding-aware session key resolution for Slack system events (reactions, modal submissions, block action interactions). Interaction events now carry `senderId` through to routing, enabling correct DM and group channel routing.

Without this fix, Slack system events (reactions, modal responses) could be dispatched to an unintended agent session in multi-agent Slack workspaces, potentially mixing context between sessions.

#### Relay ACP sessions_spawn parent streaming (#34310) (thanks @vincentkoc) ([`257e2f533`](https://github.com/openclaw/openclaw/commit/257e2f533))

New file `src/agents/acp-spawn-parent-stream.ts` introduces `startAcpSpawnParentStreamRelay()` to relay streaming output from spawned child ACP processes back to the parent session. Built-in safety bounds: `DEFAULT_MAX_RELAY_LIFETIME_MS` (6 hours), `STREAM_BUFFER_MAX_CHARS = 4_000` (buffer overflow cap), `DEFAULT_STREAM_FLUSH_MS = 2_500` ms flush interval.

`resolveAcpStreamLogPathFromSessionFile()` resolves the child's log path via `readAcpSessionEntry` with error fallback, preventing panics when the session file is missing or malformed.

#### Preserve raw media invoke for HTTP tool clients ([`7b5e64ef2`](https://github.com/openclaw/openclaw/commit/7b5e64ef2))

`handleToolsInvokeHttpRequest()` in `src/gateway/tools-invoke-http.ts` now passes `allowMediaInvokeCommands: true` when building the tool list for HTTP callers. This preserves raw media invoke payloads (screen_record, camera) instead of stripping them before delivery.

**Gap 3 status unchanged**: This commit does not add `outPath` path validation in `src/agents/tools/nodes-tool.ts`. The `screen_record` tool still accepts arbitrary paths without validation; writes remain confined to the paired node device, so exploitation requires prior device-level access.
