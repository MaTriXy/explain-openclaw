## Security Details

### Third-Party Dependency Security Patches ([`f7f75519a`](https://github.com/openclaw/openclaw/commit/f7f75519a))

Two security patches applied via package dependency updates:

**file-type 21.3.0 → 21.3.1** — Fixes an infinite-loop denial-of-service in the ASF (Advanced Systems Format) media parser. Malformed ASF input could cause the parser to loop indefinitely without making progress, enabling a local DoS against any feature that calls `fileTypeFromBuffer()`. The fix adds a loop guard to the ASF detection path. OpenClaw uses `file-type` for media-type detection in attachment and image handling paths.

**hono 4.12.5 → 4.12.7** — Hardens prototype pollution defense in `parseBody({ dot: true })` mode. The previous implementation allowed `__proto__` as a valid path segment when parsing dot-notation form bodies (e.g., `__proto__.polluted=1`), enabling prototype pollution in request handlers that use `ctx.req.parseBody({ dot: true })`. The fix filters `__proto__` (and related `constructor`/`prototype`) path segments before property assignment. The OpenClaw gateway uses Hono as its HTTP framework.

Neither fix directly addresses the 16 named audit claims (Issue #1796, Medium article). Both are defense-in-depth patches to third-party libraries used in the OpenClaw dependency tree.
