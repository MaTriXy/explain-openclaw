#### Block silent reconnect scope-upgrade escalation ([`81ebc7e03`](https://github.com/openclaw/openclaw/commit/81ebc7e034))

Adds `pairingStateAllowsRequestedAccess()` and `resolveLivePendingRequestId()` to the gateway WS connection message handler. When a device with lower-scope pairing (e.g., read-only) attempts a silent reconnect requesting higher scopes (e.g., admin), the gateway now explicitly checks that the current pairing state authorizes the requested scope level. If the pairing does not cover the requested scopes, the silent reconnect is rejected — the client must go through the full pairing/approval flow to acquire elevated scopes. Previously, a race window existed where a read-scoped paired device could silently widen its access on reconnect.

#### Enforce localRoots sandbox on Feishu docx upload file reads ([`764394c78`](https://github.com/openclaw/openclaw/commit/764394c78b))

In the Feishu extension's `resolveUploadInput()`, local file reads for docx upload attachments previously used `fs.readFile()` directly, bypassing localRoots sandbox enforcement. The fix replaces the raw `fs.readFile()` call with `getFeishuRuntime().media.loadWebMedia(resolvedPath, { maxBytes, optimizeImages: false })`, which applies the same localRoots sandbox used by `sendMediaFeishu`. This ensures Feishu docx upload file reads are confined to permitted media roots (tmp, media, workspace, sandboxes) rather than being able to access arbitrary filesystem paths.

#### Filter untrusted CWD .env entries before OpenClaw startup ([`6a7932480`](https://github.com/openclaw/openclaw/commit/6a79324802))

Previously, `loadDotEnv()` in `src/infra/dotenv.ts` loaded the CWD `.env` file with minimal filtering, allowing a maliciously placed `.env` to override `OPENCLAW_CONFIG_PATH` (loading a controlled config file, enabling jiti-based plugin code execution) or inject LD_PRELOAD/DYLD_INSERT_LIBRARIES for library preloading. This commit introduces `loadWorkspaceDotEnvFile()` and `shouldBlockWorkspaceDotEnvKey()`. The block function rejects keys in `BLOCKED_WORKSPACE_DOTENV_KEYS` (which includes `OPENCLAW_CONFIG_PATH`, `OPENCLAW_HOME`, `OPENCLAW_STATE_DIR`, `OPENCLAW_OAUTH_DIR`) plus any key matching the runtime env-var security guard (blocking LD_*, DYLD_*, NODE_OPTIONS, PATH, etc.) and keys ending in `_BASE_URL`. The CWD `.env` load path now calls `loadWorkspaceDotEnvFile(cwdEnvPath)` instead of raw dotenv.

This **fixes Audit 2 Claim #8** (env variable injection via `LD_PRELOAD`/`LD_LIBRARY_PATH` through CWD `.env` files) and closes the exploit chain described in tracked issue #11437 (CWD `.env` → `OPENCLAW_CONFIG_PATH` override → jiti plugin code execution). The upstream issue remains `NOT_PLANNED` but the local codebase is now protected.

#### Remove tool policy facade — harden sandbox blocked-tool guidance ([`501190d2e`](https://github.com/openclaw/openclaw/commit/501190d2e8))

Consolidates sandbox tool policy resolution into `src/agents/sandbox/tool-policy.ts`, removing a facade indirection layer. The primary security hardening is in `formatSandboxToolPolicyBlockedMessage()` in `src/agents/sandbox/runtime-status.ts`:

- `sanitizeForSingleLineDisplay()` (added at `:86`) escapes control characters (`\n`, `\r`, `\t`, and any codepoint below `0x20` or equal to `0x7f`) in tool names before embedding them in blocked-tool guidance messages. Prevents control character injection via maliciously crafted tool names in error output.
- `redactSessionKey()` (added at `:112`) redacts session keys in error output: short keys (≤12 chars) become `"(redacted)"`, longer keys show only the first 6 and last 6 chars through `sanitizeForSingleLineDisplay`.
- `shellEscapeSingleArg()` (added at `:123`) single-quote-escapes the session key before embedding it in the `openclaw sandbox explain --session <key>` command hint.
- `hasUnsafeControlChars()` causes the fallback to `openclaw sandbox explain --agent <agentId>` when the session key contains control characters, avoiding embedding an unsafe key in a shell command hint entirely.
