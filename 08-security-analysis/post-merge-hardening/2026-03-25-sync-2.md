#### Unify external CLI credential sync ([`8e9e2d2f4`](https://github.com/openclaw/openclaw/commit/8e9e2d2f4))

Centralizes shared OAuth comparison helpers as exported functions in `external-cli-sync.ts`: `shouldReplaceStoredOAuthCredential()` (decides whether to overwrite an existing stored credential) and `areOAuthCredentialsEquivalent()` (deep field comparison). Consolidates CLI credential caching under a generic `readCachedCliCredential<T>()` function in `cli-credentials.ts` that accepts an optional `readSourceFingerprint` callback. When provided, the fingerprint (typically a file mtime from `readFileMtimeMs()`) is compared before and after the read — if it changed during the read, the cache write is suppressed (TOCTOU defense against credential file races). Moves a hardcoded profile ID constant to `constants.ts`. This is a supporting refactor; the security-relevant freshness guard built on top of these helpers lands in the following commit.

#### Add autoThreadName 'generated' strategy ([`aa91000a5`](https://github.com/openclaw/openclaw/commit/aa91000a5))

Introduces two new modules with security-relevant error handling and token validation.

`simple-completion-runtime.ts` exports `prepareSimpleCompletionModel()` and `prepareSimpleCompletionModelForAgent()`. Both wrap `getApiKeyForModel()` calls in try/catch — on failure they return `{ error: string }` rather than propagating rejected promises, providing fail-soft behavior that prevents auth errors from crashing the caller. This partially addresses Audit 2 Claim 2 (API key resolution robustness).

`github-copilot-token.ts` exports `resolveCopilotApiToken()` for GitHub-to-Copilot token exchange with layered validation: the response token is checked for non-empty string; `expires_at` is parsed defensively (accepts both Unix seconds and milliseconds — values above `10_000_000_000` are treated as ms, others multiplied by 1000). The 5-minute safety margin in `isTokenUsable()` ensures cached tokens are not served past effective expiry. `deriveCopilotApiBaseUrlFromToken()` parses the semicolon-delimited `proxy-ep` field to extract the API base URL — this validates token structure and defends against malformed token responses. Tokens are persisted to `~/.openclaw/credentials/github-copilot.token.json` with safe cache management. This partially addresses Audit 2 Claim 6 (token field shifting — validates token format and expiry on every exchange).

#### Protect fresher codex reauth state ([`bbe6f7fdd`](https://github.com/openclaw/openclaw/commit/bbe6f7fdd))

Adds `hasNewerStoredOAuthCredential()` to `external-cli-sync.ts`. The function returns true if the existing stored credential has a later (higher) expiry than the incoming credential from an external CLI sync. `shouldReplaceStoredOAuthCredential()` is updated to return false in this case — preventing an external CLI sync from overwriting a token that was refreshed more recently in-process. This closes a token refresh race window (Audit 1 Claim 4) where a slow background sync could clobber a fresher auth state.

`readFileMtimeMs()` is added to `cli-credentials.ts` to read a file's mtime via `fs.statSync().mtimeMs`. The `readCachedCliCredential()` function (introduced in `8e9e2d2f4`) now uses this as the `readSourceFingerprint` for Codex credential files — if the file mtime changed between the pre-read fingerprint and the post-read fingerprint, the cache entry is discarded rather than written. This prevents a TOCTOU window where a credential file replacement between stat and read could result in caching a stale value, partially addressing Audit 1 Claim 8 (expiry validation — mtime-based cache freshness).
