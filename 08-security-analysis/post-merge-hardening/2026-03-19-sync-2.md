#### Block build-tool and glibc env injection vectors in host exec sandbox ([`089a43f5e`](https://github.com/openclaw/openclaw/commit/089a43f5e))

The host exec sandbox env var blocklist in `src/infra/host-env-security-policy.json` gains six new `blockedKeys` and one new `blockedOverrideKey`:

- **New `blockedKeys`:** `DOTNET_ADDITIONAL_DEPS`, `GLIBC_TUNABLES`, `MAVEN_OPTS`, `SBT_OPTS`, `GRADLE_OPTS`, `ANT_OPTS`
- **New `blockedOverrideKey`:** `GRADLE_USER_HOME`

These entries close a category of injection vectors where an attacker-controlled build-tool option or glibc tuning variable could alter the runtime behavior of processes spawned inside the host exec sandbox. `GLIBC_TUNABLES` is particularly high-value: it controls low-level allocator and threading behavior in glibc and has been used in privilege-escalation exploits (CVE-2023-4911). The Maven, Gradle, Ant, SBT, and .NET Additional Deps variables allow arbitrary JVM/CLR option injection that can load attacker-supplied code into spawned build-tool processes. `GRADLE_USER_HOME` in `blockedOverrideKeys` prevents redirecting Gradle's home to an attacker-controlled path, which could be used to override plugins or init scripts.

A corresponding generated Swift policy file (`HostEnvSecurityPolicy.generated.swift`) is updated automatically, and a regression test (`src/infra/host-env-security.test.ts`) verifies all seven new entries are correctly classified as blocked. Closes upstream issue #22681.
