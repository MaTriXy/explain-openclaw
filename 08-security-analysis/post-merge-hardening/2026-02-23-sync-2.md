## Key Security Changes

### Fail-Closed Group Policy Enforcement ([`777817392`](https://github.com/openclaw/openclaw/commit/777817392))

Channel providers across all message adapters (Discord, Slack, Telegram, WhatsApp, iMessage, Signal, Matrix, etc.) now fail closed when group policy configuration is missing. Previously, missing policy could allow unauthorized message processing. The fix adds mandatory policy checks that reject messages when the policy is undefined, ensuring defense-in-depth for channel access control.

Related to Audit 2 Claim 5 (self-approving agent) — proper group policy gating prevents unauthorized channel access.

### BlueBubbles Null Status Fail-Closed ([`888b6bc94`](https://github.com/openclaw/openclaw/commit/888b6bc94))

BlueBubbles private API status handling now treats `null` status as disabled rather than enabled. This prevents potential security bypasses where an unknown or unset status could inadvertently enable privileged features.

### Directive Tag Sanitization ([`78c3c2a54`](https://github.com/openclaw/openclaw/commit/78c3c2a54))

Chat directive tags are now sanitized to remove potentially dangerous content before processing. The `sanitizeDirectiveTags()` helper strips malformed tags that could enable injection attacks through directive processing.

### CLI Scope Expansion ([`6f7e5f92c`](https://github.com/openclaw/openclaw/commit/6f7e5f92c))

Default CLI scopes now include `operator.read` and `operator.write` permissions, enabling proper operator functionality without requiring manual scope configuration. This ensures CLI tools have appropriate authorization by default.

### Auth-Choice Unification ([`fc60f4923`](https://github.com/openclaw/openclaw/commit/fc60f4923))

API key resolution flows across providers (HuggingFace, MiniMax, etc.) are now unified through shared helpers. This reduces code duplication and ensures consistent auth handling patterns across all API providers.
