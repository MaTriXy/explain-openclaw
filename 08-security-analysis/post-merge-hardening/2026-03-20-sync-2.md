#### Increase WS handshake timeout from 3s to 10s ([`36f394c29`](https://github.com/openclaw/openclaw/commit/36f394c29))

The gateway WebSocket handshake timeout is raised from 3 seconds to 10 seconds. A new named constant `DEFAULT_HANDSHAKE_TIMEOUT_MS` (value `10_000`) is exported from `src/gateway/server-constants.ts` and referenced in the timeout getter, replacing the previous inline value. Existing constants `DEDUPE_TTL_MS` and `DEDUPE_MAX` are unaffected in value but shift to subsequent lines. The change prevents premature handshake failures under network congestion or CPU load — a reliability fix with no audit-claim impact.

#### Resolve Claude marketplace shortcuts from OS home ([`dcbcecfb8`](https://github.com/openclaw/openclaw/commit/dcbcecfb8))

Three new path-resolution helpers are added to `src/infra/home-dir.ts`: `resolveOsHomeDir()`, `resolveRequiredOsHomeDir()` (throws if home is unavailable), and `resolveOsHomeRelativePath()`. These helpers resolve paths relative to the OS-reported home directory rather than a caller-supplied working directory.

`src/plugins/marketplace.ts` is updated to use `resolveOsHomeRelativePath()` when locating `CLAUDE_KNOWN_MARKETPLACES_PATH` (`~/.claude/plugins/known_marketplaces.json`), replacing the prior `resolveUserPath()` call. Anchoring the marketplace shortcut file to the OS home directory reduces the risk of path confusion if the process working directory differs from the user home.

This commit is adjacent to Audit 1 Claim 6 (path traversal via agent-controlled directory inputs). The new helpers add a clearer, tested boundary for OS home resolution, strengthening path precision at this boundary without directly closing the claim.
