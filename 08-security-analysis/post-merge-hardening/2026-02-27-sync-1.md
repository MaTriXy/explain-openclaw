#### Pin paired reconnect metadata for node policy ([`7d8aeaaf0`](https://github.com/openclaw/openclaw/commit/7d8aeaaf0))

Introduces `buildDeviceAuthPayloadV3()` in the gateway device auth module to pin `platform` and `deviceFamily` metadata in paired device reconnection flows. The implementation validates that reconnect metadata matches the original pairing request, preventing attackers from spoofing a different device type after initial pairing. A new test validates rejection of platform identity spoofing during reconnect.

#### Harden root file guards and host writes ([`e3385a657`](https://github.com/openclaw/openclaw/commit/e3385a657))

Consolidates workspace file write safety via `writeFileWithinRoot()`, which enforces that destination paths remain within the workspace root, rejects hardlink aliases (nlink > 1), and applies atomic write semantics with lstat/stat verification. The commit covers agent patch application via `apply-patch.ts`, pi-tools write operations, and gateway agent file mutation — eliminating hardlink-based write escape vectors in all three code paths.

#### Block broken-symlink sandbox path escapes ([`4fd29a35b`](https://github.com/openclaw/openclaw/commit/4fd29a35b))

Extends `assertNoPathAliasEscape()` in the path alias guards module to detect broken (dangling) symlink targets where the final path component does not yet exist. This closes an edge case where a symlink pointing to a non-existent path outside the sandbox boundary could be created to escape containment once the target was written. Test coverage validates rejection of dangling symlink escapes at both lexical and resolved levels.

#### Harden workspace boundary path resolution ([`46eba86b4`](https://github.com/openclaw/openclaw/commit/46eba86b4))

Introduces the `boundary-path.ts` module implementing dual-layer symlink escape detection: lexical path analysis and canonical path resolution via `resolveBoundaryPath()` and `resolveBoundaryPathSync()`. The module detects both existing and dangling symlink escape patterns, and integrates with `boundary-file-read.ts` to provide consistent workspace path validation across all file operations that cross the sandbox boundary.

#### Centralize channel allowlist auth policy ([`892a9c24b`](https://github.com/openclaw/openclaw/commit/892a9c24b))

Centralizes DM/group allowlist resolution into `resolveEffectiveAllowFromLists()` in `src/security/dm-policy-shared.ts` to eliminate duplicated authorization logic across IRC, Mattermost, MS Teams, iMessage, and Telegram channel adapters. The helper ensures consistent treatment of pairing-store entries per `dmPolicy` and prevents accidental group allowlist fallback to DM allowlist that could permit unauthorized senders in group contexts.

#### Unify boundary hardening for file reads ([`eac86c208`](https://github.com/openclaw/openclaw/commit/eac86c208))

Consolidates file read boundary checking via `openBoundaryFileSync()` (and async `openBoundaryFile()`) across plugin discovery, plugin loader, plugin manifest reading, and workspace agent file operations. All paths now reject hardlinked file aliases (nlink > 1) during discovery and load phases, preventing hardlink-based escape chains where a directory entry symlinks or hardlinks to a file outside the sandbox.

#### Avoid duplicate feishu permission-error dispatch replies (#27381) ([`cf4853e2b`](https://github.com/openclaw/openclaw/commit/cf4853e2b))

Non-security commit. Fixes Feishu bot test setup to properly mock client dependencies, preventing duplicate permission error messages during inbound message handling. No production code changes.

#### Extract protected-route path policy helpers ([`6632fd1ea`](https://github.com/openclaw/openclaw/commit/6632fd1ea))

Extracts HTTP path canonicalization and protected-prefix matching into `src/gateway/security-path.ts`. The module provides `canonicalizePathForSecurity()` (handles case normalization, percent-encoding decoding, malformed encoding fail-closed), `isProtectedPluginRoutePath()`, and the `PROTECTED_PLUGIN_ROUTE_PREFIXES` constant (`/api/channels`). Protected gateway routes now enforce authentication even under case/encoding variants that could otherwise bypass prefix matching.

#### Land NO_REPLY announce suppression and auth scope assertions ([`96aad965a`](https://github.com/openclaw/openclaw/commit/96aad965a))

Lands the NO_REPLY announce suppression fix for subagent spawning (suppresses the "spawn accepted" message before Slack API call) and updates gateway auth tests to assert correct scope behavior after the scope preservation fix in `9c142993b`. The test suite now verifies that operator connections via shared token retain their scopes rather than being cleared to zero, formalizing the expected auth contract.

#### Preserve operator scopes for shared auth connections ([`9c142993b`](https://github.com/openclaw/openclaw/commit/9c142993b))

Fixes scope clearing in `clearUnboundScopes()` within the gateway WebSocket message handler: previously, connecting via a shared gateway token (no device identity) cleared operator scopes even for `sharedAuthOk` connections, causing downstream API operations to fail with "missing scope" errors. The fix guards scope clearing with `!sharedAuthOk`, ensuring headless API operator clients retain their requested scopes through the connection lifecycle.

#### Accept mode/apiKey aliases to prevent silent credential loss ([`7e7ca43a7`](https://github.com/openclaw/openclaw/commit/7e7ca43a7))

Auth-profiles parsing now accepts `mode` as an alias for the credential type field and `apiKey` as an alias for the key field, matching the `openclaw.json` documentation examples. Without this fix, users who followed the documented examples would silently lose their API credentials at auth-profile load time because the canonical field names differ from the documented aliases.

#### Sign gateway device auth with v3 payload ([`490cb5174`](https://github.com/openclaw/openclaw/commit/490cb5174))

Updates the iOS (Swift `WizardCommand`, `GatewayChannel`) and Android (`GatewaySession`) client implementations to sign device auth using the V3 payload format, aligning with the gateway's updated `buildDeviceAuthPayloadV3()` introduced in `7d8aeaaf0`. This ensures end-to-end consistency: the gateway requires V3 signatures and the mobile clients now generate them, closing a potential auth downgrade path.

#### Gate inline directives on resolved auth ([`473a27470`](https://github.com/openclaw/openclaw/commit/473a27470))

In `resolveReplyDirectives()`, inline directive processing now gates on `command.isAuthorizedSender` (the resolved authorization result) instead of the intermediate `commandAuthorized` flag. This ensures that channels where `commands.allowFrom` is the authorization source (such as LINE) correctly enable inline directives for permitted senders rather than incorrectly stripping them.

#### Fallback bundled channel specs when npm install returns 404 ([`cf311978e`](https://github.com/openclaw/openclaw/commit/cf311978e))

Adds `bundled-sources.ts` with a `BundledSourceResolver` that maintains a registry of known bundled channel plugin specifications. When the npm installer receives a 404 response for a plugin package, the CLI now falls back to the bundled source instead of failing, preventing plugin installation failures caused by network errors or package removal from npm.

#### Dedupe boundary-path canonical checks ([`7b5153f21`](https://github.com/openclaw/openclaw/commit/7b5153f21))

Refactors `boundary-path.ts` to extract duplicated canonical path checking logic into shared helpers (84 insertions, 45 deletions). Reduces the maintenance surface for the path traversal prevention code and ensures consistent canonical alias detection behavior across all callers of the boundary-path module.

#### Harden drain/abort/timeout race handling ([`c397a02c9`](https://github.com/openclaw/openclaw/commit/c397a02c9))

Hardens the command queue against race conditions: new lane enqueues are rejected once gateway drain begins; lane draining state is always reset and `onWait` callback failures are isolated; per-session abort cutoff is persisted to skip stale queued messages after abort; and false 600-second agent turn timeouts in isolated cron sessions are avoided. These changes close windows where a race between drain/abort/timeout could leave stale messages processed after session termination.

#### Harden boundary-path canonical alias handling ([`1aef45bc0`](https://github.com/openclaw/openclaw/commit/1aef45bc0))

Adds 31 tests and hardens edge-case handling in the canonical alias detection logic of `boundary-path.ts`. The fix targets cases where canonical path resolution could incorrectly classify an aliased path as safe by missing trailing-component alias patterns or non-normalized input forms.

#### Unify session-key normalization and plugin boundary checks ([`4b71de384`](https://github.com/openclaw/openclaw/commit/4b71de384))

Extracts cron isolated-agent session-key normalization into a dedicated `session-key.ts` module, unifying it with the gateway hooks boundary check logic. This eliminates divergent normalization paths that could produce inconsistent session key formats across cron and gateway contexts, and extends plugin boundary enforcement to cover the unified normalization path.
