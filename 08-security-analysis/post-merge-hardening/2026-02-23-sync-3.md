## Key Security Changes

### ThoughtSignature Sanitization for Native Google ([`9176571ec`](https://github.com/openclaw/openclaw/commit/9176571ec))

Extended `sanitizeThoughtSignatures` policy to native Google Gemini provider. This prevents context overflow from Base64 thoughtSignature blobs (2K-8K tokens per turn) when using native Google API instead of OpenRouter. The sanitization strips non-Base64 thought signatures to maintain consistent transcript handling.

### MSTeams Graph User and Token Parsing ([`8c1afc4b6`](https://github.com/openclaw/openclaw/commit/8c1afc4b6))

Added `searchGraphUsers` function with proper OData escaping for email/userPrincipalName queries and `readAccessToken` function to safely extract tokens from response objects. Improves input validation and prevents injection in Microsoft Graph API queries.

### Feishu Onboarding and Webhook Validation ([`5574eb6b3`](https://github.com/openclaw/openclaw/commit/5574eb6b3))

Refactored `FeishuSharedConfigShape` to reduce duplication and improved webhook security tests. Enhanced test coverage for content-type validation and rate limiting in webhook handling.

### BlueBubbles Chat Target Handling ([`5056f4e14`](https://github.com/openclaw/openclaw/commit/5056f4e14))

Added `sendPrivateApiJsonRequest` and `resolvePartIndex` helper functions to standardize private API calls with consistent error handling. Tightened target parsing with `parseGroupTarget` and `parseRawChatIdentifierTarget` functions.

### Extension Type Errors and Probe Mocks ([`26763d191`](https://github.com/openclaw/openclaw/commit/26763d191))

Added `LegacyRuntimeLogShape` type for backward compatibility and `isBlueBubblesPrivateApiStatusEnabled` function. Hardened probe mock modules with better type safety across BlueBubbles, Feishu, and Line extensions.

### Voice-Call Manager Outbound Behavior ([`081ab9c99`](https://github.com/openclaw/openclaw/commit/081ab9c99))

Added `lookupConnectedCall` function to validate call state, provider connectivity, and terminal states before operations like `speak`, `continueCall`, and `endCall`. Tightened error handling with explicit state validation.
