#### Harden replaceMarkers() Boundary Marker Variants ([`d1a59557b`](https://github.com/openclaw/openclaw/commit/d1a59557b))

Defense-in-depth hardening of `replaceMarkers()` in `src/security/external-content.ts`. Extends `ANGLE_BRACKET_MAP` with Unicode arrowhead variants (U+02C2, U+02C3) that LLMs could parse as angle brackets. Expands the `foldMarkerText()` regex to match both space and underscore delimiters (`[\s_]+`), catching boundary markers that use spaces, tabs, or underscores instead of the expected format. Test coverage added for mixed-delimiter and tab-separated marker variants.

#### Telegram Exec Approvals for OpenCode/Codex ([`de49a8b72`](https://github.com/openclaw/openclaw/commit/de49a8b72))

Implements explicit exec approval routing across channels, primarily Telegram. Adds `resolveExecApprovalInitiatingSurfaceState()` gating to prevent self-approving agents (Audit 2 Claim 5). New modules `exec-approval-surface.ts` and `exec-approval-reply.ts` enforce structured approval routing with configurable DMRoute. The `sendExecApprovalFollowup()` function handles approval button delivery. Gateway validates approval IDs bidirectionally. Large commit (78 files) touching exec host paths for both gateway and node contexts.

#### Mark Child Command Env with OPENCLAW_CLI ([`b48291e01`](https://github.com/openclaw/openclaw/commit/b48291e01))

Introduces `OPENCLAW_CLI` env marker via `markOpenClawExecEnv()` and `ensureOpenClawExecMarkerOnProcess()`. The marker is injected at process entry and propagated through all exec paths: host exec via `resolveCommandEnv()`, sandbox docker via `buildSandboxCreateArgs()`, and env security via `sanitizeHostExecEnv()`. Enables downstream security policies to reject unmarked subprocesses, providing defense-in-depth against LD_PRELOAD/DYLD_* injection (Audit 2 Claim 8).

#### Gateway Pending Node Work Primitives ([`ef9597541`](https://github.com/openclaw/openclaw/commit/ef9597541))

Adds async work queue infrastructure with `NodePendingWorkItem` type and `drainNodePendingWork()`/`enqueueNodePendingWork()` in `src/gateway/node-pending-work.ts`. New gateway methods `node.pending.drain` (node-only scope) and `node.pending.enqueue` (operator-only scope) enforce role-based access via `src/gateway/method-scopes.ts`. Params validated via schema with priority and TTL controls. No direct security vulnerability, but the role-based scoping is correctly enforced.

#### Chat UI Infrastructure — Add and Revert ([`5a659b0b6`](https://github.com/openclaw/openclaw/commit/5a659b0b6), [`6b8748989`](https://github.com/openclaw/openclaw/commit/6b8748989))

#### Harden Global Hook Runner State ([`12702e11a`](https://github.com/openclaw/openclaw/commit/12702e11a))

Moves plugin hook runner state from module-level `let` globals to Symbol-keyed `globalThis` storage via `getHookRunnerGlobalState()` using `Symbol.for("openclaw.plugins.hook-runner-global-state")` in `src/plugins/hook-runner-global.ts`. Prevents state leakage between module reloads and ensures singleton behavior across dynamic import boundaries. Defense-in-depth against plugin state corruption in hot-reload scenarios.
