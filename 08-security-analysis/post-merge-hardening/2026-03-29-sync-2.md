#### Trim provider oauth runtime seams ([`ce2444403`](https://github.com/openclaw/openclaw/commit/ce2444403))

Trims unused runtime helper seams from provider OAuth modules (`extensions/google/oauth.flow.ts`, `extensions/google/oauth.http.ts`, `extensions/minimax/oauth.ts`). These seams were integration points for the provider runtime layer that are no longer needed after the provider scoping refactor. No changes to actual OAuth flow logic, token handling, or state verification — purely dead code removal.

#### Canonicalize chat final session routing ([`2638b566f`](https://github.com/openclaw/openclaw/commit/2638b566f))

Fixes a regression in `handleChatFinalSession()` where session key resolution could route to the wrong session after provider canonicalization. The fix ensures `resolveCanonicalSessionKey()` is called consistently before routing decisions, preventing cross-session leakage in multi-session gateway scenarios.

#### Canonicalize chat inject session routing ([`a0f48f099`](https://github.com/openclaw/openclaw/commit/a0f48f099))

Same canonicalization fix applied to `handleChatInjectSession()`. Ensures session injection (tool call routing) uses the canonical session key, closing a potential cross-session routing gap where injected messages could land in a stale session reference.

#### Dedupe security utility suites ([`2accc0391`](https://github.com/openclaw/openclaw/commit/2accc0391))

Test-only: consolidates 7 duplicated security test suites (`audit-extra.sync`, `dm-policy-shared`, `external-content`, `fix`, `safe-regex`, `skill-scanner`, `temp-path-guard`) into shared helpers. No production code changes. Improves test maintainability and reduces false-positive test surface.

#### Preserve gateway subagent session change metadata ([`ae9b9575c`](https://github.com/openclaw/openclaw/commit/ae9b9575c))

Fixes a regression where subagent session change events lost ownership metadata during gateway relay. The fix ensures `SessionRouteMeta` (including `spawnedBy`, `subagentControlScope`) is preserved through the gateway event pipeline, maintaining session isolation boundaries for subagent routing.

#### Preserve session thread ids in agent send events ([`f4a45071e`](https://github.com/openclaw/openclaw/commit/f4a45071e))

Fixes missing thread ID propagation in `handleAgentSendEvent()`. The fix carries `threadIds` through the agent send pipeline so that thread-scoped routing (e.g., Discord thread bindings) is preserved when agents send messages, preventing thread-bound messages from escaping to the parent channel.

#### Preserve subagent session ownership metadata ([`7fadb4f7f`](https://github.com/openclaw/openclaw/commit/7fadb4f7f))

Fixes a regression in session ownership tracking across subagent boundaries. The fix propagates `GatewaySessionRow` metadata (spawned-by, fork status, control scope) through `SessionUtils` and session event types, ensuring subagents cannot escalate beyond their declared control scope.

#### Preserve session thread ids in sessions changed events ([`1fee91e43`](https://github.com/openclaw/openclaw/commit/1fee91e43))

*Escalated from Tier B.* Ensures `threadIds` are preserved in sessions-changed SSE events sent to gateway clients. Without this fix, thread-scoped sessions could lose their thread bindings after a session metadata update, causing subsequent messages to route to the wrong thread context.

#### Use origin thread metadata in tools effective context ([`53861607f`](https://github.com/openclaw/openclaw/commit/53861607f))

*Escalated from Tier B.* Fixes `toolsEffectiveContext()` to use the origin thread's metadata when building tool authorization context. This ensures tool execution respects the thread-level permissions of the originating thread, not the parent channel defaults.

#### Narrow provider runtime auth seams ([`e8866fc73`](https://github.com/openclaw/openclaw/commit/e8866fc73))

*Escalated from Tier B.* Narrows the provider runtime auth surface by removing unused auth seam exports from 15 provider modules. Trims `providerAuthRuntime` re-exports that were exposing internal auth state beyond the plugin SDK boundary. The remaining auth exports are now scoped to `providerAuthRuntime` via the plugin SDK subpath, reducing the attack surface for auth state leakage through provider plugins.

#### Normalize catalog provider ids for probe model selection ([`50c87c468`](https://github.com/openclaw/openclaw/commit/50c87c468))

*Escalated from Tier B.* Normalizes provider IDs in the model probe path to match canonical catalog format. This prevents probe requests from bypassing provider-specific configurations (including auth and scope settings) by using non-canonical provider aliases.

#### Preserve session thread ids in chat session snapshots ([`627d6c80f`](https://github.com/openclaw/openclaw/commit/627d6c80f))

*Escalated from Tier B.* Ensures `threadIds` and `routeMeta` are included in chat session snapshot payloads. Without this, session snapshots sent to clients could lose thread association, causing reconnected clients to route messages to the channel root instead of the originating thread.

#### Align model picker with bundled compat ([`67fba9c5e`](https://github.com/openclaw/openclaw/commit/67fba9c5e))

*Escalated from Tier B.* Aligns the model picker flow with the bundled provider compatibility layer, ensuring model selection respects provider-level auth requirements. Prevents models from being offered in the picker when their backing provider lacks valid credentials.

#### Keep numeric session thread ids in sessions list ([`1b16a112e`](https://github.com/openclaw/openclaw/commit/1b16a112e))

*Escalated from Tier B.* Ensures numeric thread IDs (used by Discord, Slack, etc.) are preserved in the sessions list API response. Without this fix, platforms using numeric thread IDs would have their thread bindings dropped, causing messages to route to the parent channel instead of the thread.
