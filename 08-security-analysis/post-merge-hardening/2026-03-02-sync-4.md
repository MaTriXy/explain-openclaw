#### Harden root-scoped writes against symlink races ([`710004e01`](https://github.com/openclaw/openclaw/commit/710004e01))

The `writeFileWithinRoot` function in `src/infra/fs-safe.ts` previously opened a file for writing with flags that combined creation and truncation in a single step. This allowed a race: a symlink could be retargeted after validation (confirming the path is within the root) but before truncation took effect, causing `writeFileWithinRoot` to truncate an out-of-root file.

The fix splits the single `OPEN_WRITE_FLAGS` constant into two: `OPEN_WRITE_EXISTING_FLAGS` (opens without truncation) for existing file writes, and `OPEN_WRITE_CREATE_FLAGS` (`O_EXCL` exclusive-create) for new files. Truncation via `handle.truncate(0)` is deferred until after all boundary checks and identity verification complete. If the deferred truncation would apply to an out-of-root file created by a successful race, that file is cleaned up. New test coverage validates the write-open and file-create race phases.

#### Refactor!: remove versioned system-run approval contract ([`155118751`](https://github.com/openclaw/openclaw/commit/155118751))

Removes the versioning layer from the `system-run` approval contract. Previously, `exec-approvals-analysis`, `system-run-approval-binding`, and `system-run-approval-context` carried version-dependent approval resolution branches and a version migration path. This refactor consolidates approval validation and matching into a single unversioned flow, eliminating code paths that differed based on contract version. Version mismatch edge cases — a potential bypass surface if mismatched versions produced different approval verdicts — are removed. The net change touches 22 production files and 11 test files.

#### Fixes minor security vulnerability (#30948) ([`b81e1b902`](https://github.com/openclaw/openclaw/commit/b81e1b902))

Expands homograph detection coverage in `src/security/external-content.ts`. The `ANGLE_BRACKET_MAP` now includes 8 additional Unicode angle bracket variants: guillemets (`«»‹›`), CJK angle brackets, mathematical angle brackets, flattened parentheses variants, and angle bracket ornaments. The `foldMarkerText` function normalizes these confusable characters to their ASCII `<` or `>` equivalents before external content marker detection runs. This prevents spoofing attacks where an attacker injects Unicode look-alikes to bypass marker recognition and smuggle external content into trusted context.

#### Enforce owner boundary for agent runs ([`58659b931`](https://github.com/openclaw/openclaw/commit/58659b931))

Introduces the `senderIsOwner` flag to gate owner-only tool access in agent runs. The new `resolveSenderIsOwnerFromClient` function in `src/gateway/server-methods/agent.ts` checks whether the requesting gateway client holds `ADMIN_SCOPE` (`operator.admin`): if yes, `senderIsOwner=true`; otherwise false. Local CLI invocations default to true. Node event handlers for `voice.transcript` and `link-agent` invocations explicitly set `senderIsOwner=false`. Boot runs default to true.

The flag feeds into `applyOwnerOnlyToolPolicy` which wraps tools marked `ownerOnly` — if the sender is not an owner, those tools return a rejection without executing. This enforces the RBAC boundary that prevents non-owner gateway callers (e.g. `operator.write` scope) from invoking privileged tools. See GHSA-jr6x-2q95-fh2g.

#### Unify metadata canonicalization + platform rules ([`9005e8bc0`](https://github.com/openclaw/openclaw/commit/9005e8bc0))

Extracts device metadata normalization into `src/gateway/device-metadata-normalization.ts` with two purpose-specific functions. `normalizeDeviceMetadataForAuth` applies ASCII-only lowercasing for cross-runtime consistency between TypeScript, Swift, and Kotlin auth flows. `normalizeDeviceMetadataForPolicy` applies Unicode NFKD normalization plus accent stripping, collapsing confusable characters (e.g. dotted-i İOS, Greek omicron iPhοne) to ASCII before platform/family matching.

This separates a previously unified normalization path into distinct auth (compatibility) and policy (security) transforms. Platform detection rules in `src/gateway/node-command-policy.ts` now call `normalizeDeviceMetadataForPolicy` to prevent Unicode spoofing attacks on device classification.

#### Add sessions_spawn sandbox require mode ([`bfeadb80b`](https://github.com/openclaw/openclaw/commit/bfeadb80b))

Adds a `sandbox="require"` option to the `sessions_spawn` tool in `src/agents/tools/sessions-spawn-tool.ts`. When a caller specifies `sandbox="require"`, `src/agents/subagent-spawn.ts` sets `sandboxMode` to `"require"` instead of the default `"inherit"`. The require mode asserts that the target agent runtime must be sandboxed — if it is not, spawn fails with an explicit error rather than silently running without sandbox isolation. This gives operators a declarative way to enforce sandbox boundaries when spawning sub-agents, preventing accidental escalation through unsandboxed child agents. See GHSA-p7gr-f84w-hqg5.

#### Harden sandbox media reads against TOCTOU escapes ([`c823a8530`](https://github.com/openclaw/openclaw/commit/c823a8530))

Adds `readFileWithinRoot`, `readPathWithinRoot`, and `createRootScopedReadFile` to `src/infra/fs-safe.ts`. These functions perform boundary-validated reads at the file descriptor level — the file is opened via `openFileWithinRoot` (which resolves real paths and checks symlink targets) and read via the returned handle, eliminating the check-time-to-use gap between `stat`/`realpath` validation and subsequent `readFile` calls.

Adds `createSandboxBridgeReadFile` to `src/agents/sandbox-media-paths.ts`, which routes media file reads through the sandbox bridge's `readFile` interface rather than direct `fs` calls. Image tool and embedded runner image processing are updated to use these hardened read paths. This closes TOCTOU escape vectors where an attacker could swap a valid media path for a symlink to sensitive files between the sandbox boundary check and the actual read. See GHSA-7xmq-g46g-f8pv.

#### Harden node metadata policy classification ([`0eac494db`](https://github.com/openclaw/openclaw/commit/0eac494db))

Hardens `src/gateway/node-command-policy.ts` against Unicode spoofing and unknown-metadata escalation. The commit adds inline NFKD normalization and accent stripping to platform token processing, collapsing confusable characters that bypassed the previous `trim().toLowerCase()` pass. (This normalization logic was subsequently centralized into `normalizeDeviceMetadataForPolicy` by the follow-on commit [`9005e8bc0`](https://github.com/openclaw/openclaw/commit/9005e8bc0).)

Additionally, the `PLATFORM_DEFAULTS` entry for `unknown` is tightened: `SYSTEM_COMMANDS` (host execution defaults) are removed from the unknown-platform allowlist, replaced by the narrower `UNKNOWN_PLATFORM_COMMANDS` constant containing only `CANVAS_COMMANDS`, `CAMERA_COMMANDS`, `LOCATION_COMMANDS`, and `NODE_SYSTEM_NOTIFY_COMMAND`. Nodes presenting unrecognized or spoofed platform metadata no longer receive host execution command defaults. This pairs with `9005e8bc0`'s `normalizeDeviceMetadataForPolicy` to provide a two-layer defense: normalization at the metadata layer, fail-safe defaults at the policy layer.

#### Block private-network web_search citation redirects ([`085c23ce5`](https://github.com/openclaw/openclaw/commit/085c23ce5))

The `resolveRedirectUrl` function in `src/agents/tools/web-search.ts` previously passed a permissive SSRF policy (`WEB_TOOLS_TRUSTED_NETWORK_SSRF_POLICY` from `src/agents/tools/web-guarded-fetch.ts`) that allows private network access when following citation redirect URLs. This allowed search citation redirects to reach internal network addresses, creating an SSRF vector.

The fix removes the `policy` parameter from the `resolveRedirectUrl` guarded fetch call. Without an explicit policy, the request uses the default strict SSRF policy which blocks private-network destinations. Citation redirect following now has the same network access restrictions as regular web tool fetches. See GHSA-g99v-8hwm-g76g.
