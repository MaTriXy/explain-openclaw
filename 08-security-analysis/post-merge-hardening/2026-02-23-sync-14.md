#### Unify auth credential resolution ([`66529c7aa`](https://github.com/openclaw/openclaw/commit/66529c7aa))

This refactor consolidates gateway credential resolution logic that was previously duplicated across multiple callsites with subtle inconsistencies. The new `resolveGatewayCredentialsFromConfig()` function in `src/gateway/credentials.ts` establishes a single, authoritative precedence chain:

- **Explicit credentials** (CLI flags or programmatic parameters) always take priority
- **Remote config** credentials (used when in remote gateway mode) come next
- **Environment variables** (`OPENCLAW_GATEWAY_TOKEN`, `CLAWDBOT_GATEWAY_TOKEN`, etc.) are third
- **Local config** (`gateway.auth.token`) is the final fallback

Key security improvements from the consolidation: consistent `trimToUndefined()` application prevents whitespace-only tokens from bypassing validation checks, and the precedence rules are now enforced at a single point rather than being re-implemented per callsite.

The commit also introduces `resolveConnectAuthState()` in `src/gateway/server/ws-connection/auth-context.ts`, which separates shared (gateway-level) and per-device authentication state. Device tokens can now authenticate independently from the shared gateway token, with rate limiting scoped correctly to each token type. 124 new test cases cover credential precedence ordering and device-token isolation logic.

#### Add guarded remote HTTP helper ([`f6feb4144`](https://github.com/openclaw/openclaw/commit/f6feb4144))

Introduces `src/memory/remote-http.ts` with two new functions: `withRemoteHttpResponse()` and `buildRemoteBaseUrlPolicy()`. These provide a standard way for the memory subsystem to make outbound HTTP calls that are routed through `fetchWithSsrFGuard()` — the existing SSRF guard that validates URLs against a policy before making requests. Previously the memory subsystem made outbound calls without this protection. This commit lays the infrastructure for the SSRF enforcement applied in the two commits that follow.

#### Enforce guarded remote policy for embeddings ([`f87db7c62`](https://github.com/openclaw/openclaw/commit/f87db7c62))

Migrates all embeddings API calls (Gemini, OpenAI, Voyage) to use `withRemoteHttpResponse()` from the new helper module. Any outbound HTTP request for embedding generation now passes through the SSRF guard before being dispatched. This prevents the embeddings subsystem from being redirected to internal network addresses if a user-controlled base URL were set to a malicious value.

#### Route batch APIs through guarded remote HTTP ([`eb041daee`](https://github.com/openclaw/openclaw/commit/eb041daee))

Extends the same SSRF guard enforcement to all batch API calls: batch-gemini, batch-openai, batch-voyage, and batch-upload. Together with `f6feb4144` and `f87db7c62`, this completes SSRF protection across the full memory API surface. All outbound memory HTTP — single-shot embeddings, batch operations, and remote uploads — now routes through `fetchWithSsrFGuard()`.
