#### Infra: require explicit opt-in for prerelease npm installs ([`f392b81e9`](https://github.com/openclaw/openclaw/commit/f392b81e9))

Adds a supply-chain gate that prevents accidental installation of prerelease npm packages. The new `parseRegistryNpmSpec()` function in `src/infra/npm-registry-spec.ts` classifies raw package selectors into name-only, exact-version, or dist-tag forms, with `EXACT_SEMVER_VERSION_RE` detecting prerelease semver strings. When `isPrereleaseResolutionAllowed()` determines that a bare spec would resolve to a prerelease version, installation is rejected with a `formatPrereleaseResolutionError()` message guiding users to use an explicit dist-tag (e.g. `@beta`) or exact prerelease version string. This policy is enforced in both plugin and hook installation flows.

#### Mattermost: harden interaction callback binding ([`a274ef929`](https://github.com/openclaw/openclaw/commit/a274ef929))

Adds three binding checks inside `createMattermostInteractionHandler()` to prevent cross-channel and action injection attacks on interactive Mattermost messages. First, the handler reads `SIGNED_CHANNEL_ID_CONTEXT_KEY` (`"__openclaw_channel_id"`) from the interaction context and verifies it matches `payload.channel_id`, blocking cross-channel replay where an attacker replays an interaction token in a different channel. Second, the handler fetches the original post from the Mattermost API and asserts its stored `channel_id` matches the payload, adding a server-side anchor that cannot be spoofed by a manipulated callback body. Third, the handler validates that the `actionId` from the callback exists in the post's attachment actions list — if not found, the request is rejected with HTTP 403, preventing injection of arbitrary action IDs that were never presented to the user.
