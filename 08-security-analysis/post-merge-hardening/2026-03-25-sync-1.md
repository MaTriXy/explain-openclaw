#### Resolve Aisle findings — skill installer validation, terminal sanitization, URL scheme allowlisting (#53471) ([`cb58e4513`](https://github.com/openclaw/openclaw/commit/cb58e45130))

Adds strict allowlist patterns to `src/agents/skills-install.ts` to prevent option injection and malicious package names: `SAFE_BREW_FORMULA`, `SAFE_NODE_PACKAGE`, `SAFE_GO_MODULE`, and `SAFE_UV_PACKAGE` each enforce per-installer format constraints. A new `assertSafeInstallerValue()` function validates the package/formula string against the pattern before any install command is constructed — values that start with a dash (option injection) or contain invalid characters are rejected with an error. A `trustedInstallSources` set limits scanned installs to `openclaw-bundled`, `openclaw-managed`, and `openclaw-extra` sources. The commit also adds terminal output sanitization in `src/cli/skills-cli.format.ts` and tightens URL scheme allowlisting for links opened from skill panels in the web UI. Addresses Audit 2 Claim 7 (shell injection via incomplete regex).

#### Widen installer regex allowlists and deduplicate safeExternalHref calls ([`b61a875d5`](https://github.com/openclaw/openclaw/commit/b61a875d56))

Follow-up to [`cb58e4513`](https://github.com/openclaw/openclaw/commit/cb58e45130): updates `SAFE_BREW_FORMULA` to allow `@` characters in tap path segments (needed for versioned Homebrew taps), and updates `SAFE_UV_PACKAGE` to handle additional PEP 440 version specifier formats including extras syntax (e.g., `package[extra]>=1.0`). Deduplicates the `safeExternalHref` guard in the web UI layer to remove a redundant parallel call path. Continues to address Audit 2 Claim 7.

#### Prevent unresolved SecretRef from crashing embedded agent runs ([`ac07d8814`](https://github.com/openclaw/openclaw/commit/ac07d8814a))

Adds a new `hasUnresolvedChannelSecrets()` guard in `src/auto-reply/reply/get-reply.ts`. The function inspects channel config fields (`telegram.botToken`, `slack`/`discord` token fields) for unresolved `SecretRef` objects — these appear when config is captured at startup before secrets are resolved. When `getReplyFromConfig()` is called with a stale `configOverride` that still contains raw `SecretRef` objects, `hasUnresolvedChannelSecrets()` detects this and discards the override, falling back to `loadConfig()` which reads the runtime snapshot with fully resolved secrets. This prevents the embedded agent from crashing with "unresolved SecretRef" errors on the Telegram channel monitor path (issue #45838). New defense-in-depth control not tied to existing audit claims.

#### Command auth SecretRef resolution (#52791) ([`d4e3babdc`](https://github.com/openclaw/openclaw/commit/d4e3babdcc))

Refactors `resolveProviderFromContext()` in `src/auto-reply/command-auth.ts` to return `{ providerId, hadResolutionError }` instead of a bare `ChannelId | undefined`. A new `resolveProviderAllowFrom()` function wraps the plugin's `resolveAllowFrom()` call in a try-catch, returning `hadResolutionError: true` on throw or invalid return type, and falling back to config-level `allowFrom`. This ensures that provider resolution failures surface as explicit error signals rather than silent empty results that could lead to unintended authorization bypass.

#### Preserve command auth resolution errors on empty inferred allowlists ([`17c1ee771`](https://github.com/openclaw/openclaw/commit/17c1ee7716))

Addresses a gap where provider resolution errors were silently swallowed when the inferred allowlist came back empty. Adds `commandsAllowFromConfigured` detection in `resolveCommandAuthorization()` to track when `cfg.commands.allowFrom` is explicitly configured. When `providerResolutionError` is true and `commandsAllowFromConfigured` is true, the `commands.allowFrom` list is still evaluated for authorization. `isAuthorizedSender` evaluates as `false` (fail-closed) when provider resolution errors exist and no explicit allow entry matches — errors in provider inference no longer silently bypass `commands.allowFrom` constraints.

#### Fail closed when provider inference drops errored allowlists ([`69a317995`](https://github.com/openclaw/openclaw/commit/69a317995d))

Fixes `probeInferredProviders()` in `src/auto-reply/command-auth.ts` to preserve resolution errors even when a candidate provider's inferred `allowFrom` list is empty. Previously, when `resolveAllowFrom()` threw or returned an invalid result for a candidate, the error was tracked via `hadResolutionError` but discarded when `allowFrom.length === 0` caused the candidate to be filtered out. Now `droppedResolutionError` is set to `true` for such candidates and propagated through the `InferredProviderProbe` result, so callers can detect that inference failed rather than concluding the provider simply has no configured allowFrom entries.
