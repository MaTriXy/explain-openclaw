#### Android QR Setup Code Parsing Hardened ([`036e3e633`](https://github.com/openclaw/openclaw/commit/036e3e633))

The Android `GatewayConfigResolver` QR code parsing was hardened against malformed
JSON payloads. The `jsonField()` helper previously used `.jsonPrimitive` (a Kotlin
serialization extension that throws `JsonDecodingException` when the JSON value is
not a primitive type such as a string, number, or boolean). If an attacker or
corrupted QR code provided a nested JSON object or array in the `setupCode` field,
this would cause an unhandled exception rather than graceful rejection.

The fix replaces `.jsonPrimitive` with `(obj[key] as? JsonPrimitive)`, which returns
null safely on type mismatch. This ensures `resolveScannedSetupCode()` returns null
(rejecting the payload) rather than crashing when the `setupCode` field is
unexpectedly typed. A companion test commit (`ed3412963`) adds explicit coverage for
this scenario, confirming the non-string payload path returns null correctly.

**Scope:** Android client only (`apps/android/`). Server-side security controls and
gateway authentication are unaffected. The onboarding flow now fails gracefully on
malformed QR data rather than surfacing an exception.

### Feature Changes

| SHA | Message | Notes |
|-----|---------|-------|
| [`2e3c05d9d`](https://github.com/openclaw/openclaw/commit/2e3c05d9d) | feat(android): make QR scanning first-class onboarding | New QR onboarding flow |

QR code scanning is now the primary onboarding path in the Android app
(`OnboardingFlow.kt`, `GatewayConfigResolver.kt`). Users can scan the QR code shown
in the gateway setup UI to configure the Android app without manually entering a URL
and token. The `resolveScannedSetupCode()` function supports both a raw setup code
string and a JSON-wrapped `{"setupCode": "..."}` format (as used by the QR display).
