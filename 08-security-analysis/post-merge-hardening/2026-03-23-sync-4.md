#### Guard Interface Discovery Failures ([`3faaf8984`](https://github.com/openclaw/openclaw/commit/3faaf8984f))

`pickPrimaryLanIPv4()` in `src/gateway/net.ts` now wraps `os.networkInterfaces()` in a try/catch, returning `undefined` on failure rather than throwing. The same defensive guard was applied to `src/infra/tailnet.ts` and `src/pairing/setup-code.ts`. In restricted environments (CI containers, hardened OS configs) interface enumeration can throw — these throws previously propagated uncaught and could prevent gateway binding or pairing setup from completing.

#### Preserve Runtime Path Env in Secrets Snapshots ([`24b9baeba`](https://github.com/openclaw/openclaw/commit/24b9baebab))

Adds `RUNTIME_PATH_ENV_KEYS` (HOME, USERPROFILE, OPENCLAW_HOME, OPENCLAW_STATE_DIR, and related path vars) and `mergeSecretsRuntimeEnv()` in `src/secrets/runtime.ts`. When `prepareSecretsRuntimeSnapshot()` is called with a limited `params.env` (e.g., subprocess launch with a stripped environment), the new helper backfills any missing path keys from `process.env`. Without this fix, credential resolution in `resolveUserPath()` could silently fall back to incorrect paths when runtime path vars were absent from the provided env. Relates to Legitimate Gap 1 (env var handling in secrets context).

#### Harden No-Isolate Gateway Auth and Pairing ([`6d34d6279`](https://github.com/openclaw/openclaw/commit/6d34d62795))

Two production security changes despite the `test:` prefix:

In `src/gateway/server/ws-connection/message-handler.ts`, the `clearUnboundScopes()` call is now guarded by a more precise condition: device-less connections using token/password auth or trusted-proxy auth now have their unbound scopes cleared unless the control UI is operating in locally-trusted insecure-auth mode (`preserveInsecureLocalControlUiScopes`). This tightens scope retention for unauthenticated-device connections.

In `src/infra/device-pairing.ts`, `approveDevicePairing()` now filters the scope approval check to operator-prefixed scopes only (via `OPERATOR_SCOPE_PREFIX = "operator."`), narrowing `resolveMissingRequestedScope()` to operator-level grants. Additionally, `requestDevicePairing()` now merges roles and scopes from all pending requests when superseding, preventing scope accumulation from being dropped across repair flows. Relates to Audit 1 Claim 8 (token expiry / device auth integrity).

#### Compose Shared Channel Security Adapters ([`99462776d`](https://github.com/openclaw/openclaw/commit/99462776d1))

Adds `createRestrictSendersChannelSecurity<T>()` factory in `src/plugin-sdk/channel-policy.ts`, composing `createScopedDmSecurityResolver` and `createAllowlistProviderRestrictSendersWarningCollector` into a single `ChannelSecurityAdapter<T>` object. Extension channels (bluebubbles, discord, googlechat, line, mattermost, slack) adopt this adapter, replacing per-channel duplicated DM policy + group warning logic. The typed `ChannelSecurityAdapter` interface ensures both `resolveDmPolicy` and `collectWarnings` are always implemented together — new channels cannot accidentally omit one half of the policy enforcement. Relates to Audit 2 Claim 5 (RBAC / channel authorization model).
