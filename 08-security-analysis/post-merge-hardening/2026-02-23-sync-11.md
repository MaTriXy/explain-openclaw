#### Subagent Flow Null-Guard ([`1152b2586`](https://github.com/openclaw/openclaw/commit/1152b2586))

Adds null-coalescing guards (`?? ""`) before `.trim()` calls in `resolveRequesterStoreKey()` and subagent announce flow. Prevents crashes when `request.message`, `p.runId`, or `requesterSessionKey` are null/undefined from malformed input, hardening the gateway against denial-of-service via crafted requests.

#### Centralized Safe-Bin Policy ([`0d0f4c699`](https://github.com/openclaw/openclaw/commit/0d0f4c699))

Introduces `resolveExecSafeBinRuntimePolicy()` in new file `src/infra/exec-safe-bin-runtime-policy.ts` to centralize safe-bin profile resolution across agents, gateways, and system runners. Adds `isInterpreterLikeSafeBin()` to detect script interpreters (python3, node, bash, ruby, perl, etc.) and `listInterpreterLikeSafeBins()` to audit configs. The security audit (`src/security/audit.ts`) now warns when unprofiled interpreters appear in safeBins. Addresses Audit 2 Claim 8 (env variable injection via LD_PRELOAD) by establishing per-binary profile infrastructure.

#### Explicit Safe-Bin Profiles Required ([`47c3f742b`](https://github.com/openclaw/openclaw/commit/47c3f742b))

Removes the fallback `SAFE_BIN_GENERIC_PROFILE` — `isSafeBinUsage()` now returns false for any safeBin lacking an explicit profile in caller-provided `safeBinProfiles`. Validates profiles at config load via `normalizeSafeBinProfileFixtures()`. Extends Zod schema with `ToolExecSafeBinProfileSchema` to enforce profile structure (minPositional, maxPositional, allowedValueFlags, deniedFlags). Closes the gap where interpreters could execute without per-binary constraints, directly addressing Audit 2 Claim 8.

#### Shell Env Allowlist Bypass Blocked ([`e80c803fa`](https://github.com/openclaw/openclaw/commit/e80c803fa))

Adds `SHELLOPTS` and `PS4` to `host-env-security-policy.json` blocked keys — these shell variables enable command execution during shell startup, bypassing the system.run allowlist. Introduces `sanitizeSystemRunEnvOverrides()` with `HOST_SHELL_WRAPPER_ALLOWED_OVERRIDE_ENV_KEYS` (strict allowlist: TERM, LANG, LC_ALL, LC_CTYPE, LC_MESSAGES, COLORTERM, NO_COLOR, FORCE_COLOR) to filter env overrides when shell wrappers are active. Applied in `handleSystemRunInvoke()` before general env sanitization. Includes macOS Swift parity (`HostEnvSanitizer.swift`). Directly addresses Audit 2 Claim 8 and further hardens the gateway-side env blocklist (Legitimate Gap 1, already closed in PR #12).
