#### Generalize tool schema compat cleanup ([`b236f3910`](https://github.com/openclaw/openclaw/commit/b236f3910))

Generalizes tool schema compatibility handling from XAI-specific to multi-provider support. Introduces `resolveUnsupportedToolSchemaKeywords()` (maps provider to unsupported keywords) and `stripUnsupportedSchemaKeywords()` (removes them from tool parameter schemas). Previously, only XAI provider had schema keyword stripping; this extends the protection to all providers. New test file `src/agents/pi-tools.schema.test.ts` validates the generalized logic.

**Audit 2 Claim 7 (Shell injection via incomplete regex):** The original claim alleged that regex-based schema cleanup was incomplete. This commit replaces the narrow XAI-specific approach with a generalized, provider-aware keyword resolution system — a defense-in-depth improvement that addresses the underlying concern more robustly. The claim remains unexploitable (schemas are internal, not attacker-controlled), but the refactor strengthens the control.

#### Support synthetic chat origins ([`b12f3ce6e`](https://github.com/openclaw/openclaw/commit/b12f3ce6e))

Adds `ChatSendExplicitOrigin` type with `originatingChannel`, `originatingTo`, `accountId`, and `messageThreadId` fields to the gateway chat path. Synthetic chat messages (e.g., from admin clients or cron-triggered agents) now carry structured origin metadata with mandatory field validation when using synthetic routes. Improves auditability and prevents potential message attribution confusion. New test file `src/gateway/server-methods/chat.directive-tags.test.ts` covers the validation logic.
