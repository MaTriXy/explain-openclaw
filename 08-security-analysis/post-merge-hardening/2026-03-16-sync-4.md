#### Add firecrawl onboarding search plugin ([`aa28d1c71`](https://github.com/openclaw/openclaw/commit/aa28d1c71))

A new firecrawl extension under `extensions/firecrawl/` adds third-party web scraping and search capabilities via the Firecrawl API. The core client lives in `firecrawl-client.ts`; `firecrawl-scrape-tool.ts` and `firecrawl-search-provider.ts` surface scraping and search tools to agents. In `src/agents/tools/web-fetch-utils.ts`, the new `extractBasicHtmlContent()` helper is factored out of `extractReadableContent()` to provide a simpler HTML extraction path used when Firecrawl is unavailable.

Audit 2 Claim 4 notes that web-fetch extensions can introduce SSRF paths. Firecrawl URL inputs pass through the existing SSRF guard in `src/infra/net/ssrf.ts`; the extension does not register any bypass. The Firecrawl API key is stored via the standard web runtime secret prefix mechanism (`tools.web.fetch.firecrawl.`) defined in `src/cli/command-secret-gateway.ts`.

#### Add remote openshell sandbox mode ([`ae7f18e50`](https://github.com/openclaw/openclaw/commit/ae7f18e50))

Extends the OpenShell sandbox to support a remote execution mode. The new `extensions/openshell/src/remote-fs-bridge.ts` module provides `createOpenShellRemoteFsBridge()`, which implements `SandboxFsBridge` by synchronizing files between the local agent workspace and a remote OpenShell instance over SSH. `src/agents/apply-patch.ts` and `src/agents/sandbox-media-paths.ts` are updated to accommodate remote path contexts. Isolation enforcement is delegated to the remote OpenShell host; the local sandbox policy does not govern the remote environment.

#### Land plugin auth and startup cleanup ([`8ab01c5c9`](https://github.com/openclaw/openclaw/commit/8ab01c5c9))

Adds a `providerAuthEnvVars` field to `openclaw.plugin.json` manifests across all provider extensions (e.g., `extensions/github-copilot/openclaw.plugin.json` now declares `["COPILOT_GITHUB_TOKEN", "GH_TOKEN", "GITHUB_TOKEN"]`). This field enumerates which environment variables supply authentication tokens for each provider, making the auth credential surface explicit and auditable from the manifest rather than requiring source inspection.

Audit 1 Claim 3 identified hardcoded OAuth secret handling in plugin code; centralization into manifest metadata reduces undeclared auth sources and enables static audit tooling to enumerate all auth credential paths.

#### Status: lazy-load read-only account inspectors ([`f71f44576`](https://github.com/openclaw/openclaw/commit/f71f44576))

Moves account inspector initialization for the `status` command into lazy-load paths to reduce startup overhead. Inspectors remain read-only. No security-relevant behavior changes.

#### Add openshell sandbox backend ([`d8b927ee6`](https://github.com/openclaw/openclaw/commit/d8b927ee6))

Introduces the core OpenShell sandbox backend as a new `extensions/openshell/` package. `createOpenShellSandboxBackendFactory()` builds a `SandboxBackendFactory` implementation registered with the agent sandbox subsystem; `createOpenShellSandboxBackendManager()` manages backend lifecycle. Filesystem access uses `createOpenShellFsBridge()` (local mode) or `createOpenShellRemoteFsBridge()` (remote mode, added in `ae7f18e50`). Remote SSH sessions are established via `createOpenShellSshSession()`. Temporary directory handling delegates to `resolvePreferredOpenClawTmpDir()`.

Audit 2 Claim 3 covers log traversal in sandbox artifact paths. The OpenShell backend exposes new artifact access paths scoped to the remote workspace configured in `config.ts`; path resolution uses the plugin config rather than user-supplied input directly. The gap remains open but this backend represents a new surface to track.

#### Harden read-only SecretRef command paths and diagnostics ([`a2cb81199`](https://github.com/openclaw/openclaw/commit/a2cb81199))

Renames `CommandSecretResolutionMode` from opaque legacy strings (`"strict"`, `"summary"`, `"operational_readonly"`) to explicit enum values: `"enforce_resolved"`, `"read_only_status"`, and `"read_only_operational"`. `normalizeCommandSecretResolutionMode()` provides backward-compatible mapping. `enforcesResolvedSecrets()` replaces all `mode === "strict"` comparisons throughout `src/cli/command-secret-gateway.ts`, eliminating string drift risk.

Status commands (`status.gather.ts`) now preserve `SecretRef` configuredness across probe calls even in degrade-safe modes. `read_only_status` and `read_only_operational` paths use `enforcesResolvedSecrets()` to skip resolution-failure error paths that could expose credential state. Docs (`docs/cli/security.md`, `docs/cli/status.md`, `docs/cli/daemon.md`) are aligned with the new probe semantics.

This commit directly addresses Audit 1 Claims 1, 2, 5, and 6. Cross-reference with `8ab01c5c9` for Claim 3 (auth manifest centralization).
