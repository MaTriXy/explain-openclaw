#### Reuse shared secret comparison helper ([`be10ecef7`](https://github.com/openclaw/openclaw/commit/be10ecef7))

Refactors webhook authentication across six extensions â€” BlueBubbles, Feishu, Mattermost, Telegram, Voice Call, and Zalo â€” to use the shared `safeEqualSecret()` helper from `openclaw/plugin-sdk/browser-support` instead of per-extension inline comparisons. Previously, each extension implemented its own constant-time comparison with different buffer allocation patterns, creating subtle timing variance windows between implementations. The centralized helper eliminates this fragmentation and ensures uniform constant-time comparison behavior across all webhook handlers.

**Addresses Audit Claim:** Audit 1 Claim 7 (Webhook signature bypass) â€” standardizes timing-safe comparison behavior across all webhook handlers. Also closes [GHSA-jj6q-rrrf-h66h](../official-security-advisories.md#ghsa-jj6q-rrrf-h66h-shared-secret-comparison-call-sites-leaked-length-through-timing).

#### Browser: normalize localhost absolute-form CDP hosts ([`9c22d6366`](https://github.com/openclaw/openclaw/commit/9c22d6366))

Adds `isLoopbackHost()` at `src/gateway/net.ts:340` to normalize localhost absolute-form hostnames (e.g., `localhost.`, `LOCALHOST.`, multi-label variants) before socket connection validation in the browser extension's CDP (Chrome DevTools Protocol) infrastructure. Without this normalization, DNS rebinding attacks could redirect CDP WebSocket traffic to an attacker-controlled endpoint by using trailing-dot or mixed-case hostname forms that bypass the original string-equality check. The fix ensures all loopback-equivalent hostname formats are recognized and treated as local-only, closing the rebinding vector for the CDP connection path.

**Addresses Audit Claim:** Audit 2 Claim 4 (DNS rebinding SSRF via web-fetch) â€” tightens host validation for CDP WebSocket connections.

#### Add web fetch provider boundary ([`38d2faee2`](https://github.com/openclaw/openclaw/commit/38d2faee2))

Large refactor (72 files, 3,417 insertions) establishing a plugin-owned web fetch provider infrastructure. Key security hardening:

- **Host allowlist:** `ALLOWED_FIRECRAWL_HOSTS` at `extensions/firecrawl/src/firecrawl-client.ts:33` restricts Firecrawl API calls to `api.firecrawl.dev` only. Calls to any other host are rejected before network access.
- **Strict endpoint enforcement:** `withStrictWebToolsEndpoint()` in `src/plugin-sdk/provider-web-fetch.ts` validates URLs against the allowed host set before executing web fetch requests.
- **Credential stripping:** Firecrawl base URLs are sanitized to remove username, password, query parameters, and hash fragments before use, preventing credential leakage through configured URLs.
- **Provider auto-detection:** `resolveWebFetchDefinition()` in `src/agents/tools/web-fetch.ts` resolves web fetch provider credentials with env var fallback and auto-detection ordering, replacing the previous direct core config reads.
- **Config migration:** Firecrawl config moves from `tools.web.fetch.firecrawl.*` to the plugin-owned path `plugins.entries.firecrawl.config.webFetch.*`. The `doctor` command bridges legacy config at startup.

**Addresses Audit Claims:** Audit 2 Claim 4 (DNS rebinding SSRF â€” allowlist + host normalization), provider boundary enforcement (no direct core calls to provider internals).
