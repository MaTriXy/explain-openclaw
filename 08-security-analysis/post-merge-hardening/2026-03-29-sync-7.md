#### Fix HTTP OpenAI-compatible routes missing operator.write scope checks ([`703e68a74`](https://github.com/openclaw/openclaw/commit/703e68a74))

Added missing `operator.write` scope validation to HTTP OpenAI-compatible routes. The `handleGatewayPostJsonEndpoint` helper in `src/gateway/http-endpoint-helpers.ts` now accepts a `requiredOperatorMethod` parameter and calls `authorizeOperatorScopesForMethod` to verify the caller has the required operator scope before processing the request. Previously, these routes accepted any authenticated bearer token without checking specific operator permissions, allowing unauthorized write operations.

#### Audit web search keys for all bundled providers ([`31112d598`](https://github.com/openclaw/openclaw/commit/31112d598))

Replaced hardcoded web search key check with a provider-aware audit. The new `hasBundledWebSearchCredential` function in `src/plugins/bundled-web-search-registry.ts` uses `resolveBundledPluginWebSearchProviders` to discover all bundled web search providers and checks each for configured credentials. This ensures the security audit in `src/security/audit-extra.sync.ts` covers xAI/Grok, Gemini, Kimi, Moonshot, and OpenRouter credentials — not just the previously-hardcoded OpenAI key check.
