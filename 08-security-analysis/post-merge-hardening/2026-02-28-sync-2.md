### Feishu Sender Allowlist Global Fallback ([`107be4e90`](https://github.com/openclaw/openclaw/commit/107be4e90))

The Feishu channel now supports a global `groupSenderAllowFrom` configuration field as a fallback when no per-group `allowFrom` is set. Previously, sender-level allowlist filtering in `handleFeishuMessage()` only applied per-group settings; with no per-group config, any sender could reach the bot. The updated logic uses `perGroupSenderAllowFrom.length > 0 ? perGroupSenderAllowFrom : globalSenderAllowFrom`, meaning operators can set a single global sender allowlist that applies to all Feishu groups without repeating it per group. Access control enforcement is performed by `isFeishuGroupAllowed()`.

### Feishu Outbound Local Image Path Auto-Conversion ([`0e4c24ebe`](https://github.com/openclaw/openclaw/commit/0e4c24ebe))

A new function `normalizePossibleLocalImagePath()` in `extensions/feishu/src/outbound.ts` is invoked before sending text output to Feishu. When the text payload is a whitespace-free absolute filesystem path with an image extension (`.jpg`, `.png`, `.gif`, `.webp`, etc.) and the file exists, the system auto-uploads it as a Feishu image message instead of sending the raw path string. The function guards against data/http URL schemes and non-image extensions. A race condition between `fs.existsSync()` and `fs.statSync()` is handled via try-catch. This prevents LLM-generated image paths from leaking as plaintext in outbound messages; however, it also means that LLM output containing an absolute local image path triggers a filesystem read and upload to Feishu.

### Feishu Reaction Notification Mode Gating ([`aef535510`](https://github.com/openclaw/openclaw/commit/aef535510))

A new `reactionNotifications` config field (values: `"off"`, `"own"`, `"all"`) was added to `extensions/feishu/src/config-schema.ts` to gate the reaction event ingress introduced by `8241145ad`. Without this gating, any Feishu reaction event (emoji add/remove) would be ingested as a synthetic message event. The `"own"` mode restricts ingestion to reactions on the bot's own messages; `"all"` allows any reaction; `"off"` disables reaction-based event ingestion entirely. This mode check is evaluated in `resolveReactionSyntheticEvent()` in `extensions/feishu/src/monitor.ts` before constructing the synthetic event.

### Feishu Reaction Event Ingress ([`8241145ad`](https://github.com/openclaw/openclaw/commit/8241145ad))

This commit adds support for Feishu emoji reaction events (create/delete) as a new inbound event type. The `resolveReactionSyntheticEvent()` function in `extensions/feishu/src/monitor.ts` validates the event, fetches the original message via `getMessageFeishu()`, applies a 1,500ms timeout via `withTimeout()`, and converts the reaction into a synthetic `FeishuMessageEvent` that passes through the standard message processing pipeline (including access group checks). The `* as crypto` import provides a UUID generator for synthetic event deduplication. This new event ingress path is gated by the `reactionNotifications` config from `aef535510`; without that gating, this commit would create an ungated inbound event source.

### Feishu merge_forward Message Parsing ([`f53ef73a2`](https://github.com/openclaw/openclaw/commit/f53ef73a2))

The Feishu bot now handles `merge_forward` message type, which is a Feishu "merged and forwarded" message bundle. A new `parseMergeForwardContent()` function in `extensions/feishu/src/bot.ts` fetches up to 50 sub-messages from the Feishu API, parses the JSON response, filters to sub-messages via `upper_message_id`, sorts by `create_time`, and formats each sub-message using `formatSubMessageContent()`. The content is ingested as-is into the bot's message pipeline without additional sanitization beyond type-based formatting. This increases the external content attack surface: a Feishu user can send a merged message bundle containing up to 50 sub-messages with arbitrary content, all of which will be passed to the AI agent.
