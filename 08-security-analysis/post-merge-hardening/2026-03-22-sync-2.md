#### Agents: add per-agent defaults and safe model fallback ([`c96a12aeb9`](https://github.com/openclaw/openclaw/commit/c96a12aeb9))

The agent runtime schema now supports three new per-agent configuration fields: `thinkingDefault` (accepting values off/minimal/low/medium/high/xhigh/adaptive), `reasoningDefault` (on/off/stream), and `fastModeDefault` (boolean). These allow each `AgentEntrySchema` entry to specify thinking and fast-mode behavior independently from global defaults. The safe model fallback path ensures agents whose configured model is unavailable are routed to a known-safe alternative rather than failing. No authentication or sandbox surface is affected — this is a configuration enhancement only.

#### Split telegram audit runtime seams ([`8b7f40580d`](https://github.com/openclaw/openclaw/commit/8b7f40580d))

The monolithic `src/security/audit-channel.runtime.ts` is decomposed into per-platform lazy-load boundaries: `audit-channel.allow-from.runtime.ts`, `audit-channel.discord.runtime.ts`, `audit-channel.telegram.runtime.ts`, and `audit-channel.zalouser.runtime.ts`. Each new file re-exports only the audit channel type needed by its platform, following the repo's dynamic import guardrail that prohibits mixing `await import('x')` and static `import ... from 'x'` for the same module in production code paths. The refactor reduces cold-startup overhead for channels that do not need all audit channel implementations at load time; the audit channel access-control logic itself is unchanged.

#### Remove relative extension boundary escapes ([`2b4c3c2057`](https://github.com/openclaw/openclaw/commit/2b4c3c2057))

Extension `runtime-api.ts` files in 43 production files previously used relative paths (e.g., `../../src/plugin-sdk/...`) to reach core SDK code — bypassing the intended plugin SDK import boundary. This commit replaces all such relative imports with the canonical `openclaw/plugin-sdk/*` subpath. Extensions can no longer reach internal `src/**` paths directly and must use the public SDK surface. A host-version gate ensures extensions that reference newly exposed SDK subpaths only activate on gateway hosts that provide those subpaths. This closes the import-boundary escape path documented in the plugin SDK extension isolation policy.

#### Android: escape SQL LIKE wildcards in contacts search query ([`f7866c1c15`](https://github.com/openclaw/openclaw/commit/f7866c1c15))

Android contacts search in `ContactsHandler` previously passed raw user-supplied input directly into a SQL LIKE pattern without escaping `%` and `_` wildcard characters. An attacker or application providing these characters could cause the query to match a wider result set than intended. The new `escapeLikePattern()` helper escapes backslash, percent, and underscore before the string is used in the LIKE clause, and the selection clause adds `ESCAPE '\\'` so SQLite treats them as literals. This closes the SQL LIKE wildcard injection path in the contacts search query. This commit was incorrectly demoted to SAFE during triage (false negative) and confirmed security-relevant by Tier B Cataloger escalation.

#### Android: fix incomplete JS string escaping in A2UI action status ([`e7814f7ba0`](https://github.com/openclaw/openclaw/commit/e7814f7ba0))

The `jsDispatchA2UIActionStatus()` function in `OpenClawCanvasA2UIAction` previously escaped only backslash and double-quote before embedding action IDs and error strings in JavaScript dispatched to a WebView. Unescaped newlines, tabs, and Unicode line terminators U+2028 (LINE SEPARATOR) and U+2029 (PARAGRAPH SEPARATOR) could break the JavaScript string literal boundary and allow content injection into the dispatched `openclaw:a2ui-action-status` CustomEvent. The fix introduces `jsonStringLiteral()`, a private helper that serializes strings through `JsonPrimitive` (Gson) for correct JSON escaping, then additionally escapes U+2028 and U+2029 (which JSON permits in strings but JavaScript engines treat as line terminators). This commit was also incorrectly demoted to SAFE during triage and confirmed security-relevant by escalation.

#### Make permission rationale completion single-shot ([`b67baae1f6`](https://github.com/openclaw/openclaw/commit/b67baae1f6))

The rationale dialog completion handler in `PermissionRequester` could previously call `cont.resume()` more than once — from a button press, the `ON_DESTROY` lifecycle observer, or the cancellation handler — causing an `IllegalStateException` for double-resume on a suspended coroutine continuation. An `AtomicBoolean` guard named `finished` wraps all resume paths in a single `finish()` helper that uses `compareAndSet(false, true)` to ensure only the first invocation completes the continuation. All subsequent ON_DESTROY or cancellation events are silently ignored.

#### Android: harden permission dialogs across activity teardown ([`518d2dd6a9`](https://github.com/openclaw/openclaw/commit/518d2dd6a9))

The `showRationaleDialog()` function in `PermissionRequester` previously created an `AlertDialog` without checking whether the host activity was already finishing or destroyed, which can cause `WindowManager$BadTokenException` when a dialog is attached to a dying window. A guard checks `activity.isFinishing || activity.isDestroyed` at function entry and returns `false` immediately if the activity cannot host dialogs. A `LifecycleEventObserver` is attached to the activity lifecycle and dismisses the dialog, resuming the coroutine with `false` on `ON_DESTROY`. Cancellation handling explicitly removes the observer and dismisses the dialog to prevent resource leaks on coroutine cancellation.
