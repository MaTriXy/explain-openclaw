#### Honor browser profile from request body for node proxy calls ([`fa875a6bf`](https://github.com/openclaw/openclaw/commit/fa875a6bf))

The gateway `browser.request` handler previously resolved the browser profile exclusively from `query.profile`. When a caller passed the desired profile in the POST body instead of the query string, the value was silently discarded and the node fell back to the default profile — which could silently activate Chrome extension mode even when the caller had explicitly requested the `openclaw` profile.

New helper `resolveRequestedProfile()` in `src/gateway/server-methods/browser.ts` checks `query.profile` first and falls back to `body.profile` when present. The existing `browser.proxy` invocation now receives the correctly resolved value.

This tightens the profile selection path, reducing the surface where an agent could inadvertently (or deliberately) be placed in an unintended browser profile without explicit instruction.

#### Harden spoofed system marker handling ([`5b8f492a4`](https://github.com/openclaw/openclaw/commit/5b8f492a4))

New function `sanitizeInboundSystemTags()` in `src/auto-reply/reply/inbound-text.ts` neutralizes user-controlled strings that mimic internal system markers before they reach the agent context.

Two patterns are targeted:
- **Bracketed tags:** `BRACKETED_SYSTEM_TAG_RE` matches `[System Message]`, `[System]`, `[Assistant]`, and `[Internal]` (case-insensitive, tolerates whitespace). Matched brackets are replaced with parentheses so the literal content is preserved but the marker structure is defused.
- **Line-leading `System:` prefix:** `LINE_SYSTEM_PREFIX_RE` matches a `System:` prefix at the start of a line. It is rewritten to `System (untrusted):` so the origin is clearly flagged in downstream context.

`sanitizeInboundSystemTags()` is applied in `src/auto-reply/reply/inbound-context.ts` to all inbound content fields — including `Body`, `BodyForAgent`, and `BodyForCommands` — before they are assembled into the reply run context. The companion `session-updates.ts` change replaces the raw `System:` prefix in runtime event formatting with a `## Runtime System Events` header, removing a second injection surface.
