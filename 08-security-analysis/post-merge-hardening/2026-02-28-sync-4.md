#### Feishu oc\_ Session Routing Correctness ([`6a8d83b6d`](https://github.com/openclaw/openclaw/commit/6a8d83b6d))

The `resolveFeishuSession` function in `src/infra/outbound/outbound-session.ts` previously assumed that any Feishu `oc_` chat ID always represented a group conversation, hard-coding `isGroup = true` for that prefix. This caused DM messages with `oc_` chat IDs to be incorrectly routed as group sessions, breaking session isolation for affected users. The fix introduces a `typeExplicit` boolean flag: `isGroup` and `typeExplicit` default to false; explicit `group:`/`chat:` or `user:`/`dm:` prefixes set both correctly; and `oc_` IDs that arrive without an explicit prefix are now left as their default (`isGroup = false`, i.e., direct/DM). Only `ou_`/`on_` prefixes remain reliably auto-classified as DMs.
