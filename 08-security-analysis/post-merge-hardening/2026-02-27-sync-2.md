#### Secrets: add migrate rollback and skill ref support ([`f6a854bd3`](https://github.com/openclaw/openclaw/commit/f6a854bd3))

Implements the secrets migration engine in `src/secrets/migrate.ts` via `runSecretsMigration` and `rollbackSecretsMigration` functions. The migration engine scans config files and auth stores for plaintext API keys and tokens, converts them to SecretRef pointers, and persists encrypted payloads via sops. A backup manifest records the pre-migration file state to enable safe rollback. The CLI command defaults to dry-run mode; `--write` activates permanent migration. `SkillConfig.apiKey` now accepts SecretRef in addition to plaintext strings, enabling skills to read secrets from the encrypted store. This addresses Audit 1 Claim 1 (plaintext credential storage) by providing a migration path to eliminate plaintext from config and auth-profiles files.

#### Gateway: add manual secrets reload command ([`fe5670002`](https://github.com/openclaw/openclaw/commit/fe5670002))

Adds `createSecretsHandlers` in `src/gateway/server-methods/secrets.ts`, wiring a `secrets.reload` RPC method into the gateway via the extraHandlers gateway parameter. The handler calls `activateRuntimeSecrets` (defined in `src/gateway/server.impl.ts`) to re-resolve all active SecretRef pointers in the running snapshot. This enables zero-downtime key rotation: operators rotate the backing env var or sops-encrypted file, then call `openclaw secrets reload` without restarting the gateway. Also exposes the `secrets.reload` CLI command through `src/cli/secrets-cli.ts`.

#### Config: add secret ref schema and redaction foundations ([`c3a4251a6`](https://github.com/openclaw/openclaw/commit/c3a4251a6))

Creates `src/config/types.secrets.ts` defining the `SecretRef` type (`{ source: "env" | "file", id: string }`) and `SecretsConfig` for declaring secrets provider sources. Adds `SecretRefSchema` with source-discriminated Zod validation in `src/config/zod-schema.core.ts`. Extends `redactConfigSnapshot` in `src/config/redact-snapshot.ts` to strip SecretRef objects and whole sensitive objects (e.g., Google Chat service account JSON) from logged config snapshots, preventing reference identifiers from appearing in logs. Sets the schema and redaction foundation for all secrets-as-code work in this sync.

#### Config: enforce source-specific SecretRef id validation ([`d00ed7302`](https://github.com/openclaw/openclaw/commit/d00ed7302))

Tightens `SecretRefSchema` using Zod's `discriminatedUnion`. Env-source refs must have IDs matching `/^[A-Z][A-Z0-9_]{0,127}$/` (uppercase env var name format). File-source refs must have IDs matching `/^\//` (absolute JSON Pointer per RFC 6901), validated via `isValidFileSecretRefId` in `src/config/zod-schema.core.ts`. The discriminated union prevents id format confusion across source types and rejects invalid configurations at parse time rather than silently accepting them.

#### Auth profiles: never persist plaintext when refs are present ([`e1301c31e`](https://github.com/openclaw/openclaw/commit/e1301c31e))

Modifies `saveAuthProfileStore` in `src/agents/auth-profiles/store.ts` to strip plaintext credential values before writing to disk when a corresponding ref pointer is present. For `api_key` credentials: if `credential.keyRef` exists and `credential.key` is set, the `key` field is deleted from the serialized profile. For `token` credentials: if `credential.tokenRef` exists and `credential.token` is set, the `token` field is deleted. This prevents plaintext credential re-introduction to auth-profiles files after the secrets migration runs, addressing Audit 1 Claim 1 directly.

#### Secrets: make runtime activation auth loads read-only ([`8e33ebe47`](https://github.com/openclaw/openclaw/commit/8e33ebe47))

Modifies the secrets runtime activation path to load auth profile stores in read-only mode, preventing accidental write-back of in-memory auth state to disk during secret activation. Modifies `src/secrets/runtime.ts` and `src/agents/auth-profiles/store.ts` to pass read-only flags during activation. Also extracts shared secret helpers to `src/secrets/shared.ts` and applies read-only access in `src/secrets/sops.ts`. Together with `e1301c31e`, this closes the write-back path that could re-introduce plaintext during runtime operations.

#### Gemini OAuth: resolve npm global shim install layouts ([`6daf40d3f`](https://github.com/openclaw/openclaw/commit/6daf40d3f))

Adds `resolveGeminiCliDirs` function in `extensions/google-gemini-cli-auth/oauth.ts` to generate candidate directories for both standard npm global installs and npm-global-shim wrapper layouts. `extractGeminiCliCredentials` now iterates all candidate paths, falling back to `findFile` for recursive search when standard paths yield nothing. This fixes credential discovery failures when the Gemini CLI binary is installed via npm global shim wrappers (a common setup on macOS with managed Node installations).

#### Telegram: improve webhook config guidance and startup fallback ([`840b768d9`](https://github.com/openclaw/openclaw/commit/840b768d9))

Adds detailed description fields to Zod schema entries for webhookUrl, webhookSecret, webhookPath, webhookHost, and webhookPort in `src/telegram/zod-schema.providers-core.ts`. Startup logging in `src/telegram/webhook.ts` now separately displays the local listener address and the public advertised webhook URL, clarifying the distinction for operators running behind reverse proxies. Helps operators correctly configure `webhookSecret` (HMAC verification) and avoid misconfigured webhook setups.

#### Unify v3 auth payload builders and vectors ([`081b1aa1e`](https://github.com/openclaw/openclaw/commit/081b1aa1e))

Introduces `toLowerAscii` and `normalizeDeviceMetadataForAuth` functions in `src/gateway/device-auth.ts` to ensure deterministic ASCII-only lowercase normalization of `platform` and `deviceFamily` fields across TypeScript, Swift, and Kotlin implementations. These values are included in device signature payloads verified by verifyDeviceSignature; without explicit ASCII normalization, locale-dependent toLowerCase() behavior could produce cross-platform payload mismatches. Also exports `normalizeDeviceMetadataForAuth` for use in `src/gateway/message-handler.ts` and adds `buildDeviceAuthPayloadV3` for the unified payload builder path.

#### decodeURIComponent guardrails in browser relay ([`79659b2b1`](https://github.com/openclaw/openclaw/commit/79659b2b1))

Guards malformed percent-encoded URL segments in `src/browser/extension-relay.ts` and `src/browser/routes/dispatcher.ts`. Previously, `decodeURIComponent(match[1] ?? "")` would throw a `URIError` on invalid percent-encoding sequences (e.g., `%zz`), causing an unhandled exception. The fix wraps these calls in try/catch, returning HTTP 400 with a descriptive error on decode failure. This closes a denial-of-service vector via malformed URL injection into the relay routing layer.

#### Query-token auth for /json relay routes ([`42cf32c38`](https://github.com/openclaw/openclaw/commit/42cf32c38))

Aligns HTTP `/json` relay endpoint authentication with WebSocket authentication by replacing direct header extraction (`getHeader(req, RELAY_AUTH_HEADER)`) with `getRelayAuthTokenFromRequest(req, url)` in `src/browser/extension-relay.ts`. The updated helper checks both the `X-Relay-Auth` header and a query-parameter token, closing an auth gap where `/json` endpoints accepted only header-based tokens while WebSocket endpoints accepted query-param tokens. Prevents authentication bypass via transport-format mismatch.

#### Extension relay CORS fix ([`4c75eca58`](https://github.com/openclaw/openclaw/commit/4c75eca58))

Adds explicit CORS enforcement to `src/browser/extension-relay.ts`. Extension-origin requests (`origin.startsWith("chrome-extension://")`) receive `Access-Control-Allow-Origin: <origin>` with `Vary: Origin`. CORS preflight (`OPTIONS`) requests from non-extension origins receive HTTP 403 Forbidden, while extension-origin preflight requests receive a proper 24-hour CORS response including `Access-Control-Allow-Methods: GET, PUT, POST, OPTIONS` and `Access-Control-Allow-Headers`. This prevents non-extension web pages from making cross-origin requests to the relay server.
