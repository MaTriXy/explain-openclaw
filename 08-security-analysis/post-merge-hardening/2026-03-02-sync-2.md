#### ACP: carry dedupe/projector updates onto configurable acpx branch ([`2466a9bb1`](https://github.com/openclaw/openclaw/commit/2466a9bb1))

Ports the ACP deduplication and projector feature set onto the configurable ACPX branch. Key addition: `ACP_TAG_VISIBILITY_DEFAULTS` (exported from `acp-stream-settings.ts`), a record mapping each `AcpSessionUpdateTag` to its default visibility — agent message chunks are visible by default; tool calls, usage updates, available-commands updates, config-option updates, and session-info updates are hidden. The `AcpProjectionSettings` type gains a `tagVisibility` field (`Partial<Record<AcpSessionUpdateTag, boolean>>`) that allows per-tag overrides via the `acp.stream.tagVisibility` config key. The `isAcpTagVisible()` helper resolves effective visibility by checking per-session overrides before falling back to `ACP_TAG_VISIBILITY_DEFAULTS`. Security relevance: sensitive internal events (tool calls, usage data, command availability) are hidden from streaming output channels by default; an operator must explicitly configure overrides to expose them.

#### ACP: simplify stream config to repeatSuppression ([`4e2efaf65`](https://github.com/openclaw/openclaw/commit/4e2efaf65))

Replaces the intermediate `metaMode` configuration with a single `repeatSuppression` boolean. `DEFAULT_ACP_REPEAT_SUPPRESSION = true` — repeat suppression is enabled by default, deduplicating consecutive identical ACP session updates (e.g., repeated status messages) to reduce redundant content in streaming output. The field is resolved from `acp.stream.repeatSuppression` config via `clampBoolean()`. Removing the multi-value `AcpMetaMode` type reduces the configuration surface and operator misconfiguration risk.

#### ACP: add hidden-boundary separator for hidden tool events ([`c8b958e57`](https://github.com/openclaw/openclaw/commit/c8b958e57))

Adds configurable visual separation between hidden tool-event boundaries and the next visible content. The `AcpHiddenBoundarySeparator` type (`"none" | "space" | "newline" | "paragraph"`) and default constants (`DEFAULT_ACP_HIDDEN_BOUNDARY_SEPARATOR = "paragraph"` for final-only mode; `"space"` for live mode) are introduced in `acp-stream-settings.ts`. A `HIDDEN_BOUNDARY_TAGS` set in `acp-projector.ts` marks `tool_call` and `tool_call_update` as boundary-creating events. The `resolveHiddenBoundarySeparatorText()` function translates the separator enum to actual whitespace. Security relevance: explicit boundary separators prevent hidden tool events from inadvertently concatenating adjacent visible text in ways that could produce confusing output for downstream parsers.

#### Enforce auth for abort triggers and models ([`3a93a7bb1`](https://github.com/openclaw/openclaw/commit/3a93a7bb1))

Applies `rejectUnauthorizedCommand()` (from `src/auto-reply/reply/command-gates.ts`) to three command handlers that previously used inconsistent or absent authorization:

- `handleModelsCommand()` in `src/auto-reply/reply/commands-models.ts` (line 363): previously had no authorization check; now guards model selection with `rejectUnauthorizedCommand(params, "/models")`.
- `handleStopCommand()` in `src/auto-reply/reply/commands-session-abort.ts` (line 89): replaces a direct isAuthorizedSender check that returned `{ shouldContinue: false }` silently; now sends a consistent rejection response via `rejectUnauthorizedCommand(params, "/stop")`.
- `handleAbortTrigger()` in `src/auto-reply/reply/commands-session-abort.ts` (line 146): previously had no authorization check; now guards abort triggers with `rejectUnauthorizedCommand(params, "abort trigger")`.

The centralized function ensures unauthorized senders receive a visible rejection rather than a silent drop, and extends coverage to commands that were previously ungated. Reinforces Audit 2 Claim 5 (insufficient RBAC for auto-reply commands).

#### Add SSRF policy for media downloads in proxy environments ([`39a45121d`](https://github.com/openclaw/openclaw/commit/39a45121d))

Adds `DISCORD_MEDIA_SSRF_POLICY` (lines 9-12 in `src/discord/monitor/message-utils.ts`) and a matching constant in `src/slack/monitor/media.ts`. Both define `allowedHostnames` allowlists (Discord CDN: `cdn.discordapp.com`, `media.discordapp.net`; Slack: `*.slack.com`, `*.slack-edge.com`, `*.slack-files.com`) and set `allowRfc2544BenchmarkRange: true` to tolerate fake-IP addresses emitted by transparent proxy environments (e.g., Clash/Shadowrocket in fake-IP mode, which return IPs in the 198.18.0.0/15 RFC 2544 benchmark range that the SSRF guard would otherwise block). The policies are passed as the `ssrfPolicy` parameter to `fetchRemoteMedia()` calls when downloading Discord attachments and stickers, and Slack file attachments — matching the pattern already applied to Telegram. Reinforces Audit 2 Claim 4 (SSRF via media fetch) for Discord and Slack channels.

#### Fail closed browser auth bootstrap ([`e4d22fb07`](https://github.com/openclaw/openclaw/commit/e4d22fb07))

`startBrowserControlServerFromConfig()` in `src/browser/server.ts` now fails closed when auth initialization fails. A `browserAuthBootstrapFailed` flag (line 33) is set if `ensureBrowserControlAuth()` throws (lines 40-43). Lines 45-52 add a fail-closed guard: if the flag is set AND neither `browserAuth.token` nor `browserAuth.password` is available as a fallback, the function logs an error and returns `null` (browser control server does not start). Previously, a failed bootstrap would silently fall through, potentially starting the server with no authentication credentials. The accompanying new test file `src/browser/server.auth-fail-closed.test.ts` (92 lines) covers this path with explicit assertions.
