#### Unify the shared task run registry ([`53bcd5769`](https://github.com/openclaw/openclaw/commit/53bcd5769))

Replaced fragile string-based `source` and `bindingTargetKind` enums with direct `sourceId` tracking, preventing misclassification of task origins that could affect permission denial accounting.

- `tryCreateCronTaskRecord()` (`src/cron/service/timer.ts`) and `tryCreateManualTaskRecord()` (`src/cron/service/ops.ts`) wrap task creation in error handling, so ledger write failures are caught rather than silently corrupting task state.
- `normalizeTaskStatus()` and `resolveTaskTerminalOutcome()` (`src/tasks/task-registry.ts`) reconcile inconsistent status values (e.g. "accepted"→`queued`, "done"→`succeeded`) across runtimes, making permission-denial tracking more reliable.
- Removes legacy `TaskSource` and `TaskBindingTargetKind` enums that were prone to misclassification when determining which agent context initiated a task.

**Audit relevance:** Indirect improvement to Audit 1 Claim 5 (insufficient file permission checks) and Claim 6 (path traversal in agent dirs) — more reliable `sourceId` tracking improves observability of access control failures, though it does not directly fix traversal.

#### Restore queue targeting and plugin id prefixes ([`0e47ce58b`](https://github.com/openclaw/openclaw/commit/0e47ce58b))

Restores exact-match enforcement for approval ID resolution, preventing ambiguous prefix matching from leaking approvals to unintended agents.

- `manager.lookupPendingId()` (`src/gateway/exec-approval-manager.ts`) is now called in `createPluginApprovalHandlers()` (`src/gateway/server-methods/plugin-approval.ts`) before processing an approval. The result is checked for `kind === 'exact'` — `'ambiguous'` or `'none'` results return `APPROVAL_NOT_FOUND_DETAILS` (`src/gateway/server-methods/exec-approval.ts`), rejecting the request.
- `addExecApproval()` in `ui/src/ui/controllers/exec-approval.ts` changed from `push()` to `unshift()`, placing the newest approval at the front of the queue. This ensures consistent ordering when the UI presents pending approvals.
- Prevents a self-approval race condition where an agent could resolve another agent's pending approval if the IDs shared a common prefix.

**Audit relevance:** Directly addresses Audit 2 Claim 5 (self-approving agent, no RBAC) by restoring the lookup verification that enforces exact ID match semantics before approval resolution.

#### Recover clobbered config and isolate test paths ([`809833ef9`](https://github.com/openclaw/openclaw/commit/809833ef9))

Adds a new defense-in-depth layer for config integrity: the gateway now detects when the config file has been modified by an unexpected process and auto-restores from a trusted backup.

- `maybeRecoverSuspiciousConfigRead()` (async, `src/config/io.ts:937`) and `maybeRecoverSuspiciousConfigReadSync()` (sync, `src/config/io.ts:1067`) are called on config read paths. They compare the file's current hash fingerprint against the expected in-memory baseline — if the hash mismatches with no known write having occurred, the config is considered clobbered and restored from the most recent backup.
- Protects against external processes (or a rogue subagent) silently overwriting the gateway config file between the last known-good write and the next read.
- The sync variant is invoked at `src/config/io.ts:1705` and the async variant at `src/config/io.ts:1930`.

**Audit relevance:** New control — not addressed by prior audit claims. Provides tamper-detection and recovery for the config surface, which is the primary target for config injection attacks (Audit 2 Claim 1).

#### Port safer bundle MCP naming onto latest main (#49505) (thanks @ziomancer) ([`147c2c738`](https://github.com/openclaw/openclaw/commit/147c2c738))

Introduces sanitized, collision-resistant naming for MCP bundle tool registrations to prevent tool name injection and namespace confusion attacks.

- `sanitizeServerName()` (`src/agents/pi-bundle-mcp-names.ts:15`): strips non-alphanumeric characters (via `TOOL_NAME_SAFE_RE = /[^A-Za-z0-9_-]/g`) and appends a numeric suffix for uniqueness, limiting to `TOOL_NAME_MAX_PREFIX = 30` characters.
- `sanitizeToolName()` (`src/agents/pi-bundle-mcp-names.ts:28`): applies the same character sanitization to individual tool names.
- `buildSafeToolName()` (`src/agents/pi-bundle-mcp-names.ts:36`): combines sanitized server and tool names with a `__` separator, capping the result at `TOOL_NAME_MAX_TOTAL = 64` characters.
- Without this, a MCP server with a name containing special characters or a colon could register tool names that clash with other servers' tools, enabling one server to shadow or override another server's tool registrations.

**Audit relevance:** New control — prevents MCP tool namespace collision and injection. Part of a three-commit chain (`147c2c738` → `004bffa1c` → `14f78debd`).

#### Finalize safer MCP tool naming (#49505) (thanks @ziomancer) ([`004bffa1c`](https://github.com/openclaw/openclaw/commit/004bffa1c))

Propagates the `server__tool` namespaced format through the full tool-call normalization pipeline, ensuring that the safe names introduced in `147c2c738` are preserved end-to-end.

- Updates `attempt.tool-call-normalization.ts` and `attempt.ts` in `src/agents/pi-embedded-runner/run/` to pass tool names through normalization without stripping the `__` separator.
- Updates `compact.ts` and `google.ts` in `src/agents/pi-embedded-runner/` so that serialized transcript formats also preserve namespaced tool names.
- `session-transcript-repair.ts` is updated so that the repair logic correctly maps `server__toolname` call blocks to their result blocks.

**Audit relevance:** New control — completes the safer MCP naming chain. Ensures safe tool names survive round-trips through compact transcript storage and provider-specific serializers.

#### Allow colon in tool names and dispose transport on failed connect ([`14f78debd`](https://github.com/openclaw/openclaw/commit/14f78debd))

Fixes a silent tool-call drop bug and a transport resource leak, both introduced by the MCP `server:tool` namespacing format.

- `TOOL_CALL_NAME_RE` at `src/agents/session-transcript-repair.ts:5` changed from `/^[A-Za-z0-9_-]+$/` to `/^[A-Za-z0-9_:.-]+$/`. The old regex rejected tool names containing colons (e.g., `vigil-harbor:memory_status`), causing `repairToolUseResultPairing()` to silently drop valid tool-call blocks during session repair — breaking tool-call/result pairing for any MCP server using colon-style namespacing.
- Closes a resource leak: if `client.connect()` throws after the transport is instantiated, the transport is now explicitly closed rather than leaked. (Transport disposal was implemented as part of the broader MCP runtime split in `147c2c738`.)

**Audit relevance:** New control — the silent drop bug in `TOOL_CALL_NAME_RE` constituted an integrity failure where tool calls using the `server:tool` format were invisibly discarded, potentially silencing security-relevant tool responses.
