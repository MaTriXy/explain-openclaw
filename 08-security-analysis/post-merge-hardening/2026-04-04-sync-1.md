#### Add request transport overrides ([`23719dd51`](https://github.com/openclaw/openclaw/commit/23719dd51))

Introduces a typed HTTP transport override layer for media provider requests, enabling configuration of headers, authentication schemes, proxy routing, and TLS client certificates. The new `sanitizeConfiguredProviderRequest()` function in `src/agents/provider-request-config.ts` validates these overrides after secrets resolution, using `assertSecretInputResolved()` as a fail-closed guard that rejects any request configuration still containing unresolved `SecretRef` values. This prevents accidental credential leakage from partially-resolved secret chains before values are passed to provider runtime code. Shared and capability-specific overrides are merged via `mergeProviderRequestOverrides()`, and the type hierarchy (`ConfiguredProviderRequest`, `ConfiguredProviderRequestAuth`, `ConfiguredProviderRequestProxy`, `ConfiguredProviderRequestTls`) is defined in the new `src/config/types.provider-request.ts`. Zod schema validation (`ConfiguredProviderRequestTlsSchema`, `ConfiguredProviderRequestAuthSchema`, `ConfiguredProviderRequestProxySchema`) is added in `src/config/zod-schema.core.ts`.

#### Enrich session_end lifecycle hooks ([`2b28e7582`](https://github.com/openclaw/openclaw/commit/2b28e7582))

Enriches the session_end lifecycle hook payload with complete session teardown context. The commit renames `archiveSessionTranscriptsForSession()` to `archiveSessionTranscriptsForSessionDetailed()` to expose detailed archive entry metadata, and adds `emitGatewaySessionEndPluginHook()` in `src/gateway/server-methods/sessions.ts` to emit the hook with the full payload. Plugins registered for the session_end hook now receive the session ID, store path, agent ID, archived transcript list (with paths and deletion reason), and reason code. This gives plugins the information needed for session teardown auditing — for example, to detect unexpected session terminations, scrub sensitive data post-session, or record session lifecycle events in an audit log. Previously, plugins received minimal context and had to query session state independently to determine what happened during teardown.

#### Emit passive hooks for mention-skipped group messages ([`251fa7279`](https://github.com/openclaw/openclaw/commit/251fa7279))

Adds an `ingest` boolean configuration field to Telegram topic (`TelegramTopicSchema`), Telegram group (`TelegramGroupSchema`), and Signal group (`SignalGroupEntrySchema`) entry schemas in `src/config/zod-schema.providers-core.ts`. When `ingest` is set to `false` in a channel config, mention-skipped messages in that group or topic are suppressed at the hook layer — the passive `message:received` hook is not emitted. This prevents silent ingestion of messages that did not pass mention filters in multi-user group scenarios where the bot is present but not addressed. The corresponding implementation is updated in `extensions/telegram/src/bot-message-context.ts` and `extensions/signal/src/monitor/event-handler.ts` to honor the new flag before emitting internal hooks.
