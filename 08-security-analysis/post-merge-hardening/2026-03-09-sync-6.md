#### Gateway Foreground Action Queue with Idempotency ([`e806c479f`](https://github.com/openclaw/openclaw/commit/e806c479f))

The iOS gateway now maintains a bounded pending-action queue for foreground operations that arrive while the app is backgrounded. Two new gateway server methods — `node.pending.pull` and `node.pending.ack` — are added to `NODE_ROLE_METHODS`.

The queue enforces hard bounds: `NODE_PENDING_ACTION_TTL_MS` (10 minutes) evicts stale entries, and `NODE_PENDING_ACTION_MAX_PER_NODE` (64 entries) caps per-node depth. Each queued action carries an `idempotencyKey` to prevent duplicate replay on reconnect. The `NodePendingAckParamsSchema` requires `ids: NonEmptyString[]`, preventing acknowledgment of empty batches.

This addresses the iOS foreground-resume race condition where actions queued during backgrounding could be dropped or double-fired. The TTL and max-per-node caps prevent unbounded queue growth under adversarial or pathological reconnect patterns.

#### Cron Fire-and-Forget with Prompt Mode Isolation ([`38543d819`](https://github.com/openclaw/openclaw/commit/38543d819))

The cron trigger path changes from synchronous `context.cron.run(...)` to fire-and-forget `context.cron.enqueueRun(...)`. The cron announce delivery is consolidated into a dedicated handler. A new `CronRunResult` variant adds `{ ok: true; enqueued: true; runId: string }` alongside the existing success/failure shapes.

`resolvePromptModeForSession` now returns `"minimal"` for cron sessions (previously `"full"`), reducing token consumption and LLM tool-call surface for scheduled jobs. `isInternalAnnounceRequesterSession` is refactored to use `isCronSessionKey` for consistent cron session detection.

The isolation of cron announce delivery prevents timing-dependent announce failures when cron sessions overlap with gateway restarts. The minimal prompt mode for cron reduces the available tools and context exposed to cron-triggered agents.

#### CDP Relay Flow Hardening with Scoped Page Sessions ([`362248e55`](https://github.com/openclaw/openclaw/commit/362248e55))

The browser relay CDP flow is hardened via a new dedicated module `pw-session.page-cdp.ts`. The refactor separates page-level CDP session logic from the general session manager in `pw-session.ts`, and rewrites the invoke path in `invoke-browser.ts`.

Pattern hits include `crypto.` usage (for session IDs or nonces), `new URL(` construction for CDP WebSocket endpoint handling, and dynamic `import(` usage for lazy browser module loading. The CDP relay now scopes sessions per-page rather than sharing a singleton, limiting blast radius if a session is compromised.

#### Backup Verify Path Validation Hardening ([`09acbe652`](https://github.com/openclaw/openclaw/commit/09acbe652))

The backup verification path adds explicit input validation in `backup-verify.ts`. The change extends the test suite in `backup-verify.test.ts` with path traversal and boundary cases.

This follows the backup CLI feature (`0ecfd37b4`) and closes a validation gap where malformed archive paths could bypass the verify step. The hardening applies before `fs.writeFile` operations.

#### Local Backup CLI — New Credential and OAuth Exposure Surface ([`0ecfd37b4`](https://github.com/openclaw/openclaw/commit/0ecfd37b4))

Adds a new `openclaw backup` command backed by `backup.ts`, `backup-shared.ts`, `backup-verify.ts`, and `register.backup.ts`. The command archives credentials, OAuth tokens, and environment configuration from `~/.openclaw/`.

The backup feature introduces new code paths that read from `credentials/`, process `oauth` tokens, and reference `process.env` — all flagged security patterns. The `backup-shared.ts` module (254 lines) implements the shared backup/restore logic including `fs.writeFile` with archive handling.

The `docs/cli/backup.md` documents the new `--output`, `--verify`, and `--key` flags. The `reset.ts` and `uninstall.ts` commands gain backup-related checks. From a security standpoint, the backup output file contains credentials in a portable archive; key management and output path security are the primary concerns.

#### CDP Session Scoping and Stale Target Recovery ([`a075baba8`](https://github.com/openclaw/openclaw/commit/a075baba8))

Companion to `362248e55`. This commit scopes CDP sessions to individual browser contexts using `WeakMap`/`WeakSet` per-page and per-context tracking. The `roleRefsByTarget` cache enforces a 50-entry LRU bound (`MAX_ROLE_REFS_CACHE = 50`) with explicit size enforcement. Stale target recovery is hardened to detect disconnected CDP targets and re-establish sessions rather than silently failing.

The `cachedByCdpUrl` singleton is replaced with a `Map<string, ConnectedBrowser>` supporting multiple concurrent CDP connections. Console message buffering is bounded at `MAX_CONSOLE_MESSAGES = 500`.

#### SELinux :Z Mount Option for Podman ([`bd1fe4d8b`](https://github.com/openclaw/openclaw/commit/bd1fe4d8b))

The `run-openclaw-podman.sh` script adds `:Z` relabeling to volume mount arguments when the host SELinux policy is `enforcing` or `permissive`. Without `:Z`, SELinux denies container access to host-mounted volumes on enforcing systems, causing silent failures or insecure workarounds.

The container configuration template `openclaw.container.in` is updated to include the conditional mount option. This is a deployment hardening for SELinux-enabled environments (RHEL, Fedora, Rocky Linux).
