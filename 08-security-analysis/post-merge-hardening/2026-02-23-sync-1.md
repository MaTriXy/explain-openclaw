## Key Security Changes

### Directive Tag Stripping in Webchat ([`e6490732c`](https://github.com/openclaw/openclaw/commit/e6490732c))

Added `stripMessageDirectiveTags()` function to remove `[[reply_to_current]]` and other directive tags from message content before broadcasting to webchat clients in non-streaming mode. This prevents potential HTML injection through directive tags appearing in rendered messages. Closes #23053.

### Security Audit Enhancements ([`bc78b343b`](https://github.com/openclaw/openclaw/commit/bc78b343b), [`29e41d4c0`](https://github.com/openclaw/openclaw/commit/29e41d4c0))

Expanded security audit checks to detect dangerous configuration combinations:

- **Real-IP Fallback Detection:** Critical severity when `gateway.allowRealIpFallback=true` is combined with non-loopback binding or trusted-proxy mode. This configuration could allow IP spoofing through header manipulation.
- **mDNS Full Mode Detection:** Critical/warn severity for `discovery.mdns.mode="full"` with non-loopback binding. Full mode publishes `cliPath` and `sshPort` in local-network TXT records, potentially exposing metadata.

Temp-path guard fixes ensure directory traversal protection in test helpers.

### Hardcoded Channel Fallback Removal ([`1cd3b3090`](https://github.com/openclaw/openclaw/commit/1cd3b3090))

Removed hardcoded WhatsApp fallback channel logic across CLI, gateway, and delivery components. When delivery is requested but no channel is specified, the system now:

1. Auto-selects the single configured channel if only one exists
2. Requires explicit channel specification if multiple channels are configured
3. Errors clearly instead of assuming a default

This prevents unintended message delivery through implicit fallback channels. Closes #23357.

### Telegram Update Offset Race Condition ([`e33d7fcd1`](https://github.com/openclaw/openclaw/commit/e33d7fcd1))

Fixed race condition in Telegram bot where queued updates could be skipped due to incorrect offset handling. The bot now properly tracks the last processed update ID to ensure no messages are lost between polling cycles. Closes #23284.

### Exec Sandbox Fallback Hardening ([`1b327da6e`](https://github.com/openclaw/openclaw/commit/1b327da6e))

Hardened exec sandbox fallback semantics to ensure consistent security boundaries. When sandbox configuration is unavailable or invalid, the system now fails closed rather than falling back to less restrictive defaults. Related to #23398.

### Feishu Dedupe Boundary Tightening ([`bf56196de`](https://github.com/openclaw/openclaw/commit/bf56196de))

Tightened message deduplication boundary checks in the Feishu extension to prevent edge cases where duplicate messages could slip through. Closes #23377.
