#### Exec Consent Gate for Dry-Run and Preflight ([`0ffcc308f`](https://github.com/openclaw/openclaw/commit/0ffcc308f2))

Gates exec dry-run and preflight resolution behind `--allow-exec` flag. Adds comprehensive test coverage and documentation for secrets execution consent behavior, preventing unauthorized exec operations during command resolution and application. A new exec-resolution-policy module enforces that dry-run and preflight exec paths require explicit opt-in, consistent with the general exec consent model.

**Security note:** Reinforces Audit 2 Claim 2 (exec safety). The `--allow-exec` gate ensures that exec-related preflight checks cannot silently execute without user consent, closing a potential escape path where preflight resolution could trigger exec side effects.

#### Outbound Session Routing Moved to Plugin Handlers ([`4079de21c`](https://github.com/openclaw/openclaw/commit/4079de21ce))

Relocates outbound session routing logic from core into plugin handlers. The moved code handles target resolution, peer kind inference (direct/group/channel), and session key building. Routing logic previously resident in core is now delegated to channel plugins, consistent with the plugin isolation architecture.

**Security note:** Reinforces Audit 2 Claims 9/10 (session isolation and plugin/extension isolation). Moving session routing decisions to the plugin layer prevents core code from making assumptions about channel-specific routing behavior and confines session key construction to the plugin that owns the channel.

#### Outbound Session Route Helpers Added to Plugin SDK ([`826c592de`](https://github.com/openclaw/openclaw/commit/826c592deb))

Adds `buildOutboundBaseSessionKey`, `normalizeOutboundThreadId`, and `resolveThreadSessionKeys` to the public plugin SDK. These helpers allow plugins to construct and resolve session routing information directly, providing the primitives needed by channel plugins after the routing delegation introduced in `4079de21c`.

**Security note:** Reinforces Audit 2 Claim 10 (plugin/extension isolation). By exporting these helpers via the SDK boundary, plugins get a stable API for session construction without accessing core internals directly.

#### Web Search Providers Moved to Extension Plugins ([`3de973fff`](https://github.com/openclaw/openclaw/commit/3de973ffff))

Removes the monolithic web-search-core.ts (2242 lines) and distributes web search provider implementations into extension-specific providers (Brave, Google, Moonshot, Perplexity, xAI). The moved code includes API credential handling, request signing, endpoint configuration, and response parsing. Boundary verification tests catch future import violations.

**Security note:** Reinforces Audit 2 Claims 6/10. Web search provider credentials and secrets are now isolated within each extension's scope rather than consolidated in core. Boundary tests prevent future code from bypassing the plugin isolation by importing provider internals directly.

#### Group Policy Gating Behind Plugin Boundaries — 8 Channels ([`1aae93b1f`](https://github.com/openclaw/openclaw/commit/1aae93b1fa), [`a8f433d61`](https://github.com/openclaw/openclaw/commit/a8f433d611), [`bf8702973`](https://github.com/openclaw/openclaw/commit/bf8702973f), [`1f5f3fc2e`](https://github.com/openclaw/openclaw/commit/1f5f3fc2ef), [`c29458d40`](https://github.com/openclaw/openclaw/commit/c29458d407), [`7ba8dd112`](https://github.com/openclaw/openclaw/commit/7ba8dd112f), [`0bfaa3612`](https://github.com/openclaw/openclaw/commit/0bfaa36126), [`9e556f75f`](https://github.com/openclaw/openclaw/commit/9e556f75f5))

Eight commits move channel-specific group mention and tool policy authorization from the shared `src/channels/plugins/group-mentions.ts` into per-extension `group-policy.ts` files. Channels affected: LINE, BlueBubbles, Google Chat, iMessage, WhatsApp, Telegram, Discord, Slack.

Each extension now resolves its own `requireMention` and `toolPolicy` group authorization checks, eliminating the shared code path in core. Authorization behavior is preserved — the logic is identical, only relocated. The shared `group-mentions.ts` shrinks with each channel migration, reducing core's surface area for group policy decisions.

**Security note:** Reinforces Audit 2 Claim 7 (group policy gating). Moving group policy behind the plugin boundary prevents a misconfiguration in one channel's policy from affecting others. The plugin boundary also ensures that group policy changes can only affect the owning channel extension.

#### URL Fragment Preferred Over Query Parameter for Token Handling ([`56066dccb`](https://github.com/openclaw/openclaw/commit/56066dccb0))

Hardens token parameter handling guidance in web UI documentation. Changes preference order to favor URL fragments (not sent to server or proxy logs) over legacy query parameters for token passing. Adds a test case for fragment preference and updates the UI code comment.

**Security note:** Reinforces Audit 2 Claim 6 (credential/token handling). URL fragments are not forwarded in HTTP requests and do not appear in server access logs, reducing the risk of token leakage through logging infrastructure. This follows a secure-by-default principle for token transport.

#### Channel Plugin Shared Assembly Deduplicated ([`cd5c2f4cb`](https://github.com/openclaw/openclaw/commit/cd5c2f4cb2))

**Security note:** Reinforces Audit 2 Claim 3. A single assembly helper means plugin base construction logic is defined once, reducing the risk of divergence where one extension constructs its plugin base differently (and potentially less securely) than others.

#### MSTeams SDK Self-Import Guardrail Enforced ([`3cc83cb81`](https://github.com/openclaw/openclaw/commit/3cc83cb81e))

**Security note:** Reinforces Audit 2 Claim 9 (extension SDK self-import guardrail). This prevents internal extension code from depending on public SDK imports of its own module, which could create circular dependencies or allow external modifications to the SDK to affect internal extension behavior unexpectedly.

#### Plugin SDK Requires Unified Message Discovery ([`682f4d1ca`](https://github.com/openclaw/openclaw/commit/682f4d1ca3))

Touches the boundary between plugin request routing and command execution.

**Security note:** Reinforces Audit 2 Claim 3 (plugin boundary / SDK isolation). Unified message discovery ensures that all extensions follow the same contract for exposing and executing user commands, preventing extensions from bypassing sandbox isolation by implementing custom discovery paths.

#### Gateway Trusted-Proxy Scope Regression Covered ([`870f26077`](https://github.com/openclaw/openclaw/commit/870f260772))

Adds test coverage for trusted-proxy scope handling in the control-UI authentication layer. Addresses a gateway authentication control-UI scope regression with 18 lines of auth test coverage in `src/gateway/server.auth.control-ui.suite.ts`.

**Security note:** Reinforces Audit 2 Claim 1 (gateway auth/RBAC enforcement). Trusted-proxy scope determines which requests the gateway treats as originating from a trusted upstream proxy — a misconfiguration here could allow requests to bypass auth checks. The regression test prevents future changes from silently re-introducing incorrect scope handling.
