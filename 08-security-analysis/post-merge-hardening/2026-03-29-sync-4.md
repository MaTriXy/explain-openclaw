#### Harden mcp channel bridge smoke ([`ec5877346`](https://github.com/openclaw/openclaw/commit/ec5877346))

Adds E2E smoke test infrastructure for the MCP channel bridge, including new test client and seed scripts (`mcp-channels-docker-client.ts`, `mcp-channels-seed.ts`, `mcp-channels-docker.sh`). The tests validate channel auth flows and message resolution across the MCP boundary, strengthening regression coverage for RBAC-adjacent code paths. The `resolveMessageId` and `extractFirstTextBlock` helpers are exercised to ensure transcript updates are correctly attributed.

#### Restore skill sourceInfo provenance handling ([`4e50548e4`](https://github.com/openclaw/openclaw/commit/4e50548e4))

Final commit in a three-stage chain ([`8147f5075`](https://github.com/openclaw/openclaw/commit/8147f5075) → [`5853b1aab`](https://github.com/openclaw/openclaw/commit/5853b1aab) → [`4e50548e4`](https://github.com/openclaw/openclaw/commit/4e50548e4)) that converges on reading skill provenance from `sourceInfo.source` instead of the legacy flat `source` field. The security scanner in `audit-extra.async.ts` and the bundled-skill gate in `skills/config.ts` both switch to `entry.skill.sourceInfo?.source`, ensuring the provenance trail from skill discovery through code-safety scanning uses the canonical field. Test helpers (`skills.test-helpers.ts`, `compact-format.test.ts`) adopt `createSyntheticSourceInfo()` to produce valid provenance objects in fixtures. Defense-in-depth: no new boundary, but closes a drift path where the scanner could have read stale `.source` values after the inline refactor.

#### Replay skill source drift ([`5853b1aab`](https://github.com/openclaw/openclaw/commit/5853b1aab))

Intermediate commit in the provenance chain — temporarily reverts `sourceInfo?.source` reads back to the legacy flat `.source` field across `skills-install.ts`, `skills-status.ts`, `skills/config.ts`, and `audit-extra.async.ts`. This was a drift-correction step before the final restore in [`4e50548e4`](https://github.com/openclaw/openclaw/commit/4e50548e4). No net security impact; the code was in a less-canonical state only within this commit's lifetime.

#### Inline canonical skill source reads ([`8147f5075`](https://github.com/openclaw/openclaw/commit/8147f5075))

First commit in the provenance chain. Removes the `resolveSkillSource()` helper from `src/agents/skills/source.ts` (and its test) and inlines the read as `entry.skill.sourceInfo?.source?.trim() || "unknown"` at every call site: `skills-install.ts`, `skills-status.ts`, `skills/config.ts`, and `audit-extra.async.ts`. The `source.ts` file is deleted entirely (later partially re-introduced by [`ec6fba7d0`](https://github.com/openclaw/openclaw/commit/ec6fba7d0) for the legacy fallback removal). Security impact: the security scanner now reads provenance directly from the canonical `sourceInfo` field, eliminating an indirection layer that could mask provenance bugs.
