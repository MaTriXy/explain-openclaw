#### SDK Reply Pipeline Contract Tightening ([`510f4276b`](https://github.com/openclaw/openclaw/commit/510f4276b5f3ed41f6826a26f45cae681265e376))

The plugin-SDK public subpath for channel reply pipelines was tightened: internal helpers `createReplyPrefixContext` and `createTypingCallbacks` were removed from the exported surface of `channel-reply-pipeline.js`, leaving only the consolidated `createChannelReplyPipeline()` entry point. All 10 channel extensions (Discord, Feishu, iMessage, Slack, Telegram, WhatsApp, Line, Zalo, BlueBubbles, and others) were updated to import via the unified contract. This defense-in-depth change reduces the plugin-SDK API surface that third-party extensions can reach, limiting unintended coupling to internal reply-pipeline internals.

#### Session Lifecycle and Operator Scope Hardening ([`7b61ca1b0`](https://github.com/openclaw/openclaw/commit/7b61ca1b06154cfc630244bc9535da769608680a))

A scope-preservation bug was fixed in the WebSocket connection handshake: token-authenticated backend clients (such as the agent dashboard) no longer have their operator scopes cleared when device identity is unavailable. Previously, the scope-clearing path ran unconditionally when `device` was falsy; now the check gates on `decision.kind` before clearing scopes, ensuring token-authed operators retain the scopes included in their token. Four new files were introduced to support this overhaul: `session-lifecycle-events.ts` (emit/subscribe create and delete events as subagents spawn and exit), `session-lifecycle-state.ts` (state machine tracking active sessions), `session-kill-http.ts` (admin endpoint to forcibly terminate sessions), and `sessions-history-http.ts` (HTTP/SSE endpoints for session history). This directly strengthens Audit 2 Claim 5 (self-approving agent / RBAC bypass via unvalidated scopes).
