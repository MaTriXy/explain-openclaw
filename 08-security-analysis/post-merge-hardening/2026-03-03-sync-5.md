#### Harden workspace bootstrap boundary reads ([`07b16d5ad`](https://github.com/openclaw/openclaw/commit/07b16d5ad))

Three workspace bootstrap code paths — sandbox seed population in `src/agents/sandbox/workspace.ts`, post-compaction context loading in `src/auto-reply/reply/post-compaction-context.ts`, and compaction safeguard in `src/agents/pi-extensions/compaction-safeguard.ts` �� now use `openBoundaryFile()` from `src/infra/boundary-file-read.ts` when reading files during workspace initialization. Previously, these paths opened files with plain `fs.readFile()` without boundary checks, allowing paths that resolve outside the workspace root to be read silently. The new `openBoundaryFile()` call resolves the target path to its real path, checks it is strictly under the configured root, and returns a file descriptor; callers read from the fd and close it. If the resolved path escapes the root, the function returns `{ ok: false }` and the caller skips the entry. This closes a bootstrap-time read boundary gap that complements the existing write-side `writeFileWithinRoot()` protections in `src/infra/fs-safe.ts`.

#### Split sandbox media staging and stream safe copies ([`4a8031162`](https://github.com/openclaw/openclaw/commit/4a8031162))

`src/infra/fs-safe.ts` gains `writeFileWithinRoot()`, `copyFileWithinRoot()`, and the internal `openWritableFileWithinRoot()` helper. These functions open files via `openFileWithinRoot()` (which resolves symlinks and checks the real path is under the root before returning a file handle), perform the write/copy operation, then close the handle. On write failure, any newly created file is cleaned up. `src/auto-reply/reply/stage-sandbox-media.ts` is refactored to use these new primitives for both local and remote media staging. File size is capped at `MEDIA_MAX_BYTES` (from `src/media/store.ts`) on every staging path. The refactor makes safe-copy semantics explicit and removes duplicated manual boundary checks that existed in the old staging code.

#### Split safe-regex parse and bounded matching ([`031bf0c6c`](https://github.com/openclaw/openclaw/commit/031bf0c6c))

`src/security/safe-regex.ts` is refactored to separate regex parse state from match execution. Internal types `QuantifierRead`, `TokenState`, and `ParseFrame` gain explicit length-tracking fields (`minRepeat`, `maxRepeat`, `branchMinLength`, `branchMaxLength`, `altMinLength`, `altMaxLength`). A new `createParseFrame()` factory ensures frames are initialized with consistent defaults. A new exported `testRegexWithBoundedInput()` function caps input length to `SAFE_REGEX_TEST_WINDOW` (2048) before executing a compiled safe regex — callers that previously called `.test()` directly on a compiled pattern are now exposed to a bounded API that prevents ReDoS via oversized inputs. The internal `addLength()` and `multiplyLength()` helpers handle `+Infinity` sentinel values so bounded-length analysis does not overflow for unbounded quantifiers.

#### Use NTFS junctions and platform guards for symlink tests on Windows (openclaw#28747) thanks @arosstale ([`1b462ed17`](https://github.com/openclaw/openclaw/commit/1b462ed17))

Symlink traversal tests for Windows (openclaw#28747) previously attempted to create POSIX-style symlinks via `fs.symlink()`, which requires elevated privileges on Windows and caused the test suite to either skip or fail unreliably. This commit replaces symlink creation with NTFS directory junctions (via `fs.symlink(target, dest, 'junction')`) where available and wraps each symlink test block in a platform guard that skips on Windows when junction creation fails. The change ensures path traversal boundary tests for agent directories run reliably on Windows CI without requiring elevated permissions, providing parity with the Linux/macOS test coverage for symlink-based escape attempts.

#### Harden sandbox writes and centralize atomic file writes ([`18f8393b6`](https://github.com/openclaw/openclaw/commit/18f8393b6))

`src/infra/json-files.ts` gains `writeJsonAtomic()` and `writeTextAtomic()`. Both functions write to a UUID-suffixed temp file in the same directory, optionally `chmod` it to the requested mode, then `rename()` atomically to the target path — preventing torn reads of partially-written JSON config files. Ten production files (`src/agents/sandbox/fs-bridge.ts`, `src/agents/sandbox/registry.ts`, `src/config/sessions/store.ts`, `src/hooks/bundled/session-memory/handler.ts`, `src/infra/update-startup.ts`, `src/memory/qmd-manager.ts`, `src/node-host/config.ts`, `src/plugin-sdk/json-store.ts`, `src/telegram/update-offset-store.ts`, and `src/infra/voicewake.ts`) are migrated to use these centralized atomic primitives, replacing ad-hoc `writeFile` calls and custom temp-file patterns. `src/agents/sandbox/fs-bridge.ts` additionally gates sandbox write operations through `writeFileWithinRoot()` from `src/infra/fs-safe.ts`, ensuring all sandbox writes stay within the declared workspace root.

#### Block quantified ambiguous alternation regex ([`31c7637e0`](https://github.com/openclaw/openclaw/commit/31c7637e0))

The regex safety checker in `src/security/safe-regex.ts` is extended to detect quantified ambiguous alternation, a pattern class that can cause catastrophic ReDoS in backtracking engines. The parse frame now tracks `hasAlternation`, `branchMinLength`, `branchMaxLength`, `altMinLength`, and `altMaxLength`; when a group containing an alternation closes and the branches can overlap (both min-lengths > 0 and the outer group has a quantifier), `hasAmbiguousAlternation` is set on the resulting `TokenState`. The top-level safety check returns `false` (unsafe) for any pattern where `hasAmbiguousAlternation && containsRepetition` is true. Helper functions `createParseFrame()`, `addLength()`, and `multiplyLength()` support the length-tracking arithmetic throughout the parse.

#### Harden tar archive extraction parity ([`0dbb92dd2`](https://github.com/openclaw/openclaw/commit/0dbb92dd2))

`src/infra/archive.ts` gains `createTarEntrySafetyChecker()`, a factory that returns a per-entry callback for use during tar extraction. The checker validates each entry's path against the output root via `validateArchiveEntryPath()` and `resolveArchiveOutputPath()` (both pre-existing), counts entries and tracks cumulative size against configured `ArchiveExtractLimits`, and blocks any entry whose type is in `BLOCKED_TAR_ENTRY_TYPES` — a set containing `SymbolicLink`, `Link`, `BlockDevice`, `CharacterDevice`, `FIFO`, and `Socket`. Previously the symlink-blocking logic was only applied in the main skill download extraction path; this commit adds parity to a second extraction call site in `src/agents/skills-install-download.ts` by threading the same safety checker through both paths.

#### Fail closed navigation guard with env proxy ([`ea204e65a`](https://github.com/openclaw/openclaw/commit/ea204e65a))

`src/browser/navigation-guard.ts` gains a pre-navigation env proxy guard. The guard imports `hasProxyEnvConfigured()` from `src/infra/net/proxy-env.ts` and `isPrivateNetworkAllowedByPolicy()` from `src/infra/net/ssrf.ts`. Before hostname resolution, if `hasProxyEnvConfigured()` returns true and `isPrivateNetworkAllowedByPolicy()` indicates private-network access is not explicitly allowed by policy, the guard throws `InvalidBrowserNavigationUrlError` before launching the request. The rationale is that browser network stacks apply env proxy routing at connect-time, bypassing pre-navigation DNS checks; without this gate the SSRF guard would pass while traffic routes through an attacker-controlled proxy. This commit brings the pre-navigation guard into parity with the guarded fetch behavior established by `c973b053a`.

#### Split browser context/actions and unify CDP timeout policy ([`17c434f2f`](https://github.com/openclaw/openclaw/commit/17c434f2f))

`src/browser/cdp-proxy-bypass.ts` gains the `NoProxyLeaseManager` class (line 73) with an `acquire(url)` method. The manager tracks an active `NO_PROXY` environment variable snapshot and implements reference-counted lease semantics: `acquire()` sets `NO_PROXY` to bypass the proxy for the target URL and returns a release callback; the release callback restores the previous value when the last lease for that URL is dropped. This replaces the previous non-reentrant set/restore pattern which had a race condition when multiple concurrent CDP connections to different localhost ports each tried to mutate `NO_PROXY` independently. The broader refactor splits `src/agents/tools/browser-tool.ts` into a context file and a new `src/agents/tools/browser-tool.actions.ts`, and introduces `src/browser/cdp-timeouts.ts` to centralize the timeout policy used across CDP operations.

#### Validate sessions_spawn agentId format ([`b28e472fa`](https://github.com/openclaw/openclaw/commit/b28e472fa))

`src/routing/session-key.ts` gains the exported `isValidAgentId()` function, which tests a trimmed input string against `VALID_ID_RE` (`/^[a-z0-9][a-z0-9_-]{0,63}$/i`). `src/agents/subagent-spawn.ts` is updated to call `isValidAgentId()` at the top of the spawn path and return an error immediately if the supplied `agentId` is invalid. This replaces the formerly local `STRICT_AGENT_ID_RE` constant introduced in `0c6db05cc` and centralises the format contract in `session-key.ts` alongside `normalizeAgentId()`. A new test in `src/routing/session-key.test.ts` verifies boundary cases for the exported validator. The session-spawn allowlist test is updated to remove the now-redundant local regex assertion.

#### Add strict format validation to sessions_spawn for agentId ([`0c6db05cc`](https://github.com/openclaw/openclaw/commit/0c6db05cc))

A local constant `STRICT_AGENT_ID_RE = /^[a-z0-9][a-z0-9_-]{0,63}$/i` is introduced in `src/agents/subagent-spawn.ts` as a format gate applied at the entry point of the spawn flow, before `normalizeAgentId()` runs. The commit message describes the root cause: `normalizeAgentId()` accepts arbitrary strings and converts invalid characters to hyphens, meaning an error-message-like value (e.g. `"Agent not found: xyz"`) would be silently mangled into `"agent-not-found--xyz"`, potentially creating ghost workspace directories whose names encode attacker-controlled strings. The gate rejects any `agentId` that does not match the strict format and returns a structured error with guidance to use `agents_list` to discover valid targets. This is the precursor commit to `b28e472fa` which centralises the same regex into an exported `isValidAgentId()` function.
