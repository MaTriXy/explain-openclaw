#### Harden outbound, matrix bootstrap, and plugin entry resolution ([`c424836fb`](https://github.com/openclaw/openclaw/commit/c424836fb))

#### Harden backup file permissions and clean orphan .bak files ([`3c0ec76e8`](https://github.com/openclaw/openclaw/commit/3c0ec76e8))

Addresses GitHub issue #31699 â€” config `.bak` files persisting with sensitive data and potentially loose permissions. Two new functions are introduced: `hardenBackupPermissions()` explicitly sets `0o600` on all `.bak` files in the rotation ring after creation (rather than relying on `copyFile` to preserve source permissions, which is not guaranteed on Windows or NFS mounts), and `cleanOrphanBackups()` removes `.bak` files that fall outside the managed 5-deep rotation ring (e.g. PID-stamped leftovers from interrupted writes or manual backups like `.bak.before-marketing`). The `BackupRotationFs` interface is extended with optional `chmod` and `readdir` fields to support injection of both functions. `io.ts` is updated to call both hardening functions after each backup write. The backup ring itself is preserved as a recovery mechanism â€” only the permission and stale-file surface is reduced.

#### Scope auth across media redirects ([`4a414c5e5`](https://github.com/openclaw/openclaw/commit/4a414c5e5))

Adds an `authorizationAllowHosts` parameter to `safeFetch` calls in the MS Teams media download path, ensuring Bearer tokens are only forwarded to hosts in the explicit allowlist. A new `isRedirectStatus()` helper distinguishes redirect responses (301/302/303/307/308) from auth failures in `fetchWithAuthFallback()`, so redirect responses propagate to the outer guard instead of being handled as 401/403 retry cases. This prevents Bearer tokens from leaking to arbitrary redirect destinations â€” the primary mechanism described in GHSA-7vwx-582j-j332.

#### Scope graph auth redirects ([`8937c10f1`](https://github.com/openclaw/openclaw/commit/8937c10f1))

Previously, the Graph path used an unguarded fetch, allowing any redirect target to receive the Authorization header. This is the Graph-side complement to the media download fix in `4a414c5e5`, together completing the GHSA-7vwx-582j-j332 patch.

#### Enforce guarded redirect ownership in safeFetch ([`cceecc8bd`](https://github.com/openclaw/openclaw/commit/cceecc8bd))

Implements the core `safeFetch` guard in `shared.ts`, adding DNS-based SSRF protection and redirect ownership validation for all MS Teams media downloads. Key additions:

- `isPrivateOrReservedIP` / `resolveAndValidateIP()` â€” delegates to the SDK's `isPrivateIpAddress` (which handles IPv4-mapped IPv6, NAT64, 6to4, Teredo, octal IPv4, and fails closed on parse errors) to reject hostnames that resolve to private/loopback/link-local IPs.
- `safeFetch()` â€” implements manual redirect following (up to `MAX_SAFE_REDIRECTS = 5`) with per-hop allowlist and DNS validation. Each redirect target is checked against `allowHosts` for hostname suffix match and against `resolveAndValidateIP` for private IP ranges. The `authorizationAllowHosts` parameter controls which hosts may receive the Bearer token across redirect hops â€” any redirect to a host outside this list strips the Authorization header.

This directly addresses GHSA-7qf6-h84j-8fq4 (MS Teams SSRF via media redirect) and reinforces Audit 2 Claim 4 (DNS rebinding protection).
