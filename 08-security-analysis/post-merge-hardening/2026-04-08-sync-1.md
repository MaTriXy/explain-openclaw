#### Path Traversal Defense in Telegram Local Media ([`cfc52fcf2`](https://github.com/openclaw/openclaw/commit/cfc52fcf2b))

[`cfc52fcf2`](https://github.com/openclaw/openclaw/commit/cfc52fcf2b) hardens Telegram Bot API media file handling for self-hosted deployments. It introduces a `trustedLocalFileRoots` config array and validates all absolute file paths returned by local Bot API servers against this allowlist using `resolveTrustedLocalTelegramRoot()` (`extensions/telegram/src/bot/delivery.resolve-media.ts:160`). The validator rejects paths containing `..` sequences or that escape to unintended roots, leveraging `readFileWithinRoot()` from `openclaw/plugin-sdk/infra-runtime`. This addresses a legitimate defense-in-depth gap where self-hosted local Telegram Bot API servers could return arbitrary absolute file paths that bypass intended media sandboxing.

**Files changed:** 7 production + 2 test
**Audit claims addressed:** Defense-in-depth — outPath validation for local self-hosted Telegram Bot API deployments; addresses Legitimate Gap 3
