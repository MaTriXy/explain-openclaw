#### Centralize system.run approval context and errors ([`4e690e09c`](https://github.com/openclaw/openclaw/commit/4e690e09c))

Refactored exec approval context and error handling into dedicated modules (`system-run-approval-context.ts`, `node-invoke-system-run-approval-errors.ts`). New helpers `systemRunApprovalGuardError()` and `systemRunApprovalRequired()` consolidate error formatting and payload normalization across V1 binding and V2 plan approval flows.

#### Share node command catalog ([`d06632ba4`](https://github.com/openclaw/openclaw/commit/d06632ba4))

Extracted node command constants into a shared `node-commands.ts` module exposing `NODE_SYSTEM_RUN_COMMANDS`, `NODE_SYSTEM_NOTIFY_COMMAND`, `NODE_BROWSER_PROXY_COMMAND`, and `NODE_EXEC_APPROVALS_COMMANDS`. This enables consistent command enumeration across gateway policy (`node-command-policy.ts`) and node host (`runner.ts`), reducing the risk of command catalog divergence.

#### Enforce account-scoped pairing APIs ([`bce643a0b`](https://github.com/openclaw/openclaw/commit/bce643a0b))

Enforced account-scoped pairing across all messaging channels (Discord, Slack, Telegram, Signal, iMessage, LINE, WhatsApp web, and internal delivery). Centralized DM/account validation logic in `dm-policy-shared.ts` and updated all message-handler preflight checks, onboarding flows, and auto-reply routing to require paired account context before routing.

#### Cover verification key and header helpers ([`192df12d6`](https://github.com/openclaw/openclaw/commit/192df12d6))

Extended webhook security test coverage with new `http-headers.test.ts`: added tests for `getHeader()` (case-insensitive header lookup), tests for `skipVerification` mode producing stable replay keys across Plivo, Telnyx, and Twilio providers, and a test verifying that missing `verifiedRequestKey` causes 401 rejection.

#### Enforce verified webhook key contract ([`535ef8991`](https://github.com/openclaw/openclaw/commit/535ef8991))

Refactored webhook verification to enforce the `verifiedRequestKey` contract: extracted `createSkippedVerificationReplayKey()` for stable keys in `skipVerification` mode, moved `getHeader()` to `http-headers.ts`, and updated all provider verification paths to return `verifiedRequestKey`. Extracted stale call reaper into `webhook/stale-call-reaper.ts` with a return closure pattern.

#### Harden node exec approvals against symlink rebind ([`78a7ff2d5`](https://github.com/openclaw/openclaw/commit/78a7ff2d5))

Hardened exec approvals against symlink-rebind TOCTOU attacks via a new `system.run.prepare` command and `SystemRunApprovalPlanV2` type. The plan captures canonicalized argv, cwd, rawCommand, agentId, and sessionKey at prepare time. Gateway and CLI now validate the plan via `normalizeSystemRunApprovalPlanV2()` and reject execution if cwd path components contain symlinks.

#### Add embedded pi project settings policy ([`38b6cee02`](https://github.com/openclaw/openclaw/commit/38b6cee02))

Added configuration knob `agents.defaults.embeddedPi.projectSettingsPolicy` with three modes: `trusted` (pass through), `sanitize` (strips `shellPath`/`shellCommandPrefix`), and `ignore` (disables untrusted workspace-local `.pi/config/settings.json`). Updated config schema, labels, and help text across Zod validation and types.

#### Bind webhook dedupe to verified request identity ([`1aadf26f9`](https://github.com/openclaw/openclaw/commit/1aadf26f9))

Hardened webhook deduplication against idempotency-token manipulation by binding the dedupe key to verified request identity. Added `WebhookParseOptions` type with `verifiedRequestKey` field. All providers (Twilio, Telnyx, Plivo) now use verified request key for dedupe instead of unprotected HTTP headers. Integrated `fetchWithSsrFGuard()` into Telnyx and Plivo API calls for SSRF protection.

#### Align OAuth project discovery metadata and endpoint fallbacks ([`764cd5a31`](https://github.com/openclaw/openclaw/commit/764cd5a31))

Aligned OAuth endpoint fallbacks for Gemini CLI auth with environment-specific discovery arrays (`LOAD_CODE_ASSIST_ENDPOINTS`). Extracted `resolvePlatform()` and `fetchWithTimeout()` helpers. Routed token and userinfo fetches through `fetchWithSsrFGuard()` for SSRF protection. Added environment-variable project ID fallbacks and comprehensive tests for endpoint discovery.

#### Unify outbound session context wiring ([`a1628d89e`](https://github.com/openclaw/openclaw/commit/a1628d89e))

Unified outbound session context across all delivery paths (gateway send, auto-reply, cron jobs, heartbeat, restart sentinels). Introduced `buildOutboundSessionContext()` and `OutboundSessionContext` type combining session key and agentId. Updated `deliverOutboundPayloads()` to accept a unified `session` object instead of separate parameters, and updated internal hook dispatch to use `session.key`.

Adds `stripWindowsNamespacePrefix()` and `normalizeWindowsPathForComparison()` helpers to handle Windows namespace-prefixed paths (`\\?\` and `//?/` formats) in path containment validation. Refactors `isPathInside()` in `path-guards.ts` to strip namespace prefixes before comparison, preventing CWE-22 path traversal via UNC/namespace exploitation on Windows. **Audit 2 Claim 3** (path traversal) — strengthens existing containment checks.
