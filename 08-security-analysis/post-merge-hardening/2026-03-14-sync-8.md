#### Add chrome MCP existing-session support ([`593964560`](https://github.com/openclaw/openclaw/commit/593964560b))

Adds Chrome MCP integration to support existing browser sessions. The new `chrome-mcp.ts` module enables OpenClaw to connect to and control Chrome instances that were not launched by OpenClaw's own Playwright driver. A browser profile with `driver: "existing-session"` uses this path to discover open tabs and issue commands via the Chrome DevTools Protocol. Download and trace capture operations are explicitly blocked for existing-session profiles (returning HTTP 501) pending future implementation.

#### Persist Auth Profile Env Refs for Daemon Install ([`ba9fb4d99`](https://github.com/openclaw/openclaw/commit/ba9fb4d994))

New function `collectAuthProfileServiceEnvVars()` in `src/commands/daemon-install-helpers.ts` iterates auth profile credentials and forwards those with `source='env'` secret refs into the daemon service environment. Called in `buildGatewayInstallPlan()` alongside `collectConfigServiceEnvVars()`. Previously, credentials configured via environment variable references (e.g. ANTHROPIC_API_KEY referenced by an auth profile) were silently dropped during daemon installation, causing authentication failures when the gateway ran as a system service.
