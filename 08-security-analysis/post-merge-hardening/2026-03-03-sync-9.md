#### Plugin Install Hardening and DM Route Pinning ([`b782ecb7e`](https://github.com/openclaw/openclaw/commit/b782ecb7e))

Two-part hardening commit affecting plugin lifecycle and DM routing security.

**Plugin install error codes:** `ensureOpenClawExtensions` now returns structured `PLUGIN_INSTALL_ERROR_CODE` values instead of throwing, enabling callers like `plugins-cli` to distinguish `INVALID_NPM_SPEC`, `MISSING_OPENCLAW_EXTENSIONS`, `NPM_PACKAGE_NOT_FOUND`, and `PLUGIN_ID_MISMATCH` errors. Previously, all install failures looked identical at the call site, making it impossible to distinguish a missing package from a manifest mismatch without inspecting the thrown error message string — fragile and bypass-prone.

**DM route pinning:** `shouldSkipPinnedMainDmRouteUpdate` and `resolvePinnedMainDmOwnerFromAllowlist` guard the main DM recipient update path. Before route updates proceed, the pinning check verifies that the owner and sender recipients differ and that the candidate route matches the single-entry allowlist constraint. This prevents misconfigured multi-user DM routing from overriding owner-pinned DM flows — a vector where an attacker in a shared environment could inject themselves as the pinned DM recipient by triggering a routing update.

#### Security Audit Caching ([`5a32a66aa`](https://github.com/openclaw/openclaw/commit/5a32a66aa))

Performance optimization of existing security evaluation paths via three-tier caching. No auth bypass or input validation changes.

`allowFromReadCache` in `pairing-store.ts` deduplicates async and sync allowlist reads per session, invalidating entries on `mtimeMs` changes to ensure pairing allowlist freshness without re-reading on every request. `EvaluatedBindingsIndex` in `resolve-route.ts` pre-indexes evaluated bindings by peer/guild/team/account for O(1) lookups, eliminating linear scans over the bindings set during route resolution. `CodeSafetySummaryCache` in `audit-extra.async.ts` deduplicates skill scanner results across plugins and installed skills, reducing redundant deep analysis passes during `runSecurityAudit(..., { deep: true })`.

#### Exec Allowlist Regex Literal Escaping ([`8da8756f7`](https://github.com/openclaw/openclaw/commit/8da8756f7))

Fixes a malformed character class in `globToRegExp()` within the exec allowlist path matcher. A new `escapeRegExpLiteral()` helper correctly escapes all regex metacharacters (`.*+?^${}()|[\]`) before glob pattern compilation. Without this fix, an allowlist glob pattern containing a literal `[` or `]` character would produce a malformed character class in the compiled regex, causing the matcher to either throw or silently match broader paths than the allowlist intended — potentially allowing commands outside the approved glob pattern to pass the exec approval gate.

This commit directly addresses **Audit 2 Claim 7** ("Shell injection via incomplete regex"): the specific metacharacter escaping gap described in the audit is now closed for `globToRegExp`-based allowlist path matching.
