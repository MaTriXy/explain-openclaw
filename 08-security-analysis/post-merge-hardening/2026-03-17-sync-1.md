#### SSH Sandboxing Moved into Core ([`b8bb8510a2`](https://github.com/openclaw/openclaw/commit/b8bb8510a2))

SSH sandboxing functionality has been migrated from the openshell extension into the core `src/agents/sandbox/` module. New files created: `ssh-backend.ts` (exports `sshSandboxBackendManager` and `createSshSandboxBackend`), `ssh.ts` (exports `createSshSandboxSessionFromSettings`, `runSshSandboxCommand`, `uploadDirectoryToSshTarget`, and `shellEscape`/`buildRemoteCommand` helpers), and `remote-fs-bridge.ts` (exports `createRemoteShellSandboxFsBridge`). A new `config.ts` file centralizes SSH sandbox configuration. By bringing SSH sandboxing into core, the security audit scope now includes this boundary directly — extension-specific implementations are separated from the core security infrastructure. No changes to gateway authentication or existing sandbox controls.

#### SSH Sandbox Test Coverage Expanded ([`0a2f95916b`](https://github.com/openclaw/openclaw/commit/0a2f95916b))

A new test file `src/agents/sandbox/ssh-backend.test.ts` adds comprehensive coverage for the SSH sandbox backend including SSH session creation, command execution, and directory upload operations. The tests use mocked SSH operations to validate sandbox secret references and SSH identity data configuration without requiring live SSH infrastructure. This is a defense-in-depth improvement: prior to this commit, the SSH sandbox code had minimal test coverage. No production security boundary changes.

#### Auth Policy Moved to Provider Scope ([`7a6be3d531`](https://github.com/openclaw/openclaw/commit/7a6be3d531) + [`a33caab280`](https://github.com/openclaw/openclaw/commit/a33caab280))

Two commits (7a6be3d531 and a33caab280) refactor how authentication and model policy decisions are made. Auth configuration for bundled providers (Anthropic, OpenAI, xAI) is moved from `src/plugins/bundled-provider-auth-env-vars.ts` into the respective extension `index.ts` files. `src/plugins/types.ts` gains new provider auth and model policy interfaces. Core `src/commands/models/auth.ts` and `src/secrets/provider-env-vars.ts` are updated to dispatch auth initialization to providers. This is an architectural change: auth policy is now co-located with the provider that owns it, improving encapsulation. No behavioral changes to authentication logic, credential storage, or authorization checks. Does not affect any of the 16 audit claims.
