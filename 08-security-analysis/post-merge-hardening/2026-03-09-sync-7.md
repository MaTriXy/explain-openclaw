#### ACP: add optional ingress provenance receipts ([`e3df94365`](https://github.com/openclaw/openclaw/commit/e3df94365))

Introduces optional provenance receipts for ACP bridge clients. When configured with `--provenance meta` or `--provenance meta+receipt`, `buildSystemInputProvenance()` attaches source metadata (hostname, cwd, originating session ID, bridge identifier) to inbound messages, and `buildSystemProvenanceReceipt()` builds the receipt.

Client access is gated by `isAcpBridgeClient()` — only clients matching `displayName='ACP'` and `version='acp'` can set `systemInputProvenance` or `systemProvenanceReceipt` fields; all others receive an `INVALID_REQUEST` error. Input normalization via `normalizeInputProvenance()` enforces a kind enum (`external_user | inter_session | internal_system`), trims optional strings, and rejects non-object inputs. Receipt content is sanitized via `normalizeOptionalChatSystemReceipt()`.

Provenance is informational metadata only — not a trust mechanism. The ACP-only gate prevents untrusted clients from spoofing source attribution.

#### Cover owner-only tool availability ([`03a6e3b46`](https://github.com/openclaw/openclaw/commit/03a6e3b46))

Adds test coverage for the owner-only tool availability authorization gate in cron isolated runs. This is the companion test for `41e023a80` (which restored the gate). The test verifies that the authorization gate correctly restricts tool access to the session owner in isolated cron agent contexts, preventing regression of that security control.

#### Stagger missed cron jobs on restart (#18925) (thanks @rexlunae) ([`96d17f3cb`](https://github.com/openclaw/openclaw/commit/96d17f3cb))

**Escalated due to shared files with security commits in `src/cron/service/timer.ts`. No security content.** This is a pure reliability fix: staggers missed cron job catch-ups on restart to prevent gateway overload from simultaneous execution of accumulated jobs after downtime.

#### Pin validated download roots ([`9abf014f3`](https://github.com/openclaw/openclaw/commit/9abf014f3))

`assertCanonicalPathWithinBase()` is applied to the skill installation download target directory in `src/agents/skills-install-download.ts`. This blocks directory traversal attacks where a malicious skill specification could supply a `spec.targetDir` path that escapes the designated skill tools root directory. Any `targetDir` that resolves outside the validated root is rejected before installation proceeds.

#### Bind bun and deno approval scripts ([`cf3a479bd`](https://github.com/openclaw/openclaw/commit/cf3a479bd))

Hardens approval script creation for bun and deno runtimes. Three mitigations are applied:

1. **TOCTOU prevention:** `hasMutableSymlinkPathComponentSync()` detects symlink components in the script path before writing, blocking time-of-check/time-of-use attacks where an attacker races to replace a path component with a symlink.
2. **Integrity verification:** SHA-256 hashing ensures the script content matches expected values before and after writing.
3. **Atomic writes and permissions:** `fs.writeFile` creates scripts atomically; `fs.chmod(0o755)` sets correct execute permissions.

#### Validate config before restart to prevent crash + macOS permission loss ([`eea925b12`](https://github.com/openclaw/openclaw/commit/eea925b12))

The gateway restart flow now validates configuration before initiating a restart. If validation fails, the restart is aborted. This prevents a scenario where macOS grants elevated permissions to the already-running service, which then crashes due to a bad configuration — potentially leaving the service in a degraded state with elevated permissions intact.

#### Enforce sender allowlists with route allowlists ([`88aee9161`](https://github.com/openclaw/openclaw/commit/88aee9161))

Enforces sender allowlists alongside route allowlists for MS Teams message routing. Previously, route-level filtering was applied without also enforcing sender restrictions, creating a gap where disallowed senders could reach protected routes if route matching was otherwise satisfied. The fix enforces both sender and route restrictions together.

#### Restore owner-only tools for isolated runs ([`41e023a80`](https://github.com/openclaw/openclaw/commit/41e023a80))

Restores the owner-only tool authorization gate in isolated cron agent runs. The gate restricts tool access to the session owner in isolated cron contexts, preventing non-owner requests from accessing owner-restricted tools during cron job execution. This access control had been dropped and is now reinstated with test coverage added in `03a6e3b46`.

#### Enforce redirect-hop SSRF checks ([`93775ef6a`](https://github.com/openclaw/openclaw/commit/93775ef6a))

The browser navigation guard now validates each redirect hop against the SSRF blocklist, not only the initial request URL. A redirect chain starting from an allowed URL could previously reach a blocked destination undetected. The fix introduces a navigation guard in `src/browser/navigation-guard.ts` that intercepts redirects and validates each destination URI, aborting navigation if any hop resolves to a blocked target.

#### Re-expose configured tools under restrictive profiles ([`e8775cda9`](https://github.com/openclaw/openclaw/commit/e8775cda9))

Updates tool policy in `src/agents/pi-tools.policy.ts` to re-expose user-configured tools when running under restrictive security profiles. Restrictive profiles were incorrectly suppressing tools that the user had explicitly configured, reducing functionality without a corresponding security benefit. The fix distinguishes between admin-level tool restrictions (preserved) and user-configured tool visibility (restored). `src/agents/config-state.ts` is updated to track this policy state.

#### Harden browser runtime profile handling ([`141738f71`](https://github.com/openclaw/openclaw/commit/141738f71))

Large refactor (53 files, ~790 insertions) that formalizes browser runtime profile handling across all extension runtimes and core browser components. Three new source files establish the hardened model:

- `src/browser/profile-capabilities.ts`: centralizes capability queries, replacing ad-hoc profile state checks
- `src/browser/errors.ts`: provides explicit error types for unknown or unsupported profiles, preventing silent undefined behavior
- `src/browser/runtime-lifecycle.ts`: manages profile lifecycle state transitions

All extensions (bluebubbles, discord, feishu, matrix, msteams, telegram, slack, and others) are updated to use the new capability model. The change eliminates scenarios where missing or unknown profiles could lead to undefined runtime behavior.

#### Add download timeout to prevent polling loop hang ([`4d501e4cc`](https://github.com/openclaw/openclaw/commit/4d501e4cc))

Adds a download timeout to `readResponseWithLimit` in `src/media/fetch.ts` and `src/media/read-response-with-limit.ts`, enforced for Telegram media delivery. Without this limit, a slow or stalled media server could hold a connection indefinitely, blocking the Telegram polling loop and causing denial of service through resource exhaustion. The fix bounds download duration, releasing the polling loop if a download exceeds the configured time limit.
