#### MSTeams Sender Allowlist Bypass Fix ([`897cda7d9`](https://github.com/openclaw/openclaw/commit/897cda7d99))

Fixes GHSA-g7cr-9h7q-4qxq. When a route-level (teams/channel) allowlist was configured but the sender allowlist (`allowFrom`/`groupAllowFrom`) was empty, `resolveSenderScopedGroupPolicy` would downgrade the effective group policy from "allowlist" to "open", allowing any Teams user to interact with the bot. The fix ensures that when a route allowlist is present, the sender policy stays in "allowlist" mode even if the sender-specific fields are empty.

#### Host Env Override Hardening ([`7abfff756`](https://github.com/openclaw/openclaw/commit/7abfff756d))

Strengthens environment variable override enforcement across the gateway and macOS node runtime. Adds `HOST_DANGEROUS_OVERRIDE_ENV_KEY_VALUES` and `HOST_DANGEROUS_OVERRIDE_ENV_PREFIXES` constants in `src/infra/host-env-security.ts` alongside new functions `isDangerousHostEnvOverrideVarName()`, `inspectHostExecEnvOverrides()`, and `sanitizeSystemRunEnvOverrides()`. The macOS host env sanitizer gains rejection of overrides that attempt to set dangerous keys like LD_PRELOAD or DYLD_* prefixes. The system now fails closed on override validation errors rather than silently passing through.

#### LINE Webhook Raw Body Hardening ([`c7134e629`](https://github.com/openclaw/openclaw/commit/c7134e629c))

Removes the fallback to untrusted `req.body` in LINE webhook handling. The `parseWebhookBody()` function in `src/line/webhook.ts` now accepts only the signature-verified `rawBody` parameter instead of the previous `(req, rawBody)` signature. All webhook events must derive from the cryptographically signed raw bytes, preventing tampering via pre-parsed Express request objects.

#### SSRF Guard Credential Bypass Coverage ([`a20ba7497`](https://github.com/openclaw/openclaw/commit/a20ba74978))

Adds test coverage for URL credential-based SSRF bypass vectors. New test cases in `src/infra/net/fetch-guard.ssrf.test.ts` validate that `fetchWithSsrFGuard` correctly rejects private IPv4 and IPv6 addresses even when credentials are embedded in the URL authority (e.g., `http://attacker.com@127.0.0.1:8080`). Covers redirect chains where the redirect target contains embedded credentials.

#### JSON/JSON5 Parsing Tightening ([`93fbe26ad`](https://github.com/openclaw/openclaw/commit/93fbe26adb))

Introduces `parseJsonWithJson5Fallback()` in `src/utils/parse-json-compat.ts` and applies it across config, cron store, and subagent depth parsing paths. This replaces ad-hoc JSON5 fallback patterns with a single auditable function that tries strict `JSON.parse` first and only falls back to JSON5 when strict parsing fails. Reduces the attack surface for config injection by narrowing the set of accepted input formats.

#### Pairing Setup Code Credential Scope Reduction ([`11d71ca35`](https://github.com/openclaw/openclaw/commit/11d71ca352))

Removes `token` and `password` fields from `PairingSetupPayload` in `src/pairing/setup-code.ts`, and deletes the helper (formerly `resolvePairingSetupSharedAuth`) that resolved gateway auth credentials into setup codes. Setup codes now carry only the `bootstrapToken`, preventing reusable gateway credentials from being embedded in QR codes or pairing URIs.

#### Matrix Inbound Event Deduplication ([`a05da7671`](https://github.com/openclaw/openclaw/commit/a05da76718))

Adds `createMatrixInboundEventDeduper()` in a new `extensions/matrix/src/matrix/monitor/inbound-dedupe.ts` module. This prevents replayed inbound events on Matrix client restart from being processed multiple times. The deduplication layer uses event IDs to track already-processed messages, mitigating replay attacks and duplicate message delivery.

#### Session Truncation After Compaction ([`c6968c39d`](https://github.com/openclaw/openclaw/commit/c6968c39d6))

Introduces `truncateSessionAfterCompaction()` in a new `src/agents/pi-embedded-runner/session-truncation.ts` module. After session compaction, the JSONL file is truncated to remove pre-compaction entries, preventing unbounded disk growth from long-running agent sessions. This addresses a denial-of-service vector where accumulated session files could exhaust disk space.

#### Telegram Wrong-Bot Routing Prevention ([`4e45a663e`](https://github.com/openclaw/openclaw/commit/4e45a663e7))

Hardens `resolveTelegramToken()` in `extensions/telegram/src/token.ts` to prevent silent wrong-bot routing when an `accountId` is not found in the configuration. Previously, a missing account would fall through to a default token, potentially routing messages to the wrong bot. The fix ensures explicit validation of the account lookup.
