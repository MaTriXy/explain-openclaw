#### Simplify auth rate limit coverage ([`29b9e21b7`](https://github.com/openclaw/openclaw/commit/29b9e21b7b))

The auth rate limiter test suite is simplified by introducing a `createLimiter()` helper that eliminates boilerplate across test cases. New coverage verifies that blank and whitespace scope inputs are normalized to the default scope, and that consecutive lock-triggering failures correctly extend the lockout window. Provides regression coverage for Audit 1 Claim 4 (rate control on token operations).

#### Tighten channel auth and network coverage ([`8f4e77e72`](https://github.com/openclaw/openclaw/commit/8f4e77e72f))

Channel authentication and network error path coverage is tightened across both channel-level auth checks and network failure scenarios. Tests verify that auth enforcement operates correctly under edge conditions. Provides regression coverage for Audit 1 Claim 2 (OAuth state handling in channel auth).

#### Simplify provider auth normalization coverage ([`572df9717`](https://github.com/openclaw/openclaw/commit/572df97179))

Provider auth normalization test coverage is simplified with shared setup helpers, reducing test complexity while improving readability. Verifies that provider-specific token normalization operates correctly. Provides regression coverage for Audit 1 Claim 1 (token storage normalization).

#### Refine gateway auth helper coverage ([`e25fa446e`](https://github.com/openclaw/openclaw/commit/e25fa446e8))

Gateway auth helper coverage is refined to verify authorization helper behavior across edge cases. Tests exercise the helper functions used by gateway auth enforcement paths. Provides regression coverage for Audit 1 Claim 5 (permission checks in gateway auth).

#### Align chat abort helpers with gateway handler types ([`ee1d4eb29`](https://github.com/openclaw/openclaw/commit/ee1d4eb29d))

Chat abort test helper exports are annotated with the correct types from the gateway handler type system, removing implicit `any` casts. Type-safety improvement with no behavior change; ensures future type-checked changes to abort handling are reflected in test infrastructure.

#### Preserve persona and language continuity in compaction summaries ([`72b6a11a8`](https://github.com/openclaw/openclaw/commit/72b6a11a83))

The context compaction pipeline is extended to preserve persona and language directives through compaction boundaries. Without this fix, a system-level persona override or language instruction could be silently discarded when a long conversation was compacted, causing the agent to revert to default behavior. The fix touches the extensions layer in `src/agents/pi-embedded-runner/extensions.ts` and related compaction files to carry forward these continuity directives into compacted summaries.

#### Add missing domain to wideArea Zod config schema ([`af4731aa5`](https://github.com/openclaw/openclaw/commit/af4731aa5f))

`domain: z.string().optional()` is added to the `wideArea` subschema in `src/config/zod-schema.ts`. The wide-area peer discovery config previously lacked a domain field in its Zod validator, meaning domain-scoped discovery configurations would fail silent validation. Completes schema coverage for network discovery options and removes a path where valid configurations could be quietly rejected.

#### Add groups config to Signal channel schema ([`61429230b`](https://github.com/openclaw/openclaw/commit/61429230b2))

The `groups` configuration field is added to the Signal channel schema. Signal channel configurations using group allowlists and group membership controls would previously fail runtime validation because the field was not declared in the schema, causing group-gated Signal deployments to silently reject their configuration.

#### Restore web fetch firecrawl config in runtime zod schema ([`4e68684bd`](https://github.com/openclaw/openclaw/commit/4e68684bd2))

The `firecrawl` subconfig for web fetch is restored to the runtime Zod schema. A prior refactor had removed this field from the runtime schema while it remained active in configurations, causing schema validation failures for deployments using Firecrawl-based web crawl configurations.

#### Add missing params field to agents.list[] validation schema ([`b72c87712`](https://github.com/openclaw/openclaw/commit/b72c87712d))

The `params` field is added to the `agents.list[]` schema. Agent list entries with explicit params overrides would fail Zod validation due to the missing field declaration. Completes schema coverage for agent configuration validation.

#### Harden perl approval binding ([`be8d51c30`](https://github.com/openclaw/openclaw/commit/be8d51c301))

The node-host exec approval binding gains explicit Perl flag hardening. `PERL_UNSAFE_APPROVAL_FLAGS` (`{"-I", "-M", "-m"}`) and `hasPerlUnsafeApprovalFlag()` are added to `src/node-host/invoke-system-run-plan.ts`. When Perl is invoked with `-I` (include directory injection), `-M` (module enable at startup), or `-m` (module use), `resolveMutableFileOperandIndex()` returns `null` — blocking the approval resolution path that would otherwise permit these flags to load code from attacker-controlled library or module paths. Mirrors the existing `hasRubyUnsafeApprovalFlag()` pattern for Ruby.

#### Harden pnpm approval binding ([`2f03de029`](https://github.com/openclaw/openclaw/commit/2f03de029c))

pnpm invocation unwrapping is hardened in `unwrapPnpmExecInvocation()` in `src/node-host/invoke-system-run-plan.ts`. Two new sets (`PNPM_OPTIONS_WITH_VALUE`, `PNPM_FLAG_OPTIONS`) improve parsing accuracy for pnpm option flags. The function now also handles `pnpm node` as a recognized subcommand in addition to `pnpm exec`, preventing `pnpm node` invocations from falling through as unrecognized patterns that could bypass the approval binding. A clarifying comment notes that the binding is best-effort for package-manager shims.
