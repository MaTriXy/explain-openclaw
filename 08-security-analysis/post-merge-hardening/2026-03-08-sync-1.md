#### Flatten remote markdown images ([`4bf902de5`](https://github.com/openclaw/openclaw/commit/4bf902de5))

Markdown rendering in both the HTML export template and the shared UI markdown renderer now blocks remote image URLs. The new `htmlEscapeRenderer.image` override in `ui/src/ui/markdown.ts` and a matching `renderMarkdownImage()` function in `src/auto-reply/reply/export-html/template.js` check whether an image's `href` matches `INLINE_DATA_IMAGE_RE` (the regex `/^data:image\/[a-z0-9.+-]+;base64,/i`). Any image whose href is not a base64 data URI is stripped to its escaped alt-text label via `normalizeMarkdownImageLabel()` — no `<img>` tag is emitted and no remote URL is fetched. This prevents an attacker from embedding `![x](https://attacker.com/pixel.png)` in a session message to trigger external requests or inject content into exported HTML. A dedicated `template.security.test.ts` file was added with XSS-specific tests covering remote URL stripping. iOS/macOS markdown preprocessing received the same defence via `ChatMarkdownPreprocessor.swift`.

#### Address markdown image review feedback ([`d25b493c7`](https://github.com/openclaw/openclaw/commit/d25b493c7))

Follow-up to `4bf902de5`. Refines the `renderMarkdownImage()` / `htmlEscapeRenderer.image` override based on code-review feedback: improves edge-case handling in `normalizeMarkdownImageLabel()` and extends `template.security.test.ts` with additional test cases covering boundary conditions (empty `href`, `null` text, `data:` URI variants). No behaviour regression — remote URLs remain blocked.

#### Namespace slash SessionKey by agent ([`e8f419c4e`](https://github.com/openclaw/openclaw/commit/e8f419c4e))

In `src/telegram/bot-native-commands.ts`, the Telegram slash-command `SessionKey` was changed from `telegram:slash:${senderId || chatId}` to `agent:${route.agentId}:telegram:slash:${senderId || chatId}`. Without the `agentId` prefix, two agents sharing a bot token could silently read each other's slash-command session state via session key collision. The new format scopes each key to the owning agent, preventing cross-agent session contamination in multi-agent Telegram deployments. Fixes issue #38648.
