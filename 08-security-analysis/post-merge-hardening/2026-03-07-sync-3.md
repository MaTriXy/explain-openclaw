#### Media: reject spoofed input_image MIME payloads ([`084dfd2ec`](https://github.com/openclaw/openclaw/commit/084dfd2ec))

Commit `084dfd2ec` hardens `normalizeInputImage()` in `src/media/input-files.ts` against MIME spoofing. Previously, `detectMime()` was called only for HEIC images. Now it is called unconditionally for all image inputs. A new guard checks whether the declared MIME type starts with `"image/"` while the detected content type does not; if so, the function throws `"Unsupported image MIME type"` and rejects the payload. This blocks attackers from smuggling non-image binaries — PDFs, executables, ZIP archives — disguised as base64-encoded images in `input_image` fields. One production file changed, one test, one CHANGELOG entry.
