#### Harden existing-session driver validation and session lifecycle ([`eee5d7c6b`](https://github.com/openclaw/openclaw/commit/eee5d7c6b0))

Config validation was incorrectly rejecting existing-session browser profiles that lack `cdpPort` or `cdpUrl` — these profiles use Chrome MCP auto-connect and don't need explicit port/URL fields. Fixed in `resolveProfile()`. A new `usesChromeMcp` capability flag (in `profile-capabilities.ts`) replaces approximately 20 scattered `driver === "existing-session"` string comparisons, providing a single authoritative check. Session teardown in `callTool()` (`chrome-mcp.ts`) now distinguishes transport/connection errors (which tear down the MCP session) from tool-level errors such as element not found (which preserve the session), preventing expensive `npx` re-spawns on transient failures. Shared ARIA role constants (`INTERACTIVE_ROLES`, `CONTENT_ROLES`, `STRUCTURAL_ROLES`) are extracted into a new `snapshot-roles.ts` module used by both the Playwright and Chrome MCP snapshot paths, ensuring consistent page snapshots across drivers.
