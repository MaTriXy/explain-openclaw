#### Enforce explicit ingress owner context ([`b8b8a5f31`](https://github.com/openclaw/openclaw/commit/b8b8a5f31))

Previously, agents dispatched via gateway or voice ingress could omit `senderIsOwner` from command context, leaving authorization state ambiguous at callsites. This commit introduces `agentCommandFromIngress()` as a mandatory wrapper around all ingress-path `agentCommand()` calls, requiring callers to explicitly pass `senderIsOwner: true/false`. A static analysis script (`check-ingress-agent-owner-context.mjs`) enforces at CI that no callsite bypasses the wrapper. Discord voice manager, gateway HTTP handlers, and server node event handlers were all updated. This closes a class of implicit privilege escalation where callsites could accidentally default to owner-level authorization.

#### Audit workspace skill symlink escapes ([`132794fe7`](https://github.com/openclaw/openclaw/commit/132794fe7))

Adds `collectWorkspaceSkillSymlinkEscapeFindings()` (in `src/security/audit-extra.async.ts`) to the security audit pipeline. The check scans workspace skill files (SKILL.md, agents.md, etc.) for symlinks that resolve outside the workspace root via `fs.realpathSync` canonicalization. Findings are reported with `checkId: "skills.workspace.symlink_escape"` at severity warn, capped at 12 detail rows and 2000 files per workspace scan. Prevents an attacker-controlled workspace from escaping its sandbox boundary by crafting symlinks in skill directories.

#### Retry Twilio signature verification without port in URL ([`c5ddba52d`](https://github.com/openclaw/openclaw/commit/c5ddba52d))

Twilio webhook signature validation in the voice-call extension previously rejected valid webhooks when the configured `publicUrl` included a non-standard port but Twilio signed the URL without it. Three helper functions are added to `webhook-security.ts`: `stripPortFromUrl()`, `setPortOnUrl()`, and `extractPortFromHostHeader()`. Verification now retries with multiple URL variants — including port-stripped, port-from-Host-header, and port-from-config — before failing closed. A legitimate webhook is no longer rejected due to port configuration mismatch, and no partial bypass is possible since all variants must fully verify.

#### Make /workspace bind mount read-only when workspaceAccess is not rw ([`903e4dff3`](https://github.com/openclaw/openclaw/commit/903e4dff3))

Fixes a logic bug in `appendWorkspaceMountArgs()` (`src/agents/sandbox/workspace-mounts.ts`) where the `:ro` suffix was never applied in docker containers. The old condition checked `workspaceAccess === 'ro' && workspaceDir !== agentWorkspaceDir`, which was always false in practice because the two paths were always equal in the ro case. The new logic is `workspaceAccess === 'rw' ? '' : ':ro'`, correctly making `/workspace` read-only inside the container when `workspaceAccess` is `'ro'` or `'none'`. This closes a sandbox write escape where container processes could modify host workspace files despite `workspaceAccess: ro` configuration.

#### Same fix for browser.ts ([`7cbcbbc64`](https://github.com/openclaw/openclaw/commit/7cbcbbc64))

Applies the identical `appendWorkspaceMountArgs()` fix to the browser sandbox (`src/agents/sandbox/browser.ts`). The same always-false condition was present in both the Docker and browser sandbox implementations. Both now consistently enforce `:ro` bind mounts when `workspaceAccess` is not `'rw'`.

#### Enforce sandbox workspace mount mode tests ([`02eeb08e0`](https://github.com/openclaw/openclaw/commit/02eeb08e0))

Adds parameterized test coverage in `docker.config-hash-recreate.test.ts` and `browser.create.test.ts` verifying that `workspaceAccess='ro'` and `workspaceAccess='none'` produce `/workspace:ro` bind mounts, and `workspaceAccess='rw'` produces writable mounts. Prevents regression of the sandbox workspace mount permission fix.

#### system.run approval cwd revalidation refactor ([`dc825e59f`](https://github.com/openclaw/openclaw/commit/dc825e59f))

Companion refactor extracting `invoke-system-run-plan.ts` from `invoke-system-run.ts` to centralize the approval planning phase. Creates a clean separation where the plan phase (parse, classify, check allowlists) is separate from the execute phase. This structural change enables the TOCTOU fix in `500d7cb10` to cleanly inject the CWD snapshot check at the boundary between plan and execute.

#### Revalidate approval cwd before system.run execution ([`500d7cb10`](https://github.com/openclaw/openclaw/commit/500d7cb10))

Fixes a TOCTOU (time-of-check/time-of-use) vulnerability where the approved working directory for `system.run` commands could be replaced between user approval and actual execution. Introduces `revalidateApprovedCwdSnapshot()` in `invoke-system-run-plan.ts` which captures a filesystem stat (device + inode via `sameFileIdentity()`) at approval time. Before execution, the snapshot is compared to the current filesystem state; if drift is detected (symlink swap, directory replacement), execution is denied with `APPROVAL_CWD_DRIFT_DENIED_MESSAGE`. Addresses GHSA-h3rm-6x7g-882f.

#### Fail-close camera URL downloads ([`3bf19d6f4`](https://github.com/openclaw/openclaw/commit/3bf19d6f4))

Camera node URL downloads previously used unrestricted HTTP fetching, allowing SSRF via maliciously constructed node URLs. This commit switches camera URL fetching in `nodes-camera.ts` to `fetchWithSsrFGuard()` with explicit `expectedHost` validation. `normalizeHostname()` canonicalizes both the expected and actual hostnames before comparison; any mismatch, non-https protocol, or missing `expectedHost` fails closed with an error. Content-length limits are also enforced via the guard. Addresses Audit 2 Claim 4 and GHSA-2858-xg23-26fp.

#### ACP sandbox inheritance enforcement ([`ac11f0af7`](https://github.com/openclaw/openclaw/commit/ac11f0af7))

Adds `SpawnAcpSandboxMode` type (`"inherit" | "require"`) to `src/agents/acp-spawn.ts`. When `spawnAcpDirect()` is called from a sandboxed context, `resolveSandboxRuntimeStatus()` now checks the requester's runtime state and blocks the spawn if the requester is sandboxed — preventing a sandboxed sub-agent from escalating to an unsandboxed ACP session. System prompt updated to inform agents of this restriction. Addresses GHSA-474h-prjg-mmw3.

#### Strengthen prompt injection warning for weaker models ([`11c397ef4`](https://github.com/openclaw/openclaw/commit/11c397ef4))

Documentation-only update to `docs/gateway/security/index.md` improving prompt injection risk warnings for users deploying weaker LLM models that have reduced instruction-following reliability. No code changes.
